Elastic IP Addresses

By binding a public IP address to the VMs or private IP addresses associated with a routed network, the elastic IP address service allows services in a virtual private cloud (VPC) accessible to external services using a fixed public IP address.

For example, if an internal server in the VPC needs to provide web services externally, the internal server can use an elastic IP address for Domain Name Server (DNS) resolution in the web service access. If the backup VM in the VPC needs to provide backup services externally, the backup VM can have an elastic IP address bound to provide a stable backup service access IP address. In both of these two scenarios, external users can always use a fixed IP address to access the external services provided by a server or VM in the VPC. This service ensures service continuity.

Binding an Elastic IP Address

Bind the obtained elastic IP address to the VM or private IP address associated with a routed network, so that the VM can access the public network using a fixed public IP address.

Unbinding the Elastic IP Address

If a VM is not required to provide public network access services or the VM needs to be replaced without changing the public IP address, you can unbind the elastic IP address from the VM.
