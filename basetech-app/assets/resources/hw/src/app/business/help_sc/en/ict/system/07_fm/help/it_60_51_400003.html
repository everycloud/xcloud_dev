A virtual load balancer (VLB) provides load balancing services for multiple VMs running the same service in a virtual private cloud (VPC).

Creating a VLB

Create a software VLB and bind it to a VM to provide load balancing services for the VM.

Choose VPC > My VPC, select a VPC, navigate to the VLB page, and click Create.

Modifying VLB Basic Information

Modify basic information about a VLB, such as the VLB name, maximum number of supported sessions, and allocation policies.
NOTE:

After the VLB basic information is modified, the VLB session stickiness settings must also be modified accordingly.

Choose VPC > My VPC, select a VPC, navigate to the VLB page, locate the row that contains the target VLB, and click Modify Basic VLB Info.

Modifying Session Stickiness Information

Modify session stickiness parameters of a VLB.

Choose VPC > My VPC, select a VPC, navigate to the VLB page, locate the row that contains the target VLB, click More, and select Modify Session Stickiness Info.

Modifying Health Check Information

Modify health check parameters of a VLB.

Choose VPC > My VPC, select a VPC, navigate to the VLB page, locate the row that contains the target VLB, click More, and select Modify Health Check Info.

Binding a VM to a VLB

Bind a VM to a VLB to allow the VM to use load balancing services.

Choose VPC > My VPC, select a VPC, navigate to the VLB page, locate the row that contains the target VLB, click More, and select Bind VM.

Unbinding a VM from a VLB

Unbind a VM from its bound VLB to disable the VM to provide external services using the VLB.

Choose VPC > My VPC, select a VPC, navigate to the VLB page, locate the row that contains the target VLB, click More, and select Unbind from VM.

Binding an Elastic IP Address to a VM NIC

Binding an elastic IP address to a VLB allows the VLB external IP address to be bound to the elastic IP address, so that users can access the service VMs associated with the VLB using the elastic IP address. Before elastic IP address, ensure that you have applied for an elastic IP address for the VPC.

Choose VPC > My VPC, select a VPC, navigate to the VLB page, locate the row that contains the target VLB, click More, and select Bind Elastic IP Address.

Unbinding the Elastic IP Address from a VLB

After the elastic IP address of a VLB is unbound from the VLB, the VLB will not support access from public networks.

Choose VPC > My VPC, select a VPC, navigate to the VLB page, locate the row that contains the target VLB, click More, and select Unbind Elastic IP Address.

Deleting a VLB

If a VLB is not needed any more, you can delete it from the system.

Choose VPC > My VPC, select a VPC, navigate to the VLB page, locate the row that contains the target VLB, click More, and select Delete VLB.
