Interface Function Introduction

Tenants can manage and monitor services, resources, and applications in their own VDCs as well as manage rights of users in the VDCs. For details, see Table 1.
Table 1 Functions of the tenant view

SN

Category

Functions

1

Services

Services refer to resource instances provided for users. Users can apply for cloud host, cloud hard disk, and virtual data center (VDC) services as required. The VDC administrator can create application services for VDC users.

2

Resources
Resources refer to all virtual and dedicated resources managed by ServiceCenter. You can perform the following operations on these resources:
  • Create and manage VMs.
  • Query dedicated servers.
  • Manage disks.
  • Manage the templates.

3

VPC

A VPC provides a secure, isolated network environment for the VDC. You can customize virtual networks that serve as traditional networks in the VPC and also provide advanced network features, including elastic IP address and NAT, to keep up with service deployment requirements.

4

Applications

An application instance is a collection of VMs and service software that provides specific services, for example, SQL Server and CRM/ERP.

You can manage application instances and application templates.

5

Monitoring
Monitored objects include performance, tasks and logs.
  • Performance: The performance list of ServiceCenter provides performance monitoring indicators of the VMs.
  • Tasks and logs: you can view the task status and operation logs.

6

Users
You can manage users, roles, and domains.
  • A role is a collection of diverse operation rights. You can assign roles to users so that each user can have different operation rights.
  • Domains are used to classify resources. You can customize domains at the geographical or administrative level. By associating domains with different users and resources, you can enable users to manage resource in their own domains.
