Background

A service is a resource instance template that users can use. The ServiceCenter system currently supports virtual data center (VDC), cloud host (virtual machine), cloud hard disk, dedicated server, elastic IP address, and application services.

This section uses the cloud hard disk application as an example to describe how to create, apply for, approve, and provision a service in ServiceCenter. This section is intended for you to have an overall understanding of the service provisioning process and tasks of each role in ServiceCenter.

Process Introduction

After a resource pool is created using Local FusionManager, the administrator, VDC administrator, and VDC user can access, apply for, and approve resources using ServiceCenter to provision resources to VDC users.

User Rights Introduction

ServiceCenter provides a user management system in which users are assigned with different rights. This system involves the administrator, VDC administrator, and VDC users.
  • The administrator is in charge of the cloud resource access and configuration, VDC resource management, service creation, and system configuration.
  • The VDC administrator is in charge of the service creation, service application approval, and VDC resource (such as VM) management.
  • Users are in charge of the application and management of VDC resources (such as VMs).
The administrator, VDC administrator, and users complete different tasks to provision services together.

Prerequisites

  • The administrator account and password for logging in to ServiceCenter are obtained and ServiceCenter is logged in to using the administrator account.
  • The FusionSphere OpenStack has been installed and configured.

Operation Procedures

    Add resources to ServiceCenter.

    1. Add resources to ServiceCenter. For details, see Initial Installation and Configuration > Adding Resources to ServiceCenter in ServiceCenter V100R002C20 Software Installation Guide.

    Create the VDC service, cloud hard disk service, and VDC administrator.

    1. Choose Services > Services Catalog.
    2. Click .

      The Catalog Management page is displayed.

    3. Click Create Catalog.
    4. Set the catalog name and description information, and click OK.

      Assume that the name of the created catalog is catalog.

    5. Choose Services > Services Catalog > catalog.
    6. Click Create.
    7. Select the Apply for VDC template.
    8. Click Next.

      The Create VDC Service-Basic Info page is displayed.

    9. Enter basic information about the VDC.

      Assume that the name of the the VDC to be created is vdc_sz.

      Services Catalog is set to the directory created in 5.

    10. Click Next.

      The Resource Quotas page is displayed.

    11. Select a resource quota setting mode:

      • Specify When Applying for Service: set by an applicant when the applicant applies for the service.
      • Specify When Approving Application: set by an administrator during resource approval based on user requirements.
      • Lock: set by an administrator when the administrator creates the service.

      If you select Lock, configure VDC properties, such as the number of vCPUs.

    12. Click Next.

      The Region page is displayed.

    13. Select a region setting mode.

      • Specify When Applying for Service: set by an applicant when the applicant applies for the service.
      • Specify When Approving Application: set by an administrator during resource approval based on user requirements.
      • Lock: set by an administrator when the administrator creates the service.

      If you select Lock, configure a region. Select a region from the Region drop-down list box. The selected region must exist in Services > Cloud Resource Pool.

    14. Click Next.

      The Confirm Information page is displayed.

    15. Click Create.

      The All Services list is displayed. The VDC service is created successfully if it is displayed in the list.

    16. Click Create to create a cloud disk service.
    17. Select Apply for Disk and select an approval type.

      Assume that Approved by is set to VDC Administrator.

    18. Click Next.

      The Create Block Storage Service-Location page is displayed.

    19. Select a region setting mode.

      • Specify When Applying for Service: set by an applicant when the applicant applies for the service.
      • Specify When Approving Application: set by an administrator during resource approval based on user requirements.
      • Lock: set by an administrator when the administrator creates the service.

      If you select Lock, configure a region and available zone. Select a region from the Region drop-down list box. The selected region must exist in Services > Cloud Resource Pool.

    20. Click Next.

      The Configure Storage page is displayed.

    21. Configure cloud disk parameters, such as the storage capacity and type.
    22. Click Next.

      The Basic Info page is displayed.

    23. Configure the basic information about the cloud disk.

      Assume that Name is set to disk.

      Services Catalog is set to the directory created in 5.

    24. Click Next.

      The Confirm Information page is displayed.

    25. Ensure all the information is correct and click Create.

      The All Services list is displayed. The cloud disk service is created successfully if it is displayed in the list.

    26. On the catalog page, click Publish in the vdc_sz and disk areas respectively.
    27. Choose User > User Management.

      The User Management page is displayed.

    28. Click Create User.
    29. Set VDC administrator parameters as prompted.

      Focus on the setting of the following parameters:

      • Set Role Type to Service Management Roles.
      • Set Role to vdcServiceManager.

    30. Click Create.

      When the system returns to the user list and the newly created user is displayed, the creation is successful.

    The VDC administrator applies for a VDC service.

    1. Log out of the system, and select the tenant view to log in to the system as the user created in 30.
    2. Choose Services > Services Catalog > catalog.
    3. In the vdc_sz area, click Apply For.
    4. Set parameters as prompted, and click Submit.

    The administrator approves a VDC service.

    1. Log out of the system, and use the administrator account to log in to the ServiceCenter system.
    2. Choose Services > My Todo List.
    3. Confirm the application order information, including the order number and applicant, locate the row that contains the application order to be approved, and click Approve in the Operation column.
    4. Approve the service application as prompted.
    5. Click All Applications to view the application status.

      When the status is Succeeded, the resource is successfully provisioned.

    The VDC administrator creates a user.

    1. Log out of the system, and select the tenant view to log in to the system as the user created in 30.
    2. Click Users.
    3. Click Create User.
    4. Set user parameters as prompted.

      Set Role to serviceUser.

    5. Click Create.

      When the system returns to the user list and the newly created user is displayed, the creation is successful.

    Users submit service applications.

    1. Log out of the system, and select the tenant view to log in to the system as the user created in 44.
    2. Choose Services > Services Catalog > catalog.
    3. In the disk area, click Apply For.
    4. Set parameters as prompted.
    5. Click Submit.

    The VDC administrator approves an application.

    1. Log out of the system, and select the tenant view to log in to the system as the user created in 30.
    2. Choose Services > My Todo List.
    3. Confirm the application order information, including the order number and applicant, locate the row that contains the application order to be approved, and click Approve in the Operation column.
    4. Approve the service application as prompted.
    5. Click All Applications to view the service provisioning status.

      When the status is Succeeded, the service is successfully provisioned. VDC users can use the cloud hard disk that is applied.
