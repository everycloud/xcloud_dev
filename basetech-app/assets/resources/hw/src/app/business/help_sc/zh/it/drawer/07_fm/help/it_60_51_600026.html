如何配置虚拟机进程监控功能

操作场景

Hyperic HQ是一个基于Java的软件资源监测和管理平台。部署了Hyperic HQ软件的服务端和客户端之后，便可以通过该软件监控虚拟机中的进程。其中：
  • PVM虚拟机中部署Hyperic HQ软件服务端，由管理员制作PVM虚拟机模板时手动安装。在VPC中部署服务后，PVM虚拟机上便已具备该软件。
  • 待监控虚拟机中部署Hyperic HQ软件客户端，需要用户根据实际要求，手动在虚拟机中安装软件。
目前支持使用的Hyperic HQ版本为“4.6.6”，其对于操作系统的限制和所能监控的软件进程，请以“4.6.6”版本的Hyperic HQ软件为准。

为Linux操作系统虚拟机配置进程监控功能

    1. 将功能所需的安装软件，全部放入一个ISO文件中，以便通过挂载光驱的方式，将安装文件拷贝至虚拟机中。

      目前业界已有多款优秀的ISO镜像制作工具，如UltraISO、ISOMaker等，使用这些工具能够轻松将各类文件打包制作成为ISO文件。可通过互联网或技术论坛获取更多信息。

      后续步骤中ISO文件名称以“hyperic.iso”为例，其中包含“hyperic-hq”文件夹。

    2. VNC登录虚拟机，并挂载文件“hyperic.iso”。

      挂载光驱的具体操作方法，请参考挂载光驱。

    3. 若使用图形化操作系统，则右键单击虚拟机操作系统桌面，在弹出的快捷菜单中选择“在终端中打开”。

      进入命令行模式。

    4. 执行以下命令，创建“xvdd”目录。

      mkdir xvdd

    5. 执行以下命令，挂载虚拟机光驱。

      mount 挂载路径 xvdd

      例如：mount /dev/sr0 xvdd

      根据Linux操作系统版本不同，挂载路径不同，如表2所示。

    6. 执行以下命令，将ISO中的“hyperic-hq”文件夹拷贝到虚拟机的“/home”目录下。

      cp -r xvdd/hyperic-hq/ /home

    1. 执行以下命令，进入“/home/hyperic-hq”目录。

      cd /home/hyperic-hq/

    2. 执行以下命令，解压安装包文件。

      tar xvf 安装包文件名

      例如：tar xvf hyperic-hq-agent-4.6.6-x86-linux.tar

    3. 执行以下命令，修改解压后的安装包目录执行权限。

      chmod -R 775 hyperic-hq-agent-4.6.6/

    4. 执行以下命令，进入“hyperic-hq-agent-4.6.6”中的“conf”目录，使用VI编辑器打开“agent.properties”文件。

      cd hyperic-hq-agent-4.6.6/conf

      vi agent.properties

    5. 按“i”进入编辑状态，并在文件末尾增加如下语句，设置虚拟机名称。

      虚拟机的名称在VPC内要求是唯一的。例如，增加如下语句，设置虚拟机的名称为“vm01”。

      platform.name=vm01

    6. 按“ESC”，输入:wq，保存修改并退出VI编辑器。
    7. 执行以下命令，进入“hyperic-hq-agent-4.6.6”中的“bin”目录。

      cd /home/hyperic-hq/hyperic-hq-agent-4.6.6/bin

    8. 执行以下命令，安装虚拟机进程监控功能。

      sh hq-agent.sh start

    1. 输入HQ服务器的IP地址。

      HQ服务器的IP地址是指能够与该虚拟机互通的PVM虚拟机的IP地址，即虚拟机使用的网络中至少有一个与PVM虚拟机使用的网络互通。

      PVM虚拟机的IP地址可在VPC下的“部署服务”页签中查询，可在虚拟机中通过ping命令来判断虚拟机与PVM虚拟机的哪一个IP地址互通。

      What is the HQ server IP address: 192.168.5.14

    2. 输入其他连接参数，均使用默认值。 

      Should Agent communication to HQ always be secure [default=yes]: yes
What is the HQ server SSL port [default=7443]: 7443
      回显如下信息，表示连接成功。
      Testing secure connection ... Success

    3. 输入监控服务端帐户、密码。

      监控服务端帐户为“hqadmin”，默认密码为“Huawei@123”。

    4. 输入当前虚拟机的IP地址和端口，端口使用默认值。 

      What IP should HQ use to contact the agent [default=192.168.5.25]: 192.168.5.25
What port should HQ use to contact the agent [default=2144]: 2144

    5. 输入yes确认。

      回显如下信息，表示进程监控功能安装和配置成功。大约5分钟后，即可在进程监控页面中显示监控信息。
      Are you sure you want to continue connecting? [default=no]: yes

为Windows操作系统虚拟机配置进程监控功能

    1. 将功能所需的安装软件，全部放入一个ISO文件中，以便通过挂载光驱的方式，将安装文件拷贝至虚拟机中。

    2. VNC登录虚拟机，并挂载文件“hyperic.iso”。

      挂载光驱的具体操作方法，请参考挂载光驱。

    3. 将ISO中的“hyperic-hq”文件夹拷贝到虚拟机的C盘根目录下。
    4. 解压缩安装包文件。
    5. 进入解压后的安装包文件夹中的“conf”目录，使用记事本打开“agent.properties”文件。
    6. 在文件末尾增加如下语句，设置虚拟机名称。

      platform.name=vm01

    7. 保存修改并关闭文件。
    8. 打开虚拟机命令提示符窗口。
    9. 执行以下命令，进入解压后的安装包文件夹中的“bin”目录。

      例如：cd \hyperic-hq\hyperic-hq-agent-4.6.6\bin

    10. 执行以下命令，安装虚拟机进程监控功能。

      hq-agent.bat install

      回显如下信息，表示安装成功。
      wrapper  |  Hyperic HQ Agent service installed.

    11. 执行以下命令，启动并配置虚拟机进程监控功能。

      hq-agent.bat start

      回显如下信息，表示启动成功。
      wrapper  |  Hyperic HQ Agent started.

      PVM虚拟机的IP地址可在VPC下的“部署服务”页签中查询，可在虚拟机中通过ping命令来判断虚拟机与PVM虚拟机的哪一个IP地址互通。

      Should Agent communication to HQ always be secure [default=yes]: yes
What is the HQ server SSL port [default=7443]: 7443
      回显如下信息，表示连接成功。
      Testing secure connection ... Success

    3. 输入监控服务端帐户、密码。

      监控服务端帐户为“hqadmin”，默认密码为“Huawei@123”。

    4. 输入当前虚拟机的IP地址和端口，端口使用默认值。

FAQ-如何更新配置

    当虚拟机已经按照上述方法，完成Hyperic HQ软件客户端的安装和配置，并能够正常使用进程监控功能过程中，若管理员由于PVM虚拟机故障等原因修复或重新部署了PVM虚拟机，则需使用新的PVM虚拟机IP地址更新虚拟机中的Hyperic HQ软件客户端配置后，进程监控功能才可继续使用。

    1. 进入Hyperic HQ软件安装目标目录下的“hyperic-hq-agent-4.6.6\bin”目录中。
    2. 执行以下命令，停止Hyperic HQ Agent服务。

      • 若虚拟机为Windows操作系统，执行 hq-agent.bat stop
      • 若虚拟机为Linux操作系统，执行 sh hq-agent.sh stop

    3. 删除“data”文件夹。
    4. 执行以下命令，启动并重新进行配置。

      • 若虚拟机为Windows操作系统，执行 hq-agent.bat start
      • 若虚拟机为Linux操作系统，执行 sh hq-agent.sh start
