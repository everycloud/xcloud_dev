Roles

A role is a predefined set of operation permissions.

After the Keystone service is added to ServiceCenter, the system provides two default roles:

  • vdcServiceManager: the default service management role, which has rights to manage all services in the VDC to which the user with this role belongs. If the user with this role has not been added to the VDC, the user is only allowed to apply for VDC services, manage request orders, and view role information.
  • serviceUser: the default service management role, which has rights to perform operations under Home, Services, Resources, Application, and Monitoring pages. This role does not have user management, service approval, and VPC operation rights (except for the security group and elastic IP address services), but it is allowed to view VPC and service catalog information.
