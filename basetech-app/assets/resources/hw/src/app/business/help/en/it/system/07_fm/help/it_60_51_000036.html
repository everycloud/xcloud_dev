Overview

A security group is a set of rules that isolate or control the VM access. After a security group is created on the FusionManager, the administrator can define various access rules for the security group. The VMs added to the security group are subject to these access rules and therefore are protected.

Principles

The security group feature uses the packet filtering firewall Iptables attached with the Linux operating system (OS) to filter the packets entered to a host OS. Iptables provides rules for packet filtering. The filtering rules are defined based on source addresses, destination addresses, and protocols.

After you add a network interface card (NIC) of a VM to a security group created on the FusionManager, Iptables creates a user link based on the security group ID and adds the NIC to the link. A new user link has a default rule "DROP ALL". New rules of the security group are added to this link. Before forwarding a data packet, the host matches the packet with the rules in the user link. It only forwards the packet that matches with the rules to the VM NIC.

Access mechanism of the security group

By default, the system allows all data packets that are sent out from VMs in a security group and rejects all the data packets that are sent to the VMs in a security group. You can add rules for a security group to control the data packets that are sent to the VMs in the security group. It is like whitelist management function. Precautions

Before using the security group function, pay attention to the following precautions:

  • Only the VMs in the FusionCompute hypervisor can be added to a security group, and one NIC of a VM can be added only to one security group.
  • The security group function cannot solve the problems caused by network fault or incorrect network configuration. For example, when two VMs cannot communicate with each other due to network configuration reasons, they still cannot communicate even if you configure a security group rule to allow them to communicate.
  • When the host on which a VM runs uses an intelligent NIC, the VM cannot be added to a security group.
