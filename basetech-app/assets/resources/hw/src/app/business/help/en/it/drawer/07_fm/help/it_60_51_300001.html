Application Instances

An application instance is a collection of VMs and service software that provides specific services, for example, SQL Server, customer relationship management (CRM), and enterprise resource planning (ERP).

Creating an Application Instance

You can create an application instance using one of the following methods:

  1. Create an application instance using an application template.

    Provision an application instance using the application template configured for the desired service model based on the service plan.

  2. Create a custom application instance.

    Use existing VMs to create an application instance so that these VMs can be managed by the application instance.

To ensure VM security, the VM user must change the VM login password in the earliest time.

Application Instance Power Management

Power management for an application instance includes start and stop the application instance. Essentially, stopping an application instance stops all VMs included in the application instance, and start the application instance starts all VMs in the application instance.

Locate the row that contains the target application instance and select the desired operation to perform.

Modifying Application Instance Attributes

Modifying Application Instance Attributes

You can modify the name and description of an application instance as needed.

Locate the row that contains the target application template, click More, and select Modify.

Transferring the Application Instance Owner

Only the VDC administrator is allowed to transfer the application instance ownership to other tenants in the same VDC.

Locate the row that contains the target application template, click More, and select Transfer.

Monitoring Processes of VMs in an Application Instance

Based on the Hyperic HQ architecture, FusionManager supports monitoring of service VM processes with the following conditions met:
  1. A Proxy VM (PVM) is available in the virtual private cloud (VPC), and the network associated with the PVM VM is connected to the network carrying the service VM to be monitored.
  2. The service VM has the Hyperic HQ software client installed and configured. For details about Hyperic HQ client configuration, see Configuring the VM Process Monitoring Function.
NOTE:

If the administrator has restored or redeployed the PVM due to causes such as PVM faults, the Hyperic HQ clients deployed on VMs can be used only after being reconfigured using the IP address of the new PVM. For details, see FAQ-How Do I Update the Configurations? in Configuring the VM Process Monitoring Function.

Click the name of the target application instance, navigate to Monitoring > Process, and view the VM process monitoring information.

Deleting an Application Instance

If an application instance is not needed any more, you can delete it from the system. The deletion impact varies according to the application instance creation mode.
  • If you delete an application instance that is created using an application template, all VMs included in the application instance and all disk data on the VMs will be deleted. Therefore, exercise caution when deleting such an application instance.
  • If you delete a custom application instance, the VMs included in the application instance will no be affected. The deletion only removes the application instance to which the VMs belong, and the VMs resume to their independent running status.

Locate the row that contains the target application instance, click More, and select Delete.
