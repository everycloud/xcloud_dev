Scheduling Policies

Scheduling policies are used to adjust resource workloads for scaling groups in an application instance.

A scaling group is a collection of VMs in an application instance. With scheduling policies, the system automatically adds new VMs or start stopped VMs in a scaling group if the VM workloads in the group are generally heavy. If the VM workloads are generally light, the system automatically deletes VMs from the group. This function balances the VM workloads in an application instance and maximizes the resource utilization.

Configuring Scheduling Policies

You can configure scheduling policies for an application instance using one of the following methods:

  1. Configure scheduling polices in the application template.

    When creating an application template, you can add scaling groups for the selected VMs and configure scheduling policies for the VMs. Therefore, the application instances created using this application template automatically contains the configured scheduling policies.

    For details, see Creating an Application Template.

  2. Configure scheduling polices in the application instance.

    After creating an application instance, navigate to the application instance topology, add scaling groups for the VMs in the application instance, and configure scheduling policies for the VMs.

    Choose Application > Application Instance, select an application instance, navigate to the Topology page, click a scaling group icon and add scheduling policies for the group.

Configuring Scheduled Tasks for Scheduling Policies

Configure scheduled tasks or periodic tasks for the system to automatically execute or disable the configured scheduling policies.

Choose Application > Policy Management, navigate to the Scheduled Task page, and click Create.

The system records logs about all the executed scheduled tasks. To view the task logs, choose Application > Policy Management and click Policy Log.
