Roles

  • Creating a Role
  • Modifying a Role
  • Deleting a Role

A role is a predefined set of operation permissions.

After FusionManager is installed, the system provides two default roles:

  • vdcmanager: a service management role, which has all virtual data center (VDC) service management permissions
  • user: a service management role, which has operation rights including homepage configuration and viewing, monitoring, resource management, and application management. This role does not have user management and VPC operation rights (excluding security group and elastic IP address operation) but is allowed to view the VPC information.

Creating a Role

If the default roles cannot meet the operation permission control requirements, you can create a role and specify permissions for it as required.

Choose User > Role Management and click Create Role.

Modifying a Role

To adjust the permissions of a custom role, you can modify the role permissions.

The permissions of the default roles cannot be modified.

Choose User > Role Management, locate the row that contains the target role, and click Modify.

Deleting a Role

If a role is not needed any more, you can delete it from the system. Before deleting a role, ensure that the role is not in use by any user in the system. If a user is using this role, change the user's role to other roles.

The default roles cannot be deleted.

Choose User > Role Management, locate the row that contains the target role, click Delete.
