操作场景

通过FusionManager，将ISO文件或者本地PC的光驱挂载给虚拟机，用于为虚拟机安装操作系统或业务软件。挂载方式如表1所示，用户可根据实际情况选择恰当的方式。
表1 挂载光驱对应的操作系统类型

挂载方式

优势

限制

本地
  • 支持挂载ISO文件和物理光驱。
  • 操作简单。

仅适用于如下操作系统。

  • SUSE 10 sp版本、SUSE 11及以上sp版本
  • Ubuntu 11.10 server及以上版本
  • Ubuntu 12.04 desktop及以上版本
  • Ubuntu 12.04 Server及以上版本
  • Ubuntu 12.10 Server
  • RedHat 3.0 32位
  • RedHat 3.4 32位
  • CentOS 6.1及以上版本
  • RedHat 6.1及以上版本
  • Oracle Linux Server release 6.1
  • Debian GUN/Linux 6.0.4及以上版本
  • openSUSE 11.3 64位
  • Red Flag Asianux Server 4 SP2
  • 所有Windows操作系统

共享

无操作系统限制。
  • 仅支持挂载ISO文件，不能挂载物理光驱。
  • 需要配置ISO文件的共享参数。

注意事项

  • 每个虚拟机只能挂载一个光驱。
  • 一个主机上，同时最多仅能有4个虚拟机挂载光驱。因此，需及时卸载光驱，避免该主机上的其他虚拟机挂载光驱失败。
  • 虚拟机挂载光驱后，不能进行迁移。
  • 虚拟机关闭、休眠或HA后，所挂载的光驱会自动卸载；使用本地方式挂载光驱时，若VNC窗口被关闭，光驱会自动卸载。

前提条件

  • 登录FusionManager所使用的本地PC需与FusionManager的管理网络互通。若网络隔离，则可安装并配置VPN软件，通过VPN访问管理网络，具体操作请参考安装及配置VPN（Windows）。
  • 已登录FusionManager。
  • 虚拟机的状态为“运行中”。

本地方式挂载光驱

  1. 使用VNC方式登录待挂载光驱的虚拟机。

    挂载光驱过程中，禁止关闭VNC登录窗口，否则将自动卸载光驱。

  2. 如图1所示，单击，并选择“挂载光驱(本地)”。

    图1 挂载光驱(本地)

  3. 如图2所示，选择光驱所在路径。

    • 挂载物理光驱：选择“CD驱动器”，并选择光驱所在路径。
    • 挂载ISO文件：选择“文件”，单击“浏览”，选择ISO文件。
    图2 挂载光驱(本地)

  4. 若挂载操作系统安装文件，为虚拟机安装操作系统，则勾选“立即重启虚拟机，安装操作系统”。
  5. 单击“确认”。

    若已勾选“立即重启虚拟机，安装操作系统”，则需在弹出的“失去连接”对话框中单击“是”，重新连接虚拟机。否则VNC登录窗口关闭后，将自动卸载光驱。

共享方式挂载光驱

  1. 使用VNC方式登录待挂载光驱的虚拟机。

    挂载光驱过程中，禁止关闭VNC登录窗口，否则将自动卸载光驱。

    说明：

    若在创建虚拟机模板时，也可以在FusionManager界面中配置共享参数。

  2. 如图3所示，单击，并选择“挂载光驱(共享)”。

    图3 挂载光驱(共享)

  3. 如图4所示，配置ISO文件的共享参数。

    图4 挂载光驱(共享)

  4. 若挂载操作系统安装文件，为虚拟机安装操作系统，则勾选“立即重启虚拟机，安装操作系统”。
  5. 单击“确认”。

    若已勾选“立即重启虚拟机，安装操作系统”，则需在弹出的“失去连接”对话框中单击“是”，重新连接虚拟机。否则VNC登录窗口关闭后，将自动卸载光驱。
