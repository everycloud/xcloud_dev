An application instance is a collection of VMs and service software that provides specific services, for example, SQL Server, customer relationship management (CRM), and enterprise resource planning (ERP).

Creating an Application Instance

Select the application template based on the service plan to create an application instance.

Choose Application > Application Instance and click Create.

To ensure VM security, the VM user must change the VM login password in the earliest time.

Application Instance Power Management

Power management for an application instance includes start and stop the application instance. Essentially, stopping an application instance stops all VMs included in the application instance, and start the application instance starts all VMs in the application instance.

Choose Application > Application Instance, locate the row that contains the target application instance, and select the desired operation to perform.

Modifying Application Instance Attributes

Modifying Application Instance Attributes

You can modify the name and description of an application instance as needed.

Choose Application > Application Instance, locate the row that contains the target application template, click More, and select Modify.

Transferring the Application Instance Owner

Only the VDC administrator is allowed to transfer the application instance ownership to other tenants in the same VDC.

Choose Application > Application Instance, locate the row that contains the target application template, click More, and select Transfer.

Deleting an Application Instance

If an application instance is not needed any more, you can delete it from the system. After an application instance is deleted, all VMs included in the application instance and all disk data on the VMs will be deleted. Therefore, exercise caution when performing this operation.

Choose Application > Application Instance, locate the row that contains the target application instance, click More, and select Delete.

FAQ

After I created an application instance using an application template, why are the VMs included in the created application instance always started from the image file whatever the boot device setting is configured in the VM specifications used by the VMs?

The boot device setting configured in VM specifications takes effect only when VMs are being created. When you use an application template to create an application instance, the VMs included in the application instance start using the image file by default. You can use one of the following methods to change the boot device setting for a VM included in an application instance:
  1. Create a VM using an image and the VM specifications in which the boot device is set to Cloud Hard Disk.
  2. On OpenStack, create an image using this VM and a command, for example, nova image create.
  3. Create the application instance again using the new image.
