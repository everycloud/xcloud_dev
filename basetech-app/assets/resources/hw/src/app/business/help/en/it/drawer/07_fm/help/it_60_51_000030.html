Creating a VPC and Networks

Overview

A virtual private cloud (VPC) provides tenants with isolated and secure networks. You can create virtual networks in a VPC to provide tenants with the same network functions as those provided by a traditional network. Those virtual networks can also run VMs and application instances.

FusionManager supports three types of networks in a VPC: direct network, internal network, and routed network.

Direct Network

A direct network itself contains no network resources, but it has direct access to external networks. Therefore, VMs created on a direct network actually uses the IP addresses provided by the connected external network, which can be the existing network of the customer or the Internet. Figure 1 Direct network

Internal Network

An internal network uses exclusive virtual local area network (VLAN) or subnet resources. Therefore, it can be used for deploying services that have high security requirements, because it is isolated from other networks. For example, you can deploy the database server on an internal network to prevent the server from being accessed by users from other networks.

Routed Network

A routed network provides flexible communication functions and supports various services. Routed networks in a VPC can communicate with each other or the Internet using elastic IP addresses or the network address translation (NAT) function. Figure 2 shows a routed network.A routed network can also provide advanced network functions, such as access control list (ACL) and virtual private network (VPN).

Figure 2 Routed network

Before creating a routed network in a VPC, ensure that you have applied for a virtual router for the VPC. FusionManager supports two types of routers: hardware routers and software routers. A hardware router is a firewall instance created on a physical firewall to provide router functions. A software router is a system service VM created on FusionManager to provide router functions.

Network Creation Process

Before creating a network in a VPC, ensure that the resources listed in Table 1 are available on FusionManager.
Table 1 Network creation process

Resource

Description

Web Client Navigation

VLAN pool

 The VLANs of the service networks (including external networks, internal networks, and routed networks) created on FusionManager must be added to the VLAN pool on FusionManager. The VLAN pool takes effect only after being associated with a distributed virtual switch (DVS). Choose Resource > Resource Zones, select a resource zone, and choose Network Resource Pools > VLAN Pools.

External network
An external network is required only in the following conditions:
  • Create a direct network in a VPC.
  • Apply for a software router in the VPC for creating a routed network. In this case, an external network is required to connect to the software router and provide public IP addresses for VMs in the routed network to connect to the public network.
    NOTE:

    For creating routed networks, the required external network must be able to connect to the Internet, its subnet IP address assignment mode must be set to static injection, and the subnet type must be common.

 Choose Resource > Resource Zones, select a resource zone, and choose Network Resource Pools > External Networks.

Virtual Service Appliance (VSA) management network

The VSA management network is required for applying for a software router.

The VSA management network provides the management plane for system service VMs.

 Choose Resource > Resource Zones, select a resource zone, and choose Network Resource Pools > VSA Networks.

VSA VM template

A VSA VM template is required for applying for a software router.

FusionManager can use a VSA VM template to automatically create system service VMs.

 Choose Resource > Templates and Specs > VM Templates.

Firewall device

A physical firewall device is only required for applying for a hardware router.

To use a hardware router to provide routing function, you need to create a virtual firewall instance on the physical firewall device and add the firewall device to FusionManager.

 Choose Resource > Devices > Network Devices > Firewalls.
