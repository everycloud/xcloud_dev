You can apply for the deployment service for a virtual private cloud (VPC) to implement the following functions:
  • Automatic software deployment
    • Automatic software installation: When you provision application instances using an application template in which the VM template has specific software configured, the system automatically installs the software on the VM that is to run the application instance.
    • Software distribution: After you select a software package and specify the target VMs to install the package, the system automatically installs the software on the specified VMs.
    • Automatic script execution: When you provision application instances using an application template in which the VM template has specific script file configured, the system automatically executes the script on the VM that is to run the application instance.
    • Script distribution: After you select a script file and specify the target VMs to execute the script file, the system automatically distributes the script to the specified VMs and executes the script on them.
  • Hyperic HQ-based VM process monitoring

Applying for the Deployment Service

When you apply for the deployment service, the system uses the Proxy VM (PVM) template to automatically create a system service VM for providing the deployment service. Therefore, before applying for the deployment service, ensure that you have created a Virtual Service Appliance (VSA) management network and created a PVM template to the system.

Choose VPC > My VPC, select a VPC, navigate to the Deployment Service page page, and click Apply for Deployment Service.

Stopping the Deployment Service

If the deployment service is not to be used temporarily, you can stop it.

Choose VPC > My VPC, select a VPC, navigate to the Deployment Service page, and click Stop.

Starting the Deployment Service

You can start a stopped deployment service to resume the automatic software installation and automatic software distribution functions in the system.

Choose VPC > My VPC, select a VPC, navigate to the Deployment Service page, and click Start.

Deleting the Deployment Service

After the deployment service of a VPC is deleted, the automatic software installation and automatic software distribution functions will not apply to the VMs created in the VPC.

Choose VPC > My VPC, select a VPC, navigate to the Deployment Service page, and click Delete.

Adding Networks

You can add new networks to use the deployment service in the VPC, so that the automatic software installation and automatic software distribution functions can apply to VMs associated with these networks.

Choose VPC > My VPC, select a VPC, navigate to the Deployment Service page, and click Add Networks.
