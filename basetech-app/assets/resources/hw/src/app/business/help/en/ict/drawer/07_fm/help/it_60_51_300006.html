Disks

Disks provide data storage space for VMs and are not dependent on VMs. A disk can be attached to different VMs and supports data migration among VMs.

FusionManager also supports disk snapshots for backing up disk data.

Creating a Disk

To expand the storage space for a VM, you can create an independent disk and attach it to the VM.

Before creating a disk, you need to create A virtual private cloud (VPC) must be created before you create a disk. You can also click the VPC name above the disk list to switch a VPC for the disk.

FusionManager supports two disk creation methods:
  • Create an empty disk.

    On the Disk page, click Create.

  • Create a disk using an existing disk snapshot.

    On the Disk Snapshot page, locate the row that contains the target disk snapshot and click Create Disk.

Modifying Disk Attributes

After creating a disk, you can change the disk name if needed.

Locate the row that contains the target disk, click More, and select Modify.

Attaching a Disk to a VM

A disk can store the data of a VM only after being attached to the VM.

Locate the row that contains the target disk, click More, and select Bind VM.

If the disk type is IDE, you can attach the disk only when the VM is in the Stopped state.

After attaching the disk to the VM, you also need to add the disk to the VM operating system (OS).
  • If the attached disk is a newly created disk, format the disk. For the detailed operation procedure, search from the Internet or Forum.
  • If the attached disk already contains data and the disk drive letter is the same as the disk drive letter on the VM, set the disk status from offline to online.

Detaching a Disk from a VM

Detach a disk from its attached VM.

Expand the disk details area, locate the row that contains the target VM, and click Unbind.

Creating a Snapshot

A snapshot is a reproduction of the data and state of a disk created at a specific time point, and can be used to restore the disk to the state when the snapshot was taken.

Locate the row that contains the target disk, click More, and select Create Snapshot.

Created snapshots are displayed on the Disk Snapshot page.

Modifying Disk Snapshot Attributes

After creating a disk snapshot, you can modify the name and description of the snapshot if needed.

On the Disk Snapshot page, locate the row that contains the target snapshot, click More, and select Modify.

Deleting a Disk

If a disk is not needed any more, you can delete it from the system to release storage resources. After a disk is deleted, all the data on the disk will also be deleted. Therefore, exercise caution when performing this operation.

NOTE:
Declaration: This feature is a high-risk feature. Using this feature complies with industry practices. However, end user data may be required for implementing the feature. Exercise caution and obtain end user's consent when using this feature.

Only a disk that is not attached to any disk can be can be deleted. If a disk is in the Available state, it can be deleted simply by clicking Delete. If the disk is in other states, click Forcibly Delete to delete it.

After a disk is deleted, the snapshot taken for the disk will not be deleted. The disk snapshot can be used to restore the disk to the status when the snapshot was taken.

Locate the row that contains the target disk and click Delete.

Deleting a Disk Snapshot

If a disk snapshot is not needed any more, you can delete it from the system to release virtual resources.

If a snapshot is in the Available state, it can be deleted simply by clicking Delete. If the snapshot is in other states, click Forcibly Delete to delete it.

Navigate to the Disk Snapshot page, locate the row that contains the target snapshot, click More, and select Delete.
