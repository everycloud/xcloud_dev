操作场景

使用软件自动部署功能时，虚拟机中必须安装此软件，否则不需要安装。

FusionManager提供软件自动部署功能，该功能帮助用户将应用软件或脚本快速批量的部署到虚拟机中。
  • 软件自动安装：使用应用模板发放应用实例时，若虚拟机模板中带有软件包模块，则可选择使用的软件包，应用软件会自动安装在对应的应用实例虚拟机中。
  • 软件分发：选定应用软件包，并指定分发的目的虚拟机，可自动将该应用软件安装至虚拟机中。
  • 脚本自动执行：使用应用模板发放应用实例时，若虚拟机模板中带有脚本模块，则可选择使用的脚本，脚本在对应的应用实例虚拟机中自动执行。
  • 脚本分发：选定脚本，并指定分发的目的虚拟机，可自动在虚拟机中执行该脚本。

操作步骤

    将需要安装的软件制作为ISO文件

    1. 将功能所需的安装软件，全部放入一个ISO文件中，以便通过挂载光驱的方式，将安装文件拷贝至虚拟机中。

      目前业界已有多款优秀的ISO镜像制作工具，如UltraISO、ISOMaker等，使用这些工具能够轻松将各类文件打包制作成为ISO文件。可通过互联网或技术论坛获取更多信息。

      后续步骤中ISO文件名称以“puppet.iso”为例，其中包含“puppet”文件夹。

    通过挂载光驱的方式将ISO文件中的安装文件拷贝至虚拟机中

    1. 使用VNC登录虚拟机并挂载ISO文件“puppet.iso”。

      挂载光驱的具体操作方法，请参考挂载光驱。

    2. 使用“root”帐户进入虚拟机操作系统。

      若操作系统为图形化界面，则右键单击桌面，在弹出的快捷菜单中选择“在终端中打开”，进入命令行界面。

    3. 执行以下命令，防止系统超时退出。

      TMOUT=0

    1. 执行以下命令，创建“xvdd”目录。

      mkdir xvdd

    2. 执行以下命令，挂载虚拟机光驱。

      mount 挂载路径 xvdd

      例如：mount /dev/sr0 xvdd

      根据Linux操作系统版本不同，挂载路径不同，如表2所示。

    1. 执行以下命令，将ISO中的“puppet”文件夹拷贝到虚拟机的“/home”目录下。

      cp -r xvdd/puppet/ /home

    安装Puppet软件客户端

    1. 使用已经拷贝至“/home”目录下的安装文件，安装ruby、ruby-devel和rubygems软件。
    2. 使用已经拷贝至“/home”目录下的安装文件，安装依赖软件包和Puppet客户端软件。

      具体操作方法请参考Puppet官方网站相关产品文档。

    配置Puppet软件客户端

    1. 使用“root”帐户进入虚拟机操作系统。

      若操作系统为图形化界面，则右键单击桌面，在弹出的快捷菜单中选择“在终端中打开”，进入命令行界面。

    2. 操作系统类型是否为Redhat linux enterprise5.5 64位、CentOS5.6 32位或CentOS5.6 64位操作系统？

      • 是，执行12。
      • 否，执行17。

    3. 执行以下命令，查看回显中是否包含“disabled”或“permissive”？

      sestatus -v

      包含“disabled”时回显如下所示。
      SELinux status:                 disabled
      包含“permissive”时回显如下所示。
      SELinux status:                 enabled
SELinuxfs mount:                /selinux
Current mode:                   permissive
Mode from config file:          enforcing
      • 是，执行17。
      • 否，执行13。

    4. 执行以下命令，使用VI编辑器打开“/etc/selinux/config”文件。

      vi /etc/selinux/config

    5. 按“i”进入编辑模式，并将“SELINUX”设置为“disabled”。
    6. 输入:wq，保存修改的内容并退出。
    7. 执行以下命令，重启操作系统。

      reboot

    8. 重启后，重新挂载ISO文件“puppetclient_linux.iso”至“xvdd”目录下。
    9. 执行以下命令，进入配置文件目录。

      cd xvdd/puppetclient_linux

      说明：

      若当前操作系统为 Red Hat Linux Enterprise，请检查该目录下的文件或文件夹名称是否全是小写，若全是小写，则需要重启后重新挂载光驱，再进行配置。

    10. 进入当前操作系统类型的目录。
    11. 执行以下命令，执行配置脚本。

      sh configpuppet.sh

      当界面提示如下回显，表示软件配置成功。
      GalaxManager Agent has been installed successfully!
      说明：

      配置成功后，系统中会自动生成“puppet”帐户，该用户密码为空，且不能用于系统登录，不存在安全风险。
