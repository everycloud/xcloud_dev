Availability Zones

An availability zone is a logical zone of physical resources (computing, storage, and network) available to users. The hardware devices in an availability zone are connected at layer 2. Commonly, resource clusters in a resource zone are grouped in the same availability zone based on the following rules:
  • All resource clusters in an availability zone come from the same hypervisor.
  • All resource clusters in an availability zone use the same storage resource.
  • All resource clusters in an availability zone use the same distributed virtual switches (DVSs).

Managing Resource Clusters

After an availability zone is created, you can manage the resource clusters contained in the availability zone.

Managing Tags

Tags are used to facilitate identification and selection of virtual resources, such as resources clusters and availability zone. After creating an availability zone, you can adjust the tag assigned to it.

Deleting an Availability Zone

If an availability zone is not needed any more, you can delete it from the system. However, an availability zone has been associated with resource clusters, the zone is not allowed to be deleted.
