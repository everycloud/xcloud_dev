VM-Snapshot

A snapshot is a reproduction of the data and state of a VM created at a specific time point, and can be used to restore a VM to the state when the snapshot was taken.

When you take a snapshot for a VM, the current disk on the VM is set to be read-only. The system automatically creates an incremental disk in the data store where the disk is located, and the subsequent edit operations on the data of the disk are saved on the incremental disk, that is, the incremental disk records the differences between the current state of the disk and the last snapshot of the disk. When you create another snapshot of the disk, the disk and its current incremental disk are both set to read-only, and the system creates another incremental disk in the data store.

Creating a Snapshot

Snapshots can be created before major changes of the VM happen. They can be used to restore the VM data if faults occurs because the change fails.

For details, see VM Snapshot Usage Requirements.

Restoring a VM using a Snapshot

Use the created snapshot to restore the VM data to the state when the snapshot is created if a VM is faulty or requires data restoration. The current data on the VM will be overwritten by the data in the snapshot. Therefore, exercise caution when performing this operation.

If a new disk is attached to a VM after a snapshot is taken for the VM, the disk will be automatically detached from the VM after the VM is restored using this snapshot. If necessary, you can attach this disk to the VM again.

If a disk on a VM is set to Not taken into snapshots, data on the disk will not be restored after the VM is restored using a snapshot.
  • If a snapshot is taken for a VM, and disks on the VM are detached from the VM and not attached to any other VM, the disks will be attached to the VM after the VM is restored using the snapshot. However, data on the disks will not be restored.
  • If a snapshot is taken for a VM and disks on the VM are deleted, the disks will not appear on the VM after the VM is restored using the snapshot.

You can restore a VM using a snapshot only when the VM is in the Stopped or Hibernated state.

Deleting a Snapshot

For each VM, a maximum of 32 snapshots can be created. Snapshots take space of the data store where the disk is located. To prevent insufficient storage resources, control the number of snapshots and promptly delete snapshots no longer used.
