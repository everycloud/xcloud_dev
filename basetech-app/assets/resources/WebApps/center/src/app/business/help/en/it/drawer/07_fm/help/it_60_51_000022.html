Virtual Resource Management

Overview

Virtual resources include hypervisors, resource clusters, virtual hosts, storage, and VM resources.

Adding virtual resources to FusionManager is the first step of resource management. Virtual resources on a hypervisor can be used by FusionManager for VM or service provisioning only after they are added.

Add a hypervisor.

 Adding virtual resources to FusionManager is the first step of resource management. Virtual resources on a hypervisor can be used by FusionManager for VM or service provisioning only after they are added.

Choose Resource > Virtualization > Hypervisor

Update the hypervisor.

Hypervisors are updated automatically every six hours. To make the hypervisor that has been added to FusionManager take effect immediately, update the hypervisor.

Choose Resource > Virtualization > Hypervisor, click More, and select Update.

Associate a resource cluster with FusionManager.

The hypervisor provides virtualization capabilities. Only resources in clusters can be used and managed by FusionManager. Therefore, after a hypervisor is added and updated, you can associate clusters created on the hypervisor with FusionManager.

Choose Resource > Virtualization > Resource Clusters.
