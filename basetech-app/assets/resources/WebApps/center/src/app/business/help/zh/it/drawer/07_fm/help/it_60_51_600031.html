修改共享VPC的默认网络配置

操作场景

共享VPC是系统管理员创建的VPC，为系统内所有VDC提供网络资源。该任务用于指导管理员修改共享VPC的以下配置：

  • 共享VPC中的网络不能被一个租户独占使用。但在共享VPC中创建网络之前，管理员可以通过修改默认配置，使租户能够独占使用共享VPC中的网络。
  • 租户独占共享VPC中的网络，且使用L2TP VPN时，系统会通过域名（域名前缀+序列号）区分不同独占网络的VPN用户。管理员可以修改默认的域名前缀。

前提条件

  • FusionManager节点主备部署时，需获取浮动IP地址；FusionManager单节点部署时，获取管理IP地址。
  • 已获取FusionManager节点的“galaxmanager”用户密码。

操作步骤

    1. 使用“PuTTY”，以“galaxmanager”用户登录FusionManager节点。

      • FusionManager节点主备部署时，通过浮动IP地址登录主节点。
      • FusionManager单节点部署时，以管理IP地址登录。

    2. 执行以下命令，防止“PuTTY”超时退出。

      TMOUT=0

    3. 执行以下命令，修改共享VPC的网络配置。

      confNetInfo 参数选项

      参数选项说明如表1所示。

      表1 选项说明

      选项

      说明

      Share_Network false

      租户能够独占使用共享VPC中的网络时，选择此项选项。

      Share_Network true

      租户不能独占使用共享VPC中的网络时，选择此项选项。系统默认为此配置。

      L2TP_DomainPrefix fmVpnDomain
      • 修改共享VPC下的L2TP VPN的域名前缀，默认值为“fmVpnDomain”。
      • 域名前缀规则：长度1～20，支持字母、数字以及“_”。

      例如，配置租户独占网络时，执行命令confNetInfo.sh Share_Network false；修改L2TP VPN的域名前缀时，执行命令confNetInfo.sh L2TP_DomainPrefix VpnDomainName

      回显如下信息，表示配置成功。

      configure success

    4. FusionManager节点主备部署时，执行以下命令，同步主备配置。

      haNotifySyncFiles /opt/goku/services/irm/rpool/config/network.properties

      回显如下信息，表示配置成功。

      execute command syncfile successfully.
