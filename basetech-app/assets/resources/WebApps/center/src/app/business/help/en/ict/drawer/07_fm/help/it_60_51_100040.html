Host Groups

A host group is a collection of hosts managed by FusionManager.Hosts are the physical computers in an OpenStack instance running virtualization software and providing computing resources, such as CPUs and memory, for VMs.

After creating VMs or application instances, the VMs or application instances run on the hosts in a host group included in the selected availability zone.

After an OpenStack instance is successfully associated with FusionManager, the host groups contained in the OpenStack instance are displayed on this page.

Configuring Tags

You can configure tags for a host group based on the attributes and characteristics of hosts in the host group. The tags configured for a host group can apply to a set of VM specifications. If this set of VM specifications is used to create VMs, the system only selects the hosts that fully meet the specified tags in the host group to create VMs.

For example, if host group 1 has an tag attribute configured, with its name set to computing_performance and value set to High, and this computing_performance tag is configured for the VM specifications set spec01, the system selects a qualified host from host group 1 when using spec01 to create VMs.

Locate the row that contains the target host group and click Manage Tag.

Viewing Hosts in a Host Group

You can view the hosts included in a host group.

Locate the row that contains the target host group and click the host group name to check the detailed host group information.
