System Configuration

You can configure system data, such as system license and system time. Typically, you only need to configure system data only once.

System Timeout

If you do not perform any operation within the specified time, you will be logged out automatically. A shorter system timeout duration provides higher security, but a longer timeout duration relieves the need to log in multiple times. You can set the system timeout duration based on your requirements.

After the system timeout duration is configured, the configuration does not take effect for the current user and takes effect the next time the user logs in to the system.

SNMP Station

To enable a third-party system to manage FusionManager alarms and monitoring data, you can configure an SNMP station so that the alarms and monitoring data are reported to the third-party system.

NOTE:
  • Do not set the interconnection parameters as follows because risks may arise:
    • SNMP Version is SNMPv2c.
    • Authentication Mode is MD5.
    • Key Type is DES56.
  • Set the interconnection parameters as follows, instead:
    • SNMP Version is SNMPv3.
    • Authentication Mode is SHA1.
    • Key Type is AES128, AES192, or AES256.

Backup Settings

Management system backup: Before performing an important operation, such as system upgrade or critical data modification, you can manually back up data of the FusionManager node. The data backup can be used to restore the FusionManager node if an exception occurs or the operation has not achieved the expected result.

You can back up FusionManager management data either using the FusionManager Web Client or using the command-line interface (CLI). To back up management data on the FusionManager Web Client, configure the FTP server.

For details about management system backup, see Routine Maintenance > Backup and Restoration in the FusionManager V100R005C00 Administrator Guide (Cloud Data Center).

System Logo

You can customize the system logo displayed in the address bar of the browser or the FusionManager Web Client.
NOTE:
If Internet Explorer is used, ensure that the system logo used in the address bar is in its original ico format. Pictures that are originally not created in ico format cannot be used as system logos after being changed to ico format.

License Management

After FusionManager is installed or the system license has expired, you can update the license to renew the validity period and authorize FusionManager services.

Time Management

After installing the system, configure the time synchronization policy and time zone on FusionManager, enabling FusionManager to use the local time and synchronize time with an external time server.

If FusionStorage is used, the system automatically configures the time synchronization policy and time zone for the FusionStorage Manager nodes of FusionStorage when you configure the FusionManager time synchronization policy and time zone.

You are advised to preferentially configure an external Network Time Protocol (NTP) clock source. If no external NTP clock source is available, do not configure the NTP server on the FusionManager web client.

If the time difference between FusionManager and the NTP server exceeds one minute, automatic time synchronization fails, and the system generates an alarm. In this case, the administrator needs to forcibly synchronize time with the NTP server based on the alarm help guide. Forcible time synchronization may interrupt services. Therefore, exercise caution when enabling forcible time synchronization.

The verification rules for daylight saving time (DST) are as follows (m indicates the selected month, and HH:MM indicates the selected hour and minute):
  • By date:

    Start time: If the selected date is the last day in m and HH:MM plus the offset is greater than or equal to 24 hours, set the start date to the first day in the next month of m. If either of the conditions is not met, retain m.

    End time: If the selected date is the first day in m and the offset subtracted from HH:MM is less than or equal to 1 minute, set the end date to the last day in the last month of m. If either of the conditions is not met, retain m.

    For example, if the start time is set to 02:00 on August 8, the end time is set to 00:00 on September 1, and the offset is set to 60 minutes, the settings are invalid, because 60 minutes subtracted from HH:MM is less than 1 minute. The real end time is 23:00 on August 31. However, the start date and the end date cannot be set in the same month. Therefore, set the end time to 01:01 on September 1 or later.

  • By week:

    Start time: If the selected week is the last week in m and HH:MM plus the offset is greater than or equal to 24 hours, set the start week to the first week in the next month of m. If either of the conditions is not met, retain m.

    End time: If the selected week is the first week in m and the offset subtracted from HH:MM is less than or equal to 1 minute, set the end week to the last week in the last month of m. If either of the conditions is not met, retain m.

    For example, if the start time is set to 23:00 in the last Sunday in March, the end time is set to 01:00 in the first Sunday in April, and the offset is set to 60 minutes, the settings are invalid, because 60 minutes plus HH:MM is equal to 24 hours. The real end time may be in April. However, the start date and the end date cannot be set in the same month. Therefore, set the start time to 22:59 on last Sunday in March or earlier.

  • By date and week:

    Start time: If the time span between the specified date and the last day in m is less than 8 days, set m to the next month of m. If not, retain m.

    End time: If the selected date is the first day in m and the offset subtracted from HH:MM is less than or equal to 1 minute, set the end week to the last week in the last month of m. If either of the conditions is not met, retain m.

    For example, if the start time is set to 08:00 on May 26, Sunday, and the end time is set to 00:00 on June 8, Wednesday, the settings are invalid, because the time span between May 26 and the last day in May is less than 8 days. The real start time may be in June. However, the start date and the end date cannot be set in the same month. Therefore, set the start date to May 22 or earlier.

Email Server

You need to configure the email server settings only when the alarm notification by email function is required. This function enables the system to send system alarms to your specified email box. The email server is the server of the email box that is to send the selected alarms.
