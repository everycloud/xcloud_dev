Subracks

Servers That Can Be Managed by FusionManager

FusionManager can monitor and manage servers added to it.

Servers that can be managed by FusionManager can be classified into the following types in terms of their appearance:
  • Rack servers: also referred to as rack-mounted servers, are standard size servers used for mounting inside a data center server cabinet.
  • Blade servers: stripped down servers with a modular design to minimize the use of physical space and energy. A blade server commonly consists of a blade enclosure, which is also called a subrack if being housed in a server cabinet, and server blades.

Adding a Subrack to FusionManager

A subrack housing blade servers must be added to FusionManager before the blade servers are added to it.

  1. Choose Resource > Devices > Computing Devices > Subracks and click Add Subrack.
  2. Add the subrack is added, set the blade server adding mode:
    • To add a host controlled in a hypervisor to FusionManager, leave Bare Metal Access unchecked.
    • To add a bare metal server to FusionManager, select Bare Metal Access.
After a blade server is added to FusionManager:
  • The blade server subrack information is displayed on the Subracks page.
  • The blade information is displayed on the Servers page.

If the required server model is not available for selection, include more compatible hardware devices into the system. For details, see Adaptation Package Management.

Updating Subrack Information

Configuring Blade Server Adding Parameters

If the management information or Baseboard Management Controller (BMC) information of a server is changed in the following conditions, you need to synchronize the new information to FusionManager in a timely manner. Otherwise, FusionManager will lose management for the server.
  • The blade server subrack has been added to FusionManager but blade servers have not been added.
  • Both the blade servers and their subrack have been added to FusionManager.

Locate the row that contains the target subrack, click More, and select Set Blade Adding Parameters.

Changing the Subrack Management Account Password

If the management account password of a blade server subrack has been modified after the subrack is added to FusionManager, you need to synchronize the new password to FusionManager in a timely manner. Otherwise, FusionManager will lose management for the blade servers in the subrack.

Locate the row that contains the target subrack, click More, and select Change SMM Password.

Deleting a Subrack

If the blade servers in a subrack are not needed any more, you can delete the whole subrack to delete the blade server from the system. Blade servers can only be deleted by deleting their whole subrack. After a subrack is deleted, FusionManager will not manage or monitor the subrack and all the blade servers in it.
