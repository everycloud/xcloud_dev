System Reports

  • Configuring a Report Policy
  • Downloading a System Report
  • Enabling or Disabling a Report

System reports provide statistics on physical servers, hosts, VMs, clusters, networks, and storage resources. These statistical objects are default in the system. You can change the enabling status of a system report and modify its configuration, but you are not allowed to add or delete a system report. In comparison, custom reports can be added or deleted.

You can also check resource usage and running information by downloading historical or real-time reports.

Configuring a Report Policy

A report policy includes the generation period and maximum number of copies to save.

To configure a report policy, choose Monitoring > System Reports, locate the row that contains the report, change the status in the Status column, and click Modify Settings.

Downloading a System Report

You can download historical or real-time reports.

To download a historical or real-time report, choose Monitoring > System Reports and click Download Real-Time Report or Download Historical Report in the Operation column.

Enabling or Disabling a Report

After a report is enabled, the system generates the report based on the report policy. After a report is disabled, the system does not generate the report. You can enable or disable either a single report or all reports.

To enable or disable all reports, choose Monitoring > System Reports and click Enable all or Disable all. Alternatively, to enable or disable a single report, you can locate the row that contains the report and change the status in the Status column.
