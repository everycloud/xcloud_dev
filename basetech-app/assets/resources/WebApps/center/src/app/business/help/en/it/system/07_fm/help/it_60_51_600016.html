Configuring VLAN Aggregation on the Switch

  • Scenarios
  • Prerequisites
  • Procedure
  • Example

Scenarios

This topic describes how to configure virtual local area network (VLAN) aggregation on the switch. VLAN aggregation allows a subnet to support multiple VLANs, so that users in different departments can use IP addresses in the same subnet but these IP addresses can be grouped in different VLANs to isolate user services. This function improves the service security and also saves IP address resources.

In FusionManager, such a subnet is referred to as a super subnet, which allows multiple virtual private clouds (VPCs) to use the same subnet but have their IP addresses isolated from each other.

Prerequisites

You have logged in to the switch using a terminal emulation tool.

Procedure

  1. Log in to the aggregation switch.
  2. In the system view interface, run the vlan VLAN ID command to create the planned VLAN and enter the VLAN view.
  3. Run the aggregate-vlan command to set the VLAN to an aggregate VLAN.
  4. Run the access-vlan VLAN ID1 VLAN ID2 VLAN ID3 command to add the planned access VLANs to the aggregate VLAN.
  5. Run the interface vlanif Aggregate VLAN ID command to enter the VLANIF interface of the aggregate VLAN.
  6. Run the ip address VLANIF IP address Subnet mask command to configure the IP address and mask for the subnet.
  7. Run the quit command to exit the VLANIF interface.
  8. Run the following commands to add the access VLANs to the ports on the aggregation switch:

    interface GigabitEthernetx/x/xx

    port hybrid tagged vlan vlan id1 to vlan id3

Example

For example, if the planned VLAN ID of the aggregate VLAN is 20, its subnet mask is 255.255.255.0, its gateway address is 192.160.18.1, and the VLAN IDs of the access VLANs are 21, 22, 23, run the following commands on the switch to configure:

<Quidway> system-view

[Quidway] vlan 20

[Quidway-Vlan20] aggregate-vlan

[Quidway-Vlan20] access-vlan 21 22 23 24

[Quidway] interface vlanif 20

[Quidway-Vlanif20] ip address 192.160.18.1 24

[Quidway-Vlanif20] quit

[Quidway] interface GigabitEthernet3/0/37

[Quidway] port hybrid tagged vlan 21 to 24
