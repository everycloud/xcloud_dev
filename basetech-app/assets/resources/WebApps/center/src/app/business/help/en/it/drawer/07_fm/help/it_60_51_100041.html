Cloud Resource Pools

A cloud resource pool contains all resources managed in a FusionSphere system.

FusionManager is able to manage multiple cloud resource pools.

If FusionManager is deployed in all-in-one mode, you do not need to manually connect cloud resource pools to the system, because the system in this deployment mode supports automatic cloud resource pool connection. You can modify the cloud resource pool information, including the name and region, based on actual conditions. However, if you intend to use an all-in-one FusionManager to manage other Local FusionManager systems that are deployed in Top-Local mode, you need to manually connect these Local FusionManager systems to the all-in-one FusionManager.

Adding FusionSphere to FusionManager

After a cloud resource pool is added to FusionManager, FusionManager can use and manage all the resources in the pool in a unified manner. A cloud resource pool can be connected to only one FusionManager system. Otherwise, system exceptions may occur.

The username used for connecting a cloud resource pool to FusionManager is FMRest, and its default password is Huawei@CLOUD8!.

Modifying a Cloud Resource Pool

You can modify the attributes of a cloud resource pool.

Locate the row that contains the cloud resource pool, click More, and select Modify.

Suspending or Resuming a Cloud Resource Pool

If the resources in a cloud resource pool are insufficient or you need to perform maintenance for the pool, for example, scale up the pool, suspend the cloud resource pool. In this case, however, using existing resources in the pool, such as VMs, is not affected.

Locate the row that contains the cloud resource pool, click More, and select the required operation.

Deleting a Cloud Resource Pool

If a cloud resource pool is not needed any more, for example, the pool is to be deleted for system capacity reduction, delete all resources in the resource pool and delete the cloud resource pool.

Locate the row that contains the cloud resource pool, and click Delete.

Adding an Availability Zone

After a cloud resource pool is connected to the system, if the Local FusionManager administrator has created an availability zone in the cloud resource pool, the Top FusionManager administrator needs to manually add this availability zone to the system to make the availability zone take effect.

Select a cloud resource pool, navigate to the Availability Zones page, and click Add.

Managing the Tag Assigned to an Availability Zone

Tags are used to facilitate identification and selection of virtual resources, such as resources clusters and availability zone. After creating an availability zone, you can adjust the tag assigned to it.

Select a cloud resource pool, navigate to the Availability Zones page, locate the row that contains the availability zone, and click Tags.

Managing the VDC of an Availability Zone

Manage the VDC of an AZ.

Select a cloud resource pool, navigate to the Availability Zones page, locate the row that contains the availability zone, click More, and select VDCs.

Suspending or Resuming a Availability Zone

If the resources in a availability zone are insufficient or you need to perform maintenance for the availability zone, suspend the availability zone. In this case, however, using existing resources in the availability zone, such as VMs, is not affected.

Select a cloud resource pool, navigate to the Availability Zones page, locate the row that contains the availability zone, click More, and select the required operation.

Deleting an Availability Zone

If an availability zone is not needed any more, you can delete it from the system.

Select a cloud resource pool, navigate to the Availability Zones page, locate the row that contains the availability zone, click More, and select Delete.
