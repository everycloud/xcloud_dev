VM specifications define the VM hardware settings, including the number of CPUs, memory space, and system disk space.

You can define VM specifications for provisioning VMs with the defined specifications. VMs can have different specifications specified to meet different performance requirements.

Choose Resource > OpenStack Instances, select an OpenStack instance, navigate to the VM Specifications page and click Create.

Setting the Boot Device

The default boot device for defining VM specifications is the image. After the VM specifications are defined, you can change the boot device. The following boot devices are available for VM specifications:
  • Image: In this mode, the VM system disks are created using the local disk file system of the host. Cold migrating VMs or modifying VM specifications requires configuration of the host mutual trust relationships. However, configuring the host mutual trust relationships is not recommended because it may cause risks. If you need to configure the mutual trust relationships, visit OpenStack official website and see related configuration information.
  • Cloud Hard Disk: In this mode, the VM system disks are created using the cinder-volume storage resource, which can be provided by a local host disk or a shared storage device, such as IP SAN or FusionStorage. The VM system disks can be managed by FusionManager. If a shared storage device provides storage space for the system disks, configuration of the host mutual trust relationships is not required.
The boot device setting configured in VM specifications takes effect only when VMs are being created. When you use an application template to create an application instance, the VMs included in the application instance start using the image file by default.

Choose Resource > OpenStack Instances, select an OpenStack instance, navigate to the VM Specifications page, locate the row that contains the target VM specifications, click More, and select Set Boot Device.

Configuring Tags

The tags configured for a host group can apply to a set of VM specifications. If this set of VM specifications is used to create VMs, the system only selects the hosts that fully meet the specified tags in the host group to create VMs.

For example, if host group 1 has an tag attribute configured, with its name set to computing_performance and value set to High, and this computing_performance tag is configured for the VM specifications set spec01, the system selects a qualified host from host group 1 when using spec01 to create VMs.

Choose Resource > OpenStack Instances, select an OpenStack instance, navigate to the VM Specifications page, locate the row that contains the target host group, click More, and select Manage Tag.

Deleting VM Specifications

If a set of VM specifications is not needed any more, you can delete it from the system.

Locate the row that contains the target VM specification set and click Delete.
