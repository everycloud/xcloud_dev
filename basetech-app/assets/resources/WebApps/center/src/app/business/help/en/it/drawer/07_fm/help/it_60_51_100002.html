Servers

Servers That Can Be Managed by FusionManager

FusionManager can monitor and manage servers added to it.

Servers that can be managed by FusionManager can be classified into the following types in terms of their appearance:
  • Rack servers: also referred to as rack-mounted servers, are standard size servers used for mounting inside a data center server cabinet.
  • Blade servers: stripped down servers with a modular design to minimize the use of physical space and energy. A blade server commonly consists of a blade enclosure, which is also called a subrack if being housed in a server cabinet, and server blades.

Adding a Rack Server to FusionManager

Choose Resource > Devices > Computing Devices > Servers and click Add Rack Server.After a server is added to FusionManager, the server information is displayed on the Servers page.

If the required server model is not available for selection, include more compatible hardware devices into the system. For details, see Adaptation Package Management.

The configurations of some models of physical devices must be modified after the devices are added to FusionManager, so that the device monitoring and alarm information can be reported to FusionManager. For types and configuration requirements, see Device Management Overview.

Server Power Management

Power management for a server is dependent on the Baseboard Management Controller (BMC) port of the server. Therefore, the server BMC information set in the FusionManager system must be consistent with the actual server BMC information.

Power management for a server includes power on, power off, and restart the server. Power off and restart a server can be performed in a safe or forcible manner. You are advised to use the safe manner. Forcibly power off or restart a server only if the server cannot be safely powered off or restarted due to an OS fault. Forcibly powering off or restarting a server may damage the server disks and cause data loss.

Locate the row that contains the target server and click More to select the desired operation.

Modifying Server Connection Parameters

If the management information or BMC information has been modified after a server is added to FusionManager, you need to synchronize the new information to FusionManager in a timely manner. Otherwise, FusionManager will lose management for the server.

Deleting a Server

If a server is not needed any more, you can delete it from the system. After a server is deleted, FusionManager will not manage or monitor it. A rack server can be deleted by deleting the single server. A blade server can only be deleted by deleting the whole subrack containing the blade. After a host is deleted, VMs or applications carried on the host can still work properly.

Locate the row that contains the target server, click More, and select Delete.
