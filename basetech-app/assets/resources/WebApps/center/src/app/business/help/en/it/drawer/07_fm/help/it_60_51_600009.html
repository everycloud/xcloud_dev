Configuring a CommVault VM Backup Server

Scenario

Configure a CommVault VM backup server to back up VM disk data. If a VM is faulty, the data backups on the backup server can be used to restore the VM.

FusionManager supports Huawei HyperDP backup system and the CommVault Simpana backup system. Simpana is the data management software from the CommVault Systems company, and HyperDP is a data protection solution provided by Huawei for the FusionCompute hypervisor. These backup servers can conquer the challenges of large-scale data backups and data redundancy to improve VM data security.

Prerequisites

  • You have obtained the backup server name, username, and password.
  • You have logged in to FusionManager.
  • You have configured the storage policies on the CommVault server.
  • You have obtained the private key file.

Procedure

    Install the iDataAgent client.

    1. Use PuTTY to log in to the active FusionManager node as user galaxmanager.

      The default password for user galaxmanager is Huawei@CLOUD8.

    2. Run the following command to disable logout on timeout:

      TMOUT=0

    3. Run the following command to configure the CommVault operating environment:

      modConfig cv -b

      The configuration is successful if information similar to the following is displayed:

      configure cv before install success

    4. Install the OpenSSL tool on the local PC. For details, see Creating and Replacing a FusionManager Security Protection Certificate.
    5. Obtain the CommVault ISO software package and private key and store them in the directory where the OpenSSL tool is installed.

      For example, the OpenSSL installation directory is D:\OpenSSL-Win32\bin.

    6. Run the following command to generate a digital signature for the CommVault ISO software package:

      openssl dgst -sha256 -out File name after signature -sign File name of the private key File name of the CommVault software package

      The Software package file name.sig format is recommended for the file name after signature.

      If the software package file name is CommVault.zip and the file name of the generated private key is private.key, run the following command to generate a digital signature:

      openssl dgst -sha256 -out CommVault.zip.sig -sign private.key CommVault.zip

    7. Use WinSCP to log in to the active FusionManager node as user galaxmanager and copy the CommVault ISO software package and signature file to the /opt/goku/data/upload directory of the active FusionManager node.

      The default password for user galaxmanager is Huawei@CLOUD8.

    8. Run the following command to start installing the CommVault server:

      installComponents cv Simpana_10_DVD03_UNIX_03042013.iso

    9. Install the CommVault client, iDataAgent, on the active and standby FusionManager nodes, respectively. For details, see the CommVault product documents.

      You can visit http://www.commvault.com.cn/services-and-support and download the product documents for the corresponding CommVault version.

      NOTE:

      Select the file system components to be installed when installing the iDataAgent.

    10. Run the following command to configure the CommVault server:

      modConfig cv -a -i IP address of the CommVault server -s"Storage policy name"

      Note the following when executing this command:

      • Enter the name of the storage policy that has been configured on the CommVault server.
      • If multiple storage policies are available, use vertical bars (|) to separate them.

      For example, to configure the CommVault server, run the modConfig cv -a -i 192.168.0.123 -s "storage01|storage02" command.

    11. Install the CommVault server on the standby FusionManager node. For details, see 1 to 10.
    12. Configure the CommVault server to regularly collect storage usage information and upload the report to a specific directory on FusionManager.

      The reports are divided into groups according to the customer client group, the report names are ended with summary.csv, and the reports are stored in the report storage directory configured on the iDataAgent client.

    Configure a CommVault backup server.

    1. On FusionManager, choose System > Backup Settings.
    2. Select Configure CommVault Backup Server in the Virtual Backup Server area.
    3. Select Use CommVault backup server.
    4. Set Server name to the name of the CommVault backup server.
    5. Set Username, Password, and Confirm Password for the CommVault backup server.
    6. Click Save to save the configuration.
