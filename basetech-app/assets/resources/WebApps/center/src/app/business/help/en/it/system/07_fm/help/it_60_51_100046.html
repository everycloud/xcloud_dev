Adjust policies and configurations on demand after a VM is created.

Modifying OS Attributes

Modify operating system (OS) attributes when the VM OS is changed because of a VM failure or reinstallation to keep the OS attributes information displayed on the system to be consistent with the actual information.

You can modify the OS attributes only when the VM is in the Stopped state.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, select a VM, navigate to the Options page and click OS.

Configuring the HA Policy for a VM

Virtual machine high availability (HA) is enabled for the system to monitor all physical hosts. If a host fails, all virtual machines based on the host are transferred to other functional hosts.

You can configure the HA policy only when the VM is in the Running or Stopped state.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, select a VM, navigate to the Options page and click HA.

Configuring the VM Boot Device for a VM

The VM boot device includes the network, DVD/CD-ROM drive, and hard disk. If the boot device is configured to Network or CD-ROM, the second default boot device is Hard disk.

You can configure the VM boot device only when the VM is in the Running or Stopped state. You are advised to change the VM boot device on FusionManager. If you change the VM boot device in the hypervisor, you need to synchronize the change on FusionManager to make the change take effect.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, select a VM, navigate to the Options page and click Boot Device.

Configuring the Time Synchronization Policy for a VM

Configure the time synchronization policy for a VM to determine whether the VM regularly synchronizes time from the host.

You can configure the time synchronization policy for a VM only when the VM is in the Stopped state. For VMs in the VMware hypervisor, if VMware Tools has been installed on the VM, and the tool is running, the VMs automatically synchronize time from the host after they restart even if Sync time with host is not selected.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, select a VM, navigate to the Options page and click Clock Sync.

Configuring Hot Swap Policies for a VM

Configure hot swap policies for a VM.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, select a VM, navigate to the Options page and click Hot Swap.

For details, see Setting Hot Swap Policies for a VM.

Configuring the VM Upgrade Mode

Configure the upgrade mode for the Tools on a VM in the FusionCompute hypervisor. Fore more details about Tools, see Tools Introduction.

You can configure the VM upgrade mode only when the VM is in the Running or Stopped state. The system automatically upgrades the VM only when Tools is running properly on the VM.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, select a VM, navigate to the Options page and click Upgrade Mode.

Configuring the Basic Block Storage Advanced Functions for a VM

Basic block storage includes non-virtualized SAN storage, non-virtualized local disks, and data stores using FusionStorage.

Set whether to enable the following functions:
  • Enable virtual disks using the basic block storage to be live migrated to virtualized storage.
  • Enable virtual disks using FusionStorage type to support host-based replication DR.

You can configure the basic block storage advanced functions for a VM only when the VM is in the Stopped state. VMs in the VMware hypervisor do not support this function.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, select a VM, navigate to the Options page and click Basic Block Storage Advanced Functions.

Configuring the Antivirus Function for a VM

Enable or disable the antivirus function for a VM. After the antivirus function is enabled, the VM can be set to either of the following types:
  • Secure service VM: A service VM that provide secure user VMs with antivirus services, such as virus scanning, removal, and real-time monitoring functions. Its VM template is provided by the antivirus vendor.
  • Secure user VM: An end user VM that uses the antivirus function provided by the secure service VM.

The antivirus function can be configured only for a Stopped VM. The function takes effect only after the host has the antivirus function enabled. For details about the VM antivirus function and its deployment plans, see Installing the Antivirus Software Client Driver.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, select a VM, navigate to the Options page, and click VM Anti-Virus Protection.
