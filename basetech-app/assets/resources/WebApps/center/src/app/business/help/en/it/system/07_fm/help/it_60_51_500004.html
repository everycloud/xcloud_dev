FusionManager provides performance monitoring indicators of the clusters, hosts, and VMs in the hypervisor connected to it. The following uses FusionCompute as an example to describe its performance indicators.

To view performance indicators, choose Monitoring > Monitoring View.

CPU Usage

Definition

Measures the CPU usage of a measurement object.

The measurement objects can be a clusters, host, or VM.

Formula

  • Cluster CPU usage = Total CPU usage of all hosts in a cluster/Number of hosts
  • Host CPU usage = (Dom0 CPU usage of a host x Number of Dom0 cores + DomU CPU usage of the host x Number of DomU cores)/(Number of Dom0 cores + Number of DomU cores)
  • VM CPU usage = CPU usage of a VM/Number of VM CPU cores

Description

None

CPU Reservation

Definition

Measures the percentage of the number of reserved CPUs in the total number of CPUs of a measurement object.

The measurement object can be a cluster.

Formula

CPU reservation rate = Reserved CPU capacity/(Reserved CPU capacity + Available CPU capacity) x 100%

where
  • Reserved CPU capacity: specifies the total number of CPUs reserved for the running VMs of the measurement object.
  • Available CPU capacity: specifies the total number of CPUs that can be allocated to VMs of the measurement object.
NOTE:

In the VMware hypervisor, the reserved CPU capacity does not include the capacity reserved for the VMware system.

Description

The value of this performance indicator ranges from 0% to 100%.
  • If the value is 100%, no CPU can be allocated.
  • If the value is 0%, no CPU is reserved for VMs. That is, all the CPUs are available to the measurement object.

Memory Usage

Definition

Measures the memory usage of a measurement object.

The measurement objects can be a cluster, host, or VM.

Formula

  • Cluster memory usage = Total memory usage of all hosts in a cluster/Number of hosts
  • Host memory usage = (DomU memory usage of a host x Dom0 memory usage of a host)/(Total memory of DomU + Total memory of Dom0)
  • VM memory usage = Memory usage of a VM/Total memory of the VM

Description

None

Memory Reservation

Definition

Measures the percentage of the reserved memory capacity in the total memory capacity of a measurement object.

The measurement object can be a cluster.

Formula

Memory reservation rate = Reserved memory capacity/(Reserved memory capacity + Available memory capacity) x 100%

where
  • Reserved memory capacity:
    • Specifies the sum of memory and GPU for the running VMs of the measurement object when memory overcommitment is disabled.
    • Specifies the sum of reserved memory and GPU for the running VMs of the measurement object when memory overcommitment is enabled.
  • Available memory capacity: specifies the total amount of memory that can be allocated to VMs of the measurement object.
NOTE:

In the VMware hypervisor, the reserved memory capacity does not include the capacity reserved for the VMware system.

Description

The value of this performance indicator ranges from 0% to 100%.
  • If the value is 100%, no memory can be allocated.
  • If the value is 0%, no memory is reserved for VMs. That is, all the memory is available to the measurement object.

Storage Usage

Definition

Measures the percentage of the used storage capacity in the total storage capacity of a measurement object.

The measurement object can be a cluster.

Formula

Storage usage = Used storage capacity/(Used storage capacity + Remaining storage capacity) x 100%

where
  • Used storage capacity: specifies the total storage capacity that has been used by a measurement object.
  • Remaining storage capacity: specifies the total storage capacity that has not been used by a measurement object.

Description

The value of this performance indicator ranges from 0% to 100%.
  • If the value is 100%, no storage space can be allocated.
  • If the value is 0%, no storage space has been used.

Storage Allocation

Definition

Measures the percentage of the allocated storage capacity in the total storage capacity of a measurement object.

The measurement object can be a cluster.

Formula

Storage allocation rate = Allocated storage capacity/(Used storage capacity + Remaining storage capacity) x 100%

where
  • Allocated storage capacity: specifies the total storage capacity that has been allocated to a measurement object.
  • Used storage capacity: specifies the total storage capacity that has been used by a measurement object.
  • Remaining storage capacity: specifies the total physical storage capacity that has not been used by a measurement object.
NOTE:
For example, the real physical storage space of a storage pool is 100 GB. After thin provisioning is enabled, the allocatable storage space is 200 GB. If 50 GB of the storage space is allocated to VM A that uses 20 GB to store its files, and 80 GB of the storage space is allocated to VM B that uses 10 GB to store its database, then:
  • Allocated storage capacity: 130 GB
  • Used storage capacity: 30 GB
  • Remaining storage capacity: 70 GB
  • Storage allocation rate: 130%

Description

The lower limit of this performance indicator is 0%, and the upper limit depends on the thin provisioning ratio.

Network Inbound

Definition

Measures the amount of inbound network traffic per second of a measurement object.

The measurement objects can be a host or VM.

Formula

  • Network inbound rate of a host = Number of inbound bytes of the host/Measurement period
  • Network inbound rate of a VM = Number of inbound bytes of the VM/Measurement period

Description

None

Network Outbound

Definition

Measures the amount of outbound network traffic per second of a measurement object.

The measurement objects can be a host or VM.

Formula

  • Network outbound rate of a host = Number of outbound bytes of the host/Measurement period
  • Network outbound rate of a VM = Number of outbound bytes of the VM/Measurement period

Description

None

Disk Write (kbit/s)

Definition

Measures the amount of data written to the measurement object per second.

The measurement objects can be a host or VM.

Formula

  • Disk writes of a host = Total number of bytes written to host disks/Measurement period
  • Disk writes of a VM = Total number of bytes written to VM disks/Measurement period

Description

None

Disk Read (kbit/s)

Definition

Measures the amount of data read from the measurement object per second.

The measurement objects can be a host or VM.

Formula

  • Disk reads of a host = Total number of bytes read from host disks/Measurement period
  • Disk reads of a VM = Total number of bytes read from VM disks/Measurement period

Description

None

Disk Usage

Definition

Measures the disk usage of a measurement object.

The measurement objects can be a host or VM.

Formula

  • Host disk usage = Used disk capacity of a host/Total disk capacity of the host
  • VM Disk usage = Occupied disk capacity of a VM/Total disk capacity of a VM

Description

None
