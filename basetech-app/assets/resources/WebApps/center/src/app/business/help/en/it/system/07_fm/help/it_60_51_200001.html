A virtual data center (VDC) is the primary unit of virtual resources managed in FusionManager. For example, an enterprise can assign an independent VDC for each of its department, and each VDC contains all virtual resources that can be used by the assigned department, such as computing, storage, and network resources.

VDC resources are managed by the VDC administrator. You can add or delete members for a VDC and configure scope and quota for resources in the VDC.

After FusionManager is installed, the system provides two default VDCs DefaultSharedORG and DefaultVPCORG.

Creating a VDC

You can create a VDC, select availability zones and resources for the VDC, and configure a VDC administrator that have permission to manage resources in the VDC.

Choose VDC > VDCs and click Create VDC.

Adding a VDC Administrator

Resources in a VDC are under management of the VDC administrator.

After creating a VDC, you can use one of the following methods to add administrators for the VDC:
  • Add an existing user in the system to the VDC administrator.
  • Create a new tenant for the VDC. An ftp user account for uploading files is automatically created during tenant creation.

Choose VDC > VDCs, locate the row that contains the target VDC, click More, and select Member Management.

NOTE:
When creating a user, you can set a phone number and email address for the user as contact information. Ensure that you comply with local laws and regulation when using the specified personal information.
  • Phone number: used for alarm notification by SMS. This function allows the system to send alarm information to the administrator or tenants using SMS messages after an alarm is generated. The system administrator can view the phone numbers specified for all VDC administrators and all tenants in the VDC. The administrator of a VDC can view the phone numbers specified for all tenants in the VDC. After a phone number is configured for a user, the number will be kept in the system until being deleted from the system. In this version of FusionManager, the alarm notification by SMS function has not yet been delivered for commercial use, though the phone number can be specified as part of the personal information of a user added to the system. However, the system will not use the collected phone number information for other purposes.
  • Email address: used for alarm notification by email. This function allows the system to send alarm information to the administrator or tenants using emails after an alarm is generated. The system administrator can view the email addresses specified for all VDC administrators and all tenants in the VDC. The administrator of a VDC can view the email addresses specified for all tenants in the VDC. After an email address is configured for a user, the email address will be kept in the system until being deleted from the system.

Configuring Availability Zones for a VDC

You can add an availability zone to or delete an availability zone from a VDC as need.

Choose VDC > VDCs, locate the row that contains the target VDC, click More, and select Availability Zone Management.

Managing VDC Resource Quota

After creating a VDC, you can modify the resource quota for the VDC as needed.

Choose VDC > VDCs, locate the row that contains the target VDC, and select Quota Management.

Managing an External Network

By default, a VDC is allowed to use all external networks in the availability zone of the VDC. Using the external network management function, you can allocate a specific external network in the availability zone to the VDC.

Choose VDC > VDCs, locate the row that contains the target VDC, click More, and select External Network Management.

Exporting VDC Information

You can export the information about all available VDCs and save it to the local PC as a backup or for statistics purpose.

Choose VDC > VDCs and click Export.

Deleting a VDC

If a VDC is not needed any more, you can delete the VDC from the system. Before deleting the VDC, ensure that the resources in the VDC have been deleted.

Choose VDC > VDCs, locate the row that contains the target VDC, click More, and select Delete.
