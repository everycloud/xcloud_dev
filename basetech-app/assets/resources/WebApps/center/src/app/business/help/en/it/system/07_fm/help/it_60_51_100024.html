Customers' live networks are external networks of virtual private clouds (VPCs) configured in FusionManager. External networks added to FusionManager can be used to provision VMs and create direct networks in a VPC.The created external networks that meet the following requirements can connect to a software router to provide public IP addresses for the elastic IP address service: the connection mode is subnet (common VLAN), the network is connected to the Internet, the subnet IP address assignment mode is IPv4 static injection, and the network does not contain an IPv6 subnet.

Creating an External Network

Before creating an external network, ensure that you have configured the required virtual local area networks (VLANs) or subnets on the switch and have added the VLANs to a VLAN pool. If the following conditions are met, you also need to perform the required operations on the physical switch:
  • If the external network Connection Mode parameter is set to super VLAN, you need to configure a super VLAN and a sub VLAN on the interconnected switch. For the detailed operation procedure, see Configuring VLAN Aggregation on the Switch.
    NOTE:

    Add the sub VLAN ID to the VLAN pool, and do not add the super VLAN ID to the VLAN pool.

  • If the external network is required to support IPv6, perform the following steps:
    1. Enable IPv6 on the aggregation switch. For the detailed operation procedure, see Configuring IPv6 Forwarding on the Switch.
    2. On FusionCompute, choose Network > Network Configuration and enable IPv6 forwarding settings.
  • If the external network uses an external Dynamic Host Configuration Protocol (DHCP) server to assign IP addresses, configure the DHCP server information and the DHCP trunk function on the aggregation switch. For the detailed operation procedure, see Configuring Network Information for the DHCP Server.
  • If the external network uses an internal DHCP server to assign IP addresses, perform the following steps to create a system service VM functioning as an internal DHCP server:
    1. Add a Virtual Service Appliance (VSA) management network.
    2. Import a system service VM template to the system.

    3. Choose Resource > Resource Zones, select a resource zone, choose Network Resource Pools > External Networks, navigate to the DHCP Server page, and click Create DHCP Server.

    4. Configure the DHCP server information and the DHCP trunk function on the aggregation switch. For the detailed operation procedure, see Configuring Network Information for the DHCP Server.

Choose Resource > Resource Zones, select a resource zone, choose Network Resource Pools > External Networks, and click Create.

Modifying an External Network

Modify the attributes of an external network.

Choose Resource > Resource Zones, select a resource zone, choose Network Resource Pools > External Networks, locate the row that contains the external network to be modified, and click Modify.

Deleting an External Network

If an external network is not needed any more, you can delete it from the system. However, an external network that is in use by a virtual data center (VDC) is not allowed to be deleted.

Choose Resource > Resource Zones, select a resource zone, choose Network Resource Pools > External Networks, locate the row that contains the target external network, and click Delete.

FAQ

What Is a Super Subnet?

A super subnet is a subnet that supports multiple VLANs to isolate IP addresses in the subnet. A super subnet allows multiple virtual private clouds (VPCs) to use public IP addresses in the same network segment. This feature not only ensures public IP address isolation between VPCs but also maximizes the public IP address usage. A VLAN of a super subnet can be used by only one VPC. The IP addresses available to a super subnet depends on the subnet settings. The IP addresses in a subnet can be used by multiple VLANs configured for the subnet.
