Configuring IPv6 Forwarding on the Switch

  • Scenarios
  • Prerequisites
  • Procedure
  • Example

Scenarios

To enable FusionManager to support an IPv6 subnet, the IPv6 forwarding function must be configured on the aggregation switch. This topic describes how to configure the IPv6 function on the aggregation switch.

Prerequisites

  • The aggregation switch supports IPv6.
  • You have logged in to the switch using a terminal emulation tool.

Procedure

  1. Log in to the aggregation switch.
  2. In the system view interface, run the ipv6 command to enable global mode.
  3. Run the interface vlanif VLAN ID command to enter the VLANIF interface of the planned IPv6 VLAN.
  4. Run the ipv6 enable command to enable the IPv6 forwarding function.
  5. Run the ipv6 address IPv6 address command to configure the IPv6 address for the VLANIF interface.
  6. Run the quit command to exit the VLANIF interface.

Example

To enable IPv6 on VLAN 100 and set the VLANIF IPv6 address to 1001::1/64, run the following command on the aggregation switch:

<Quidway> system-view

[Quidway] ipv6

[Quidway] interface vlanif 100

[Quidway-Vlanif100] ipv6 enable

[Quidway-Vlanif100] ipv6 address 1001::1/64

[Quidway-Vlanif100] quit
