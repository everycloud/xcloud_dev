操作场景

该任务指导管理员配置VPC互通功能。开启VPC互通功能后，不同VPC之间的网络可以通过路由器实现互通，同时租户在VPC中可以使用ACL、VPN功能。

前提条件

  • 已部署netMatrix。
  • 已获取FusionManager节点“galaxmanager”用户的密码。

操作步骤

    1. 使用“PuTTY”，以“galaxmanager”用户登录FusionManager节点。

      • FusionManager节点主备部署时，通过浮动IP地址登录主节点。
      • FusionManager单节点部署时，以管理IP地址登录。

    2. 执行以下命令，防止“PuTTY”超时退出。

      TMOUT=0

    3. 执行以下命令，配置VPC互通功能。

      modConfig confRoutersCommunicate -vxlanRange startVxlan-endVxlan -isCommunicate (true|false)

      • “startVxlan-endVxlan”表示共享网络的起始VXLAN ID和结束VXLAN ID，这里的共享网络用于为路由器分配网关地址，在创建业务网络时不能使用此VXLAN范围。

      • “true”表示开启VPC互通功能；“false”表示关闭VPC互通功能，系统默认关闭此功能。

      例如，执行以下命令，将VXLAN ID为8000-8002的网络设置为VPC互通使用的共享网络，并开启VPC互通功能。

      modConfig confRoutersCommunicate -vxlanRange 8001-8002 -isCommunicate true

      回显信息如下：
      configure success

    4. FusionManager节点主备部署时，执行以下命令，同步主备配置。

      haNotifySyncFiles /opt/goku/services/irm/rpool/config/network.properties

      回显信息如下：
      execute command syncfile successfully.
