VNC Login Troubleshooting

  • Context
  • Possible Causes
  • Symptom and Solution

Context

If VM page cannot be displayed when users log in to a VM using VNC in FusionManager, the possible causes may be as follows.

Possible Causes

  • The browser settings are incorrect.
  • The JRE version is incorrect. The JRE must be Java Plug-in 1.7.0_15 (32–bit) or later (32–bit).

Symptom and Solution

Symptom: A pop-up blocking message or a security warning is displayed, or plug-ins are blocked.

Solution:
  • Disable the pop-up blocker of the browser.
  • Run this application program.
  • Select Always activate plugins for this site from the Activate All Plugins drop-down list.

Symptom: The user is prompted to download and install the JRE

Solution:
  1. Download and install the JRE.
  2. Log in to the VM using VNC again.

Symptom: After the JRE is installed, the user is still prompted to install the JRE when logging in to the VM using VNC.

Solution: Enable Deployment Toolkit. The following steps describe how to resolve the problem using the Internet Explorer 8 (IE8).
  1. Choose Tools and select Manage Add-ons.
  2. Select Run without permission from the Show drop-down list.
  3. Right-click Deployment Toolkit in the right pane and choose Enable.
  4. Log in to the VM using VNC again.

Symptom: The user still fails to log in to the VM using VNC after installing the JRE as prompted.

Solution:
  1. Right-click on the toolbar at the lower right corner of the PC and select Open Control Panel.
    NOTE:
    If is not displayed on the toolbar at the lower right corner of the PC, open the Java Control Panel by performing the following steps:
    1. Click Start.
    2. Enter cmd In the Search programs and files text box.
    3. Enter java -version in the displayed command line interface (CLI), and check the version number in the command output.
    4. Open the version path in the jre program installation directory, and run javacpl.exe to display the Java Control Panel.

      For example, if the Java version is 1.7.0_15, and the version path in the jre program installation directory is C:\Program Files\Java\jre7\bin, run javacpl.exe in this path to display the Java Control Panel.

  2. Click Advanced.
  3. Deselect Enable the next-generation Java Plug-in(requires browser restart).
  4. Click Security.
  5. Select Enable Java content in the browser.
  6. Log in to the VM using VNC again.

Symptom: The user still fails to log in to the VM using VNC after installing the JRE as prompted and performing corresponding configuration.

Solution:
  1. Uninstall all jre programs on the PC and ensure that no jre or jdk folder exists in the jre installation directory.
  2. After uninstallation is complete, re-install the Java Plug-in 1.7.0_15 (32–bit) or later (32–bit)
  3. Log in to the VM using VNC again.

Symptom: The keyboard does not function in a VM.

Solution:

Click Alt or Ctrl on the VNC toolbar to unlock the keyboard.

Symptom: When Internet Explorer is used, the displayed VNC login window is empty without showing any message, and the VM window does not appear either.

Solution:

Press F12 and set Browser to Internet Explorer 9 in the menu bar of the Developer Tools window, and set Document Mode to Internet Explorer 9 Standards(Page Default).
