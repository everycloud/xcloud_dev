Configuring the VM SID Customization

  • Scenarios
  • Prerequisites
  • Procedure

Scenarios

To enable the security identifier (SID) of the VMs created using the template to be different or to be added to a domain, this operation is required.

If there is no requirements for the VM SIDs, this operation is not required.

NOTE:

Declaration: This feature is a high-risk feature. Using this feature complies with industry practices. However, end user data may be required for implementing the feature. Exercise caution and obtain end user's consent when using this feature.

This operation configures user VM registries and the script executed for the initial startup. If the end user data is involved, obtain end user's consent before you perform this operation.

Prerequisites

Conditions

  • The OS and the Tools have been installed on the VM.

  • You have obtained the software package of the ISO image that contains the configuration file.

    The software package name is FusionCompute V100R005C00SPC200_GuestOSDrivers.zip, and the ISO file name is FusionCompute V100R005C00SPC200_Customization.iso.

    To obtain the software package
    • For enterprise users, visit http://support.huawei.com/enterprise and choose Software > IT > FusionCloud > FusionSphere > FusionCompute > V100R005C00SPC100.
    • For telecom carrier users, visit http://support.huawei.com and choose Software > Carrier Software & Core Network > Carrier Cloud Computing > IT Cloud Infrastructure Solution > Cloud OS > FusionCompute > V100R005C00SPC100.
    NOTE:
    Verify the obtained software packages. For details, see Verifying Software Packages.

Procedure

    Delete the historical file.

    1. In the VNC login window, enable the user administrator of the OS and delete other users.
    2. Go to the root directory of the C drive and check whether the setpass.vbs file exists.

      If yes, delete the setpass.vbs file.

    Check the VM OS version.

    1. Perform the required steps based on the VM OS version.

      • If the OS is Windows XP, perform 4 to 11.
      • If the OS is Windows 7, perform 17 to 27.
      • If the OS is Windows 8, perform 32 to 42.
      • If the OS is Windows Server 2003, perform 47 to 58.
      • If the OS is Windows Server 2008, perform 63 to 73.
      • If the OS is Windows Server 2012, perform 78 to 88.
        NOTE:
        The Windows Server 2012 R2 Datacenter 64-bit OS does not support SID configuration.
      • If the OS is not mentioned above, VM SID configuration is not supported.

    Configure SID for a Windows XP VM.

    1. In the VNC login window, enable the user administrator of the OS and delete other users.
    2. Mount the ISO image that contains the configuration file.

      The ISO image name is FusionCompute V100R005C00SPC200_Customization.iso.

      NOTE:
      If you did not obtain the ISO image, you can copy the configuration file from the /home/GalaX8800/customization directory on any host in the cluster.

    3. Copy the sysprep folder of the VM OS in the ISO image to the root directory of drive C on the VM.
    4. Change the following parameters in the sysprep.inf file in the C:\sysprep directory on the VM:

      1. Between the quotation marks on the right of AdminPassword, set the password for logging in to the current VM.
      2. ProductKey: Change it to the serial number of the purchased Windows OS.

      3. Set the value of TimeZone to the local time zone index value.

        You can visit http://msdn.microsoft.com/en-us and enter Microsoft Time Zone Index Values in the search box to query the required local time zone index value.

      4. Change the value of LanguageGroup to the language group ID of the local language.

        You can visit http://support.microsoft.com/?ln=en-us and enter Language Group IDs in the search box to query the required language group ID.

    5. Mount the OS ISO file to the VM.
    6. Decompress the DEPLOY.CAB file in the ISO file to the SUPPORT\TOOLS\ directory.
    7. Copy all the files generated after DEPLOY.CAB is decompressed to the C:\sysprep directory on the VM.
    8. In the command line interface (CLI) of the VM in the VNC login window, run the following command to configure the VM SID:

      C:\sysprep\sysprep -mini -noreboot -reseal -quiet

    9. On the VM desktop displayed in the VNC window, click Start, and enter regedit in the search box, and press Enter.

      A dialog box is displayed.

    10. Choose HKEY_LOCAL_MACHINE > SOFTWARE > Microsoft > Windows > CurrentVersion > Run.

      All registry entries under Run are displayed.

    11. Double click (Default).

      A dialog box is displayed.

    12. Enter c:\setpass.vbs in Value data.
    13. Click OK.

      The dialog box is closed.

    No further action is required.

    Configure SID for a Windows 7 VM.

    1. In the VNC login window, enable the user administrator of the OS and delete other users.
    2. Mount the ISO image that contains the configuration file.

      The ISO image name is FusionCompute V100R005C00SPC200_Customization.iso.

      NOTE:
      If you did not obtain the ISO image, you can copy the configuration file from the /home/GalaX8800/customization directory on any host in the cluster.

    3. Copy the X86 or X64 folder of the VM OS in the ISO image to the C:\Windows\System32 directory on the VM.

      • If the VM runs a 32-bit OS, copy the X86 folder.
      • If the VM runs a 64-bit OS, copy the X64 folder.

    4. Modify the VM directory file on the VM.

      • If the VM runs a 32-bit OS, the directory file is C:\Windows\System32\X86\professional\Untitled.xml.
      • If the VM runs a 64-bit OS, the directory file is C:\Windows\System32\X64\professional\Untitled.xml.

      The modification includes:

      1. In the VM directory file, locate the Value tags between the Password tags and set the password for logging in to the current VM between the Value tags.

      2. Change the system language setting in the following field to the current VM OS language:

        <InputLocale>zh-CN</InputLocale>
<SystemLocale>zh-CN</SystemLocale>
<UILanguage>zh-CN</UILanguage>
<UILanguageFallback>zh-CN</UILanguageFallback>
<UserLocale>zh-CN</UserLocale>

        The default system language is zh-CN, which indicates the simple Chinese.

        To change the language to English, change zh-CN to en-US.

        For more language settings, enter Available Language Packs at the Microsoft official website to search for language IDs.

      3. Set the value of TimeZone to the current time zone name and change the value of RegisteredOwner in the previous row to the name of the country you currently reside.

        You can visit http://msdn.microsoft.com/en-us and enter Microsoft Time Zone Index Values in the search box to query the time zone name.

    5. In the CLI, run the following command to configure the VM SID:

      • If the VM runs a 32-bit OS, run the C:\Windows\System32\X86\professional\SysprepCmd.cmd command.
      • If the VM runs a 64-bit OS, run the C:\Windows\System32\X64\professional\SysprepCmd.cmd command.

      NOTE:
      • If no error message is displayed, the command is executed successfully.
      • Do not run this command repeatedly.

    6. Click Start, and enter regedit in the search box, and press Enter.

      A dialog box is displayed.

    7. Choose HKEY_LOCAL_MACHINE > SYSTEM > Setup.

      All registry entries under Setup are displayed.

    8. Change the value of Cmdline from oobe\windeploy.exe to X86\professional\deploy.cmd or X64\professional\deploy.cmd using the OS registry editor.

      • If the VM runs a 32-bit OS, change the value to X86\professional\deploy.cmd.
      • If the VM runs a 64-bit OS, change the value to X64\professional\deploy.cmd.

    9. Choose HKEY_LOCAL_MACHINE > SOFTWARE > Microsoft > Windows NT > CurrentVersion > Winlogon.

      All registry entries under Winlogon are displayed.

    10. Change the value of DefaultPassword to the password for logging in to the current VM using the registry editor.

      If DefaultPassword is not displayed, create a string value and name it DefaultPassword.

    11. Change the AutoAdminLogon value to 1.

      If AutoAdminLogon is not displayed, create a string value and name it AutoAdminLogon.

      No further action is required.

    12. Choose HKEY_LOCAL_MACHINE > SOFTWARE > Microsoft > Windows > CurrentVersion > Run.

      All registry entries under Run are displayed.

    13. Double click (Default).

      A dialog box is displayed.

    14. Enter c:\setpass.vbs in Value data.
    15. Click OK.

      The dialog box is closed.

    No further action is required.
