Load Balancers

  • Adding a Load Balancer to FusionManager.
  • Modifying Load Balancer Connection Parameters
  • Deleting a Load Balancer

A load balancer manages received data traffic and increases effective network bandwidth by distributing network connection requests to multiple servers using a distribution algorithm. After a load balancer is added to FusionManager, it can provide virtual load balancing services to provide users with the optimum network service experience.

After being added to FusionManager, a load balancer can be used to:
  • Balance workloads on hosts added to the system.
  • Provide virtual load balancer (VLB) services to balance workloads on VMs in the system.

Adding a Load Balancer to FusionManager.

Choose Resource > Devices > Network Devices > Load Balancers and click Add Load Balancer.

After a load balancer is added to FusionManager, the load balancer information is displayed on the Load Balancers page.

If the required server model is not available for selection, include more compatible hardware devices into the system. For details, see Adaptation Package Management.

Modifying Load Balancer Connection Parameters

If the management information of a load balancer has been modified after the load balancer is added to FusionManager, you need to synchronize the new information to FusionManager in a timely manner. Otherwise, FusionManager will lose management for the load balancer.

Choose Resource > Devices > Network Devices > Load Balancers, locate the row that contains the target load balancer, and click Modify Connection Parameters.

Deleting a Load Balancer

If a load balancer is not needed any more, you can delete it from the system. After a load balancer is deleted, FusionManager will not monitor or manage this load balancer, and the services associated with this load balancer will be unavailable.

Choose Resource > Devices > Network Devices > Load Balancers, locate the row that contains the target load balancer, and click Delete.
