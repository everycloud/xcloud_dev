VMs

A virtual machine (VM) is a software computer that functions like a physical computer, running an operating system (OS) and applications.

A VM runs on a host and obtains CPUs, memory, graphics cards, universal serial bus (USB) devices, network resources, and storage access capabilities of the host. Multiple VMs can run concurrently on a host.

VM Power Management

Power management for a VM includes hibernate, start or wake up, stop, restart, pause, and unpause the VM. Stopping or restarting a VM can be performed in a safe or forcible manner. You are advised to use the safe manner. Forcibly stop or restart a VM only if the VM cannot be safely stopped or restarted due to an OS fault. Forcibly stopping or restarting a VM may damage the VM disks and cause data loss.

Starting or stopping a large number of VMs increases host workload. If you need to start or stop a large number of VMs on a host, perform the start or stop operations in batches to prevent VM services on the host from being adversely affected.

Locate the row that contains the target VM, and select Start or the desired operation in More.

Migrating a VM

For details about requirements, see VM Migration and Specification Modification Requirements.

Live migration

If the service load is over light or heavy on the host where the VMs run or the host is faulty, migrate the VMs to ensure that they work properly.
  • If the service load is too heavy on the host or the host is faulty, migrate the VMs to another host to prevent service interruption due to a host exception.
  • If the service load is too light on multiple hosts, migrate the VMs to a small number of hosts to consolidate host resources and then power off the idle hosts. This improves resource utilization and reduces power consumption.

Locate the row that contains the target VM, click More, and select Live Migrate.

Cold Migration

Locate the row that contains the target VM, click More, and select Cold Migrate.

After cold migration, you also need to click More and select Confirm Modification to make the system continue the subsequent operations.

Confirming VM Modifications

After performing a cold VM migration or modifying the VM specifications, the system can continue the subsequent operations only after you manually confirm the modifications.

Locate the row that contains the target VM, click More, and select Confirm Modification.
