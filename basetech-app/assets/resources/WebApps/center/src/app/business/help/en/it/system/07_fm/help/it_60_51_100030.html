A VM template is a duplicate of a VM, which includes an OS, software packages, and a set of VM specifications. VM templates can be used to:
  • Provision VMs. Creating VMs using a VM template can greatly reduce the time for creating the VMs and configuring operating systems (OSs) for them.
  • Provision application instances: A VM template can be used to create multiple VMs for an application instance. An application instance can be used to provision application instances.

Obtaining a VM Template

You can obtain a VM in either of the following ways:

  1. Discover an existing VM template in FusionManager

    Discover a VM template that is created in the hypervisor and has been added to FusionManager.

    Choose Resource > Templates and Specs > VM Templates and click Discover.

  2. Customize a VM template

    Use an OS ISO image file to create a VM template and configure the VM OS as needed.

    Choose Resource > Templates and Specs > VM Templates and click Create.

    For the detailed operation procedure, see Creating a VM Template (Windows) or Creating a VM Template (Linux).

NOTE:

How to obtain a VSA or PVM VM template, For details, see Importing a VSA VM Template or Creating a PVM Template.

Modifying a VM Template

Modifying Basic Information About a VM Template

Modify basic information about a VM template, including the name, OS type, and OS version.

Choose Resource > Templates and Specs > VM Templates, locate the row that contains the target VM template, click More, and select Modify VM Template Information.

Changing Software in a VM Template

Install new software or uninstall existing software in a VM template.

  1. Choose Resource > Templates and Specs > VM Templates, locate the row that contains the target VM template, click More, and select Modify VM Template Software.
  2. Choose Resource > Templates and Specs > VM Templates, locate the row that contains the target VM template, click More, and select Install Software.

Changing VM Template Attributes

Change the hardware attributes of a VM template, including the CPUs, memory, NICs, and disks. A VM template is equivalent to a stopped VM, and changing its hardware attributes is the same as changing VM hardware attributes.

Choose Resource > Templates and Specs > VM Templates, locate the row that contains the target VM template, click More, and select Manage VM Template Attributes.

Converting a VM Template into a VM

Convert a VM template into a VM.

Choose Resource > Templates and Specs > VM Templates, locate the row that contains the target VM template, click More, and select Convert to VM.

Deleting a VM Template

If a VM template is not needed any more, you can delete it from the system.

Choose Resource > Templates and Specs > VM Templates, locate the row that contains the target VM template, click More, and select Delete.

VM Logical Template

A VM logical template is a set of VM templates with the same specifications and OS settings. When FusionManager functions as a service provider, it provides VM templates to other management platforms using VM logical templates. When the VM templates with the same specifications in different hypervisors are provided to upper-layer management platforms as one VM logical template, the differences of the VM templates in the hypervisors are hidden from the management platforms.

For example, in data centers, FusionManager functions as a service provider that manages multiple hypervisors, and the VM creation and provisioning are implemented by upper-layer software platform. In this case, FusionManager needs to provide the VM logical templates for upper-layer software to provision VMs.

You can perform the following operations to create a VM logical template:

  1. Choose Resource > Templates and Specs > VM Templates, click the VM Logical Template tab, and click Create.

  2. Associate the VM templates with the same specifications and OS settings with the VM logical template.

    Choose Resource > Templates and Specs > VM Templates, locate the row that contains the target VM template, click More, and select Associate with Logical Template.

  3. Choose Resource > Templates and Specs > VM Templates, click the VM Logical Template tab, click More, and select Activate.
