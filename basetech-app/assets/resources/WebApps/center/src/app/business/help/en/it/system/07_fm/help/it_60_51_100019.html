A computing resource pool is a collection of computing resources to be allocated to VMs or application instances in the FusionManager system.

Computing resources in a resource cluster that is added to FusionManager can be used for service provisioning only after the resource cluster is associated with a resource zone in FusionManager.

Associating a Resource Cluster with a Resource Zone

Associate the resource clusters in a hypervisor to a resource zone to provide computing resources for service provisioning.

Choose Resource > Resource Zones, select a resource zone, navigate to the Computing Resource Pools page, and click Associate with Cluster.

After a resource cluster is associated with a resource zone in FusionManager, all the computing and storage resources contained in the resource cluster are added to FusionManager and respectively displayed on the Computing Resource Pools and Storage Resource Pools pages. If some resources in a hypervisor are not included in its resource clusters, these resources cannot be added to FusionManager resource pools.

During resource cluster association, configure storage media for storages in the resource cluster, which means that you need to configure a storage SLA flag for each storage in the cluster. Then, you can select required storages to create VMs or disks based on the configured storage media.

Disassociating a Resource Cluster from Its Resource Zone

To change the resource zone to which the resource cluster belongs or delete a resource cluster from its associated resource zone, disassociate the resource cluster from the resource zone first. If the resources in the resource cluster are already in use, the resource cluster cannot be deleted.

Choose Resource > Resource Zones, select a resource zone, navigate to the Computing Resource Pools page, locate the row that contains the target resource cluster, and click Disassociate.

Managing Tags

After defining tags, you can create VMs using the resources with required tags. For example, because cluster A contains sufficient resources and provides high reliability, you can add create tag SLA whose value is set to GOLD for cluster A. In this case, you can create VMs in the cluster tagged with GOLD to ensure VM performance.

The tags available are all defined on the Services > Tags page.

Choose Resource > Resource Zones, select a resource zone, navigate to the Computing Resource Pools page, locate the row that contains the target resource cluster, and click Tags.
