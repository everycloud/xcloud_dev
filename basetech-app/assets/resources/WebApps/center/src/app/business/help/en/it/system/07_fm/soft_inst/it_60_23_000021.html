Scenarios

Verify the obtained software package using sha256deep.

NOTE:
This topic uses a the local PC that runs Windows 7 as an example to describe how to verify a software package.

Prerequisites

  • You have obtained the software package and its verification file. The verification file has the same name as that of the software package, and its file name extension is *.sha256.
  • You have obtained sha256deep.exe.
    It is a free tool and can be obtained by visiting http://sourceforge.net/projects/md5deep/files/ to download md5deep-x.x.zip.
    NOTE:

    After decompressing the md5deep-x.x.zip file, select the sha256deep tool whose version matches the local PC OS. If the local PC runs Windows XP or Windows 7 32-bit, select sha256deep.exe. If the local PC runs Windows 7 64-bit, select sha256deep64.exe.

Software

You have obtained

FusionCompute V100R005C00SPC200_VSAM.zip

.

Procedure

    1. On the local PC, choose Start > Run.
    2. Enter cmd, click OK.
    3. Run the following command to switch to the disk containing sha256deep:

      Drive letter of the drive containing sha256deep:

      For example, if sha256deep is located in drive D, run the following command:

      D:

    4. Run the following command to switch to the directory containing sha256deep:

      cd / directory containing sha256deep

      For example, if sha256deep is located in the admin/test/ directory, run the following command:

      cd /admin/test/

    5. Run the following command to verify the software package:

      sha256deep.exe -x Path to verification file Path to software package

      For example, if the path to the verification file is D:\software\FusionManager V100R005C00_SV.iso.sha256, and the path to the software package is D:\software\FusionManager V100R005C00_SV.iso, run the following command:

      sha256deep.exe -x "D:\software\FusionManager V100R005C00_SV.iso.sha256" "D:\software\FusionManager V100R005C00_SV.iso"

      • The package is correct if no path to the software package is displayed. For example, if information similar to the following is displayed, the package is correct:

        D:\admin\test>

      • The package is incorrect if the path to the software package is displayed. You need to download a new software package. For example, if information similar to the following is displayed, the package is incorrect:

        D:\software\FusionManager V100R005C00_SV.iso

D:\admin\test>
