Introduction to the FusionManager Web Client

  • Functions of the Administrator View

The FusionManager Web Client provides two operation views, administrator view and tenant view. The administrator can manage all resources in the system, whereas tenants can manage only the resources of their own virtual data centers (VDCs). This part describes the functions of the administrator view.

Functions of the Administrator View

Table 1 Functions of the administrator view

SN

Category

Functions

1

Services
You can manage public resources or objects in the system, including cloud resource pools, tags, and application templates.
  • Cloud resource pool: You can add FusionSphere OpenStack nodes to one FusionManager node to manage.
  • Application template: An application template is a service model that combines logical components, including VMs, software packages, scripts, and networks, based on the configured relationship, thereby enabling VM creation in batches and application instance deployment with one click.
  • Topology: A topology displays the hierarchical relationship of system resources.

2

Resources

Resources refer to all hardware and virtual resources managed by FusionManager, including FusionSphere OpenStack instances, devices, and images.

3

VDC

You can create a VDC and specify tenants for the VDC. Only the tenants of a VDC can manage VMs in this VDC.

On the VDC page, you can manage VDCs, VMs, and disks.

4

Users
You can manage users, roles, and keystone.
  • A role is a collection of diverse operation rights. You can assign roles to users so that each user can have different operation rights.
  • Keystone is a rights management and authentication service provided by FusionSphere OpenStack. After the Keystone service is added to FusionManager, the Keystone rights management function applies to FusionManager.

5

Monitoring
Monitored objects include alarms and performance.
  • Alarms: The alarm list of FusionManager displays the alarms generated for both FusionSphere and third-party components connected to FusionManager. Using the alarm information, you can find and locate faults promptly.
  • Performance: The performance list of FusionManager provides performance monitoring indicators of the hosts, and VMs in the system.
  • Reports: System reports can be downloaded as required.

6

System

You can configure system license and system time. Typically, you only need to configure system data only once.
