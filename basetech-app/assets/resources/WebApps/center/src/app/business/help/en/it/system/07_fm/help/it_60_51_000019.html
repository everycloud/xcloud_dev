Monitoring

Monitored objects include alarms, reports, and performance. Table 1 lists the monitoring scenarios.
Table 1 Monitoring scenarios

Category

Subcategory

Application Scenario

Alarms
  • Alarm list
  • Alarm settings
  • Alarm statistics

You can ascertain the running status of the entire system by viewing alarms and clear alarms promptly to ensure proper system running.

During initial system configuration, you can configure alarm attributes, including the alarm threshold, alarm masking function, and alarm notification by email. You can configure these attributes only once and modify them as required.

Reports
  • System reports
  • Custom reports

The system provides report information about servers, hosts, VMs, resource clusters, networks, and storage.

To obtain more information, you can customize reports.

Performance

Monitoring view

Performance monitoring provides visibility into details about performance indicators, such as the CPU usage of a resource cluster, host, or VM.
