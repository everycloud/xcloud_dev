Prerequisites

Conditions

  • The OS has been installed on the VM.
  • You have obtained the ISO file that contains the new kernel files.

    The software package name is FusionCompute V100R005C00SPC200_GuestOSDrivers.zip, and the ISO file name is FusionCompute V100R005C00SPC200_GuestOS.iso.

    To obtain the software package
    • For enterprise users, visit http://support.huawei.com/enterprise and choose Software > IT > FusionCloud > FusionSphere > FusionCompute > V100R005C00SPC100.
    • For telecom carrier users, visit http://support.huawei.com and choose Software > Carrier Software & Core Network > Carrier Cloud Computing > IT Cloud Infrastructure Solution > Cloud OS > FusionCompute > V100R005C00SPC100.
    NOTE:
    Verify the obtained software packages. For details, see Verifying Software Packages.

Procedure

    Mount an ISO file.

    1. Log in to the VM using VNC as user root.
    2. Mount a CD/DVD-ROM drive or an ISO File.

      The FusionCompute V100R005C00SPC200_GuestOS.iso is required ISO file to be mounted.

    Replace the kernel file.

    1. On the VM desktop displayed in the VNC window, right-click the desktop, and choose Open in Terminal.

      The command line interface (CLI) window is displayed.

    2. Run the following command to create the xvdd directory:

      mkdir xvdd

    3. Run the following commands to mount a CD/DVD-ROM drive to the VM:

      mount Path to be mounted xvdd

      For example, mount /dev/sr0 xvdd

      Table 2 lists the directories for mounting a CD/DVD-ROM drive or ISO file to a VM running the Linux OS.

      If the VMware hypervisor is used, change the CD-ROM drive mounting directory by following the procedures in the documentation for the hypervisor and the OS in use.

    4. Run the following command to enter the path saving the kernel package:

      cd xvdd/pvDriver/kernel\ patch

    5. Run the following command to view the kernel file name:

      ls

    6. Run the following command to copy the new kernel file to the root folder:

      cp Kernel file name /root

      For example, cp kernel_patch-redhat_enterprise_linux_4.7.tar.gz /root.

    7. Run the following commands to decompress the package containing the kernel files:

      cd /root

      tar -xzf Name of the package

      For example, run the tar -xzf kernel_patch-redhat_enterprise_linux_4.7.tar.gz command.

    8. Run the following command to replace the existing kernel files with the copied kernel files:

      rpm -ivh --force --nodeps Kernel file name

      For example, run the following commands:

      rpm -ivh --force --nodeps kernel-2.6.9-78.EL.i686.rpm

      rpm -ivh --force --nodeps kernel-devel-2.6.9-78.EL.i686.rpm

      NOTE:

      Ignore the alerts that may be displayed during the kernel replacement process.

    9. Run the following command to restart the OS of the VM to make the new kernel files take effect:

      reboot

      No further action is required.
