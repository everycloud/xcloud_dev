Virtual Firewalls

The Virtual Firewalls page displays all virtual firewalls created in the virtual private clouds (VPCs) of a resource zone. Services including elastic IP address, network address translation (NAT), access control list (ACL), and virtual private network (VPN) require virtual firewalls.

Virtual firewalls can be classified in to the following types:
  • Hardware firewall: provided by a virtual firewall of physical firewall.
  • Software firewall: provided by a system service VMs in the FusionManager.

Obtaining a Virtual Firewall

Prerequisites

  • To obtain a hardware firewall, ensure that you have added the required physical firewall to FusionManager and have created virtual firewalls on the physical firewall.
  • To obtain a software firewall, ensure that you have added a Virtual Software Appliance (VSA) management network in the network resource pool of the current resource zone and have imported the VSA VM template to the system.

How to Obtain

Apply for a virtual firewall in the VPC.

Configuring DNAT Parameters

Perform this operation if the DNAT function is required. The parameters include the public network start port, end port, and access duration. The access duration limits the time period allowed for a public network user to use the VM. After the duration expires, the user is automatically disconnected from the VM and needs to reconnect to the VM if attempting to continue using the VM. For details, see NAT.

Choose Resource > Resource Zones, select a resource zone, choose Network Resource Pools > Virtual Firewalls, and click Configure DNAT Parameters.
