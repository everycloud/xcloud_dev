A Virtual Service Appliance (VSA) management network, which provides management network resources for system service VMs, is a basic network configuration required by FusionManager.

System service VMs are automatically created by FusionManager to provide system services. For more information about system service VMs, see System Service VMs.

Creating a VSA Management Network

Before creating a VSA management network, ensure that you have configured the required virtual local area network (VLAN) and gateway on the switch and have added the VLAN to the distributed virtual switch (DVS) created on the FusionCompute hypervisor. This VLAN cannot be added to the VLAN pool set on FusionManager.
  • If the network resources are insufficient and the VLAN ID of the FusionManager node is not 0, the FusionManager node VLAN and subnets can be shared with the VSA management network, but the VSA management network cannot use the IP address segments set for the FusionManager node.
  • To use new VLAN and subnet resources for the VSA management network, configure the following settings on the switch:
    1. Configure the aggregation switch and all the access switches to allow pass of the packets from the VSA management network VLAN.
    2. Configure the VSA management network subnet and gateway on the aggregation switch.
    3. Add a route between the VSA management network gateway and the FusionManager management plane gateway, if these two gateways are in different Virtual Routing and Forwarding (VRF) tables.

Choose Resource > Resource Zones, select a resource zone, choose Network Resource Pools > VSA Networks and click Create VSA Management Networks.

Deleting a VSA Management Network

If a VSA management network is not needed any more, you can delete it from the system. However, an in-use VSA management network is not allowed to be deleted.

Choose Resource > Resource Zones, select a resource zone, choose Network Resource Pools > VSA Networks page, locate the row that contains the target VSA network, and click Delete.

After deleting the VSA management network, manually delete the matching VLAN information on the switch to release IP address and subnet resources in the network.

Creating a VXLAN Tunnel Endpoint (VTEP) Network

If the virtual extensible LAN (VXLAN) function is enabled, a VTEP network can assign IP addresses for the software router in the system to allow communication between the software router and the hosts over a VTEP channel.

Choose Resource > Resource Zones, select a resource zone, choose Network Resource Pools > VSA Networks, select VTEP Network, and click Create VTEP Network.

Modifying Attributes of a VTEP Network

You can modify the attributes of a VTEP network as needed.

Choose Resource > Resource Zones, select a resource zone, choose Network Resource Pools > VSA Networks, select VTEP Network, locate the row that contains the target VTEP network, and click Modify.

Deleting a VTEP Network

If a VTEP network is not needed any more, you can delete it from the system to release network resources. However, if a VTEP network is in use, it cannot be deleted.

Choose Resource > Resource Zones, select a resource zone, choose Network Resource Pools > VSA Networks, select VTEP Network, locate the row that contains the target VTEP network, and click Delete.
