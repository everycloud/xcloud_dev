VDCs

A virtual data center (VDC) is the primary unit of virtual resources managed in FusionManager. For example, an enterprise can assign an independent VDC for each of its department, and each VDC contains all virtual resources that can be used by the assigned department, such as computing, storage, and network resources.

VDC resources are managed by the VDC administrator. You can add or delete members for a VDC and configure scope and quota for resources in the VDC.

Creating a VDC

You can create a VDC, and configure a VDC administrator that have permission to manage resources in the VDC.

A VDC is equivalent to a domain in FusionSphere OpenStack.

Adding a VDC Administrator

Resources in a VDC are under management of the VDC administrator.

After creating a VDC, you can create a new tenant for the VDC to add administrators for the VDC.

Locate the row that contains the target VDC, click More, and select Member Management.

NOTE:
When creating a user, you can set a email address for the user as contact information. Ensure that you comply with local laws and regulation when using the specified personal information.

Email address is used for alarm notification by email. This function allows the system to send alarm information to the administrator or tenants using emails after an alarm is generated. The system administrator can view the email addresses specified for all VDC administrators and all tenants in the VDC. The administrator of a VDC can view the email addresses specified for all tenants in the VDC. After an email address is configured for a user, the email address will be kept in the system until being deleted from the system.

Managing VDC Resource Quota

After creating a VDC, you can modify the resource quota for the VDC as needed.

Locate the row that contains the target VDC and select Quota Management.

Managing a Cloud Resource Pool

After creating a VDC, manage the cloud resource pool available to the VDC based on the service requirements.

Locate the row that contains the target VDC, click More, and select Cloud Resource Pool Management.

Exporting VDC Information

You can export the information about all available VDCs and save it to the local PC as a backup or for statistics purpose.

Deleting a VDC

If a VDC is not needed any more, you can delete the VDC from the system. Before deleting the VDC, ensure that the resources in the VDC have been deleted.

Locate the row that contains the target VDC, click More, and select Delete.
