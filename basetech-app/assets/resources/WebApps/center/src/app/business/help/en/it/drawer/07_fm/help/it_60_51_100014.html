Resource Cluster

A resource cluster is a collection of computing, storage, and network resources that have the same resource attributes in a hypervisor. A resource cluster can be displayed on the resource pool page only after it is associated with a resource zone and can be used by FusionManager to provision VMs or application instances only after it is added to an availability zone (AZ).

Multiple resource clusters constitute an AZ. Both AZs and resource clusters can serve as the resource pool for provisioning VMs and application instances.

Associating a Resource Cluster

FusionManager can manage only its associated resource clusters.

During resource cluster association, configure storage media for storages in the resource cluster, which means that you need to configure a storage SLA flag for each storage in the cluster. Then, you can select required storages to create VMs or disks based on the configured storage media.

Disassociating a Resource Cluster

To change the resource zone to which the resource cluster belongs or delete a resource cluster from a resource zone, disassociate the resource cluster from the resource zone first. The resource cluster that has been added to an AZ cannot be disassociated.

Managing Tags

After defining tags, you can create VMs using the resources with required tags. For example, because cluster A contains sufficient resources and provides high reliability, you can add create tag SLA whose value is set to GOLD for cluster A. In this case, you can create VMs in the cluster tagged with GOLD to ensure VM performance.

The tags available are all defined on the Services > Tags page.
