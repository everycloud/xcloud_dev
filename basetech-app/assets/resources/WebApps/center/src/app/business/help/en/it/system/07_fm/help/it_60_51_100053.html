A virtual machine (VM) is a software computer that functions like a physical computer, running an operating system (OS) and applications.

A VM runs on a host and obtains CPUs, memory, graphics cards, universal serial bus (USB) devices, network resources, and storage access capabilities of the host. Multiple VMs can run concurrently on a host.

Creating a VM

You can use a VM template to create a VM. During the creation process, you can adjust the VM attributes as needed. However, the adjusted attributes only apply to the newly created VM and will not change the attributes of the VM template.
  • Using a template available in the system: Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, and click Create.
  • Using a local template: Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, and click Import VM.

To ensure VM security, the VM user must change the VM login password in the earliest time.

Logging In to a VM Using VNC

VNC is a remote terminal access tool. You can use VNC to remotely connect to a VM, view the VM operating page, and perform management and maintenance operations on it.

However, if the VM operating system (OS) requires password input for login, you need to enter the correct OS username and password to access the VM operating page.

Note the following points when you log in to a VM using VNC:
  • Declaration: This feature is a high-risk feature. Using this feature complies with industry practices. However, end user data may be required for implementing the feature. Exercise caution and obtain end user's consent when using this feature.This operation involves user privacy issues. Therefore, ensure that your login complies with the local laws.

  • To ensure VM security, the VM user must change the VM login password in the earliest time.

  • If you, as an administrator, have logged in to a VM, the VM service VM will not be allowed to log in to the VM before you log out.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, locate the row that contains the target VM, click VNC Login.

VM Power Management

Power management for a VM includes hibernating, starting or waking up, stopping, and restarting the VM. Stopping or restarting a VM can be performed in a safe or forcible manner. You are advised to use the safe manner. Forcibly stop or restart a VM only if the VM cannot be safely stopped or restarted due to an OS fault. Forcibly stopping or restarting a VM may damage the VM disks and cause data loss.

Starting or stopping a large number of VMs increases host workload. If you need to start or stop a large number of VMs on a host, perform the start or stop operations in batches to prevent VM services on the host from being adversely affected.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, locate the row that contains the target VM, and select the desired operation to perform.

Migrating a VM

VM migration refers to the process that a running VM is moved from one host to another host without service interruption.

If the service load is over light or heavy on the host where the VMs run or the host is faulty, migrate the VMs to ensure that they work properly.
  • If the service load is too heavy on the host or the host is faulty, migrate the VMs to another host to prevent service interruption due to a host exception.
  • If the service load is too light on multiple hosts, migrate the VMs to a small number of hosts to consolidate host resources and then power off the idle hosts. This improves resource utilization and reduces power consumption.

For details about the requirements for VM migration, see VM Migration Requirements.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, locate the row that contains the target VM, click More, and select Migrate.

Exporting the VM List

You can export all the VMs available in the system to an .xls file and save it on the local PC for information backup and statistics.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, click Export VM List.

Setting a Tag for a VM

Label a tag for a group of VMs with the same characteristics to facilitate VM search and management.

To delete the tag set for a VM, change the tag value of the VM to empty.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, locate the row that contains the target VM, click More, and select Set Tag.

Adding a VM to FusionManager for Management

Add a VM created in the in-use hypervisor to FusionManager so that FusionManager can manage this VM, for example, monitor alarms of the VM or assign the VM to a VDC. After the VM is added to FusionManager, the network information about the VM configured in the hypervisor still cannot be managed by FusionManager.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, click the VM, click More, and select Incorporate.

Assigning a VM

You can assign a VM to the VDC administrator of a specific VPC. However, VM assignment may cause unavailability of the VM NC. Therefore, after assigning a VM, inform the VDC administrator of allocating the VM to a user and modifying the VM network attributes in a timely manner.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, click the VM, click More, and select Allocate.

Changing the VM Upgrade Mode

Change the upgrade mode for the Tools on a VM in the FusionCompute hypervisor. Fore more details about Tools, see Tools Introduction.

You can modify the VM upgrade mode only when the VM is in the Running or Stopped state. The system can automatically upgrade a VM only when Tools is installed and running properly on the VM.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, locate the row that contains the target VM, click More, and select Modify Upgrade Mode.

Mounting a CD/DVD-ROM Drive to a VM

Mount the ISO image file or CD/DVD-ROM drive in the local PC to the VM, and then the VM can use the ISO image file or DC/DVD-ROM drive to install OS or software. For details, see Mounting a CD/DVD-ROM Drive or an ISO File.

Deleting a VM

If a VM is not needed any more, you can delete it to release VM resources. After a VM is deleted, all the data on the VM disks will also be deleted. Therefore, exercise caution when performing this operation.

FusionManager also supports a lazy delete function, which allows the system to retain the VM in the specified lazy delete duration after the VM deletion operation is triggered. In this duration, the VM is in the Reclaiming state, and you can cancel the VM deletion if needed.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, locate the row that contains the target VM, click More, and select Delete.

Recovering a VM in Reclaiming State

If the lazy delete function is enabled for a VM and the VM is in the Reclaiming, you can recover the VM to cancel the VM deletion.

Choose Resource > Resource Clusters, select a resource cluster, navigate to the VMs page, locate the row that contains the target VM, click More, and select Restore Reclaimed VM.
