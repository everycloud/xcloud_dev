Introduction to Resource Pool Management

Overview

Resources in the resource pool refer to the collection of resources that can be managed and used by FusionManager in the unit of resource zone.

A resource zone is a collection of independent layer-2 network resources available to administrators, and it is the largest resource unit in a cloud resource pool. Resource zones are isolated from each other at layer 2, and each resource zone has independent virtual local area network (VLAN) segments, network egress, and security devices. Commonly, a physical data center is a resource zone. In large-scale deployment scenarios, a physical data center can also be divided into multiple resource zones.

Resource Zone Creation and Management

A resource zone is the basic unit for cloud resource pool management. All resources, including physical devices and resource clusters in a hypervisor, can be managed by FusionManager only after being included in a resource zone.

After FusionManager interconnects with the virtualization environment, you need to create a resource zone and then perform zone-based resource configuration and management. Choose Resource > Resource Pool > Resource Zones, as shown in Figure 1.
