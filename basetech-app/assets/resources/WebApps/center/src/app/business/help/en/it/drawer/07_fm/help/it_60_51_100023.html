VLAN Pools

Virtual local area network (VLAN) pools are basic network settings required by FusionManager. The VLANs required by external networks, virtual private cloud (VPC) networks, and system services, such as virtual extensible LAN (VXLAN), must all be obtained from a VLAN pool configured in the system.

Adding a VLAN Pool

VMs communicate with access or aggregation switches using the host network interface card (NIC) over a DVS. Therefore, the VLAN resources must be both associated with a DVS and configured on the access and aggregation switches before being used by VMs.

Associating a VLAN Pool with a DVS

After a VLAN pool is associated with a DVS, you can select VLANs in the VLAN pool when creating a network.

  • A VLAN pool can associate with only one DVS in a resource cluster.
  • If the VLAN pool is to be used by an VDC that has multiple resource clusters, the VLAN pool must associate with a DVS in each of these resource clusters. For example, if the VDC uses two resource clusters, Cluster1 where DVS1 and DVS2 are deployed and Cluster2 where DVS3 is deployed, only the following VLAN pools can be used by the VDC:
    • VLAN pool with which DVS1 and DVS3 are associated
    • VLAN pool with which DVS2 and DVS3 are associated

Modifying a VLAN Pool

The attributes of a VLAN pool, such as the pool name and description, can be modified if needed.

Deleting a VLAN Pool

If a VLAN pool is not needed any more, you can delete it from the system. However, an in-use VLAN pool is not allowed to be deleted.

Adding a VXLAN Multicast IP Address Pool

A multicast IP address pool is required for implementing the VXLAN functions. The IP addresses in a multicast IP address pool are used to identify a VXLAN network. A VXLAN multicast IP address pool can contain a maximum of eight multiple IP address pools.

Modifying Attributes of a Multicast IP Address Pool

You can modify the attributes of a multicast IP address pool, including the pool name, start IP address, end IP address, and description.

Deleting a Multicast IP Address Pool

If a multicast IP address pool is not needed any more, you can delete it from the system. However, if the multicast IP addresses in the multicast IP address pool have been assigned, the multicast pool cannot be deleted.
