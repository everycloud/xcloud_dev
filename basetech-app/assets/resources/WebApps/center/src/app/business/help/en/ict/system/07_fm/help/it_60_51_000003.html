Scenarios

Log in to FusionManager and perform operations on it. After operations are complete, log out of FusionManager to ensure user security.

Prerequisites

  • You have installed the browser of the required version. If you use Internet Explorer or Mozilla Firefox, ensure that the browser you use has been configured properly.
  • You have obtained the floating IP address of the FusionManager node.
  • You have obtained the username, password, and user domain for logging in to FusionManager.
  • The required resolution has been set on the local PC.

Login Using Internet Explorer

The following uses Internet Explorer 9.0 as an example to describe how to log in to FusionManager.
  1. Open Internet Explorer.
  2. Enter the network address of FusionManager in the address box and press Enter.

    The network address format is http://FusionManager floating IP address.

    For example, enter http://192.168.40.2.

  3. If the security certificate is not installed, a security warning dialog box is displayed. Click Continue to this website (not recommended) to continue login.

    If the Firefox or Google Chrome browser is used, proceed as required:

    • If the browser is Firefox, click Add Exception.
    • If the browser is Google Chrome, click Advanced.

    The login certificate provided by FusionManager is a self-signed certificate. When you attempt to log in to the system, a message is displayed indicating that no security certificate is installed. If you expect to use your custom certificate to authenticate login, you can apply for a certificate from the third-party Certificate Authority (CA) and replace the default certificate with your own certificate by following steps described in Replacing the nginx Certificate of the FusionManager Web Server. After the certificate is replaced, the system will not display the security-certificate-absent messages any more.

  4. Enter Username and Password.
  5. Select a view type.
  6. Enter the verification code.
  7. Click Login.

    The FusionManager page is displayed.

    If the system displays a message indicating that the login password must be changed upon the first login or be reset, change the password as instructed.

    NOTE:

    On the FusionManager Web Client, do not press Backspace after a dialog box is displayed. Otherwise, the Web Client may fail to work properly. If this situation occurs, you can refresh the page.

Logout

In the upper right corner of the login page, click username and select Logout to log out of FusionManager.

FAQ

How Do I Log in to two FusionManager Web Clients on the Same PC?
To log in to two FusionManager Web Clients on your local PC, ensure that the following requirements are met:
  • Different usernames are used for logging in to two FusionManager Web Clients.
  • The FusionManager Web Clients are opened in different browsers.
