如何将裸设备映射的磁盘，从虚拟机解绑定后，再绑定给物理服务器

  • 操作场景
  • 操作步骤

操作场景

将SAN存储的LUN作为磁盘绑定给虚拟机后，解绑定该磁盘，再将LUN绑定给物理服务器。

操作步骤

    1. 解绑定磁盘。

      解绑定磁盘后，不能删除裸设备映射的磁盘和数据存储，否则LUN中的数据会丢失。

    2. 在物理服务器上，执行以下命令，查看该服务器能访问的存储设备。

      ll /dev/disk/by-id/

      回显如下类似信息，其中“scsi-360022a110005022b31fba15d00000036”为该存储设备的名称，“dm-0”为对应的文件系统的名称。

      ...
lrwxrwxrwx 1 root root 10 Dec  9 10:11 scsi-360022a110005022b31fba15d00000036 ->                                             ../../dm-0
...

    3. 执行以下命令，查看该存储设备的分区？

      file -s /dev/文件系统名称

      例如：file -s /dev/dm-0

      回显如下类似信息。
      /dev/dm-0: x86 boot sector; partition 1; ID=0x83, starthead 1, startsector 63, 838849977 sectors

    4. 执行以下命令，添加设备分区。

      kpartx -a /dev/文件系统名称

      例如：kpartx -a /dev/dm-0

    5. 执行以下命令，查看设备分区。

      kpartx -l /dev/文件系统名称

      例如：kpartx -l /dev/dm-0

      回显如下类似信息，其中“360022a1100092627004d04ed00000014p1”为设备分区号。
      360022a1100092627004d04ed00000014p1 : 0 838849977 /dev/dm-0 63

    6. 执行以下命令，将分区绑定到物理服务器。

      mount /dev/mapper/设备分区号 绑定目录

      例如：mount /dev/mapper/360022a1100092627004d04ed00000014p1 /mnt/sample
