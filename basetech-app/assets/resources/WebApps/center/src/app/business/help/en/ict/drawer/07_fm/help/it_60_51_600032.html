Enabling VPC Interconnection

Scenarios

Enable the virtual private cloud (VPC) interconnection function to allow different VPCs to communicate with each through a router. This function also allows tenants to use the access control list (ACL) and virtual private network (VPN) functions in the VPC.

Prerequisites

  • netMatrix is deployed.
  • You have obtained the galaxmanager password for logging in to FusionManager.

Procedure

    1. Use PuTTY to log in to FusionManager as user galaxmanager.

      • If FusionManager is deployed in active/standby mode, use the FusionManager floating IP address to log in to the active FusionManager node.
      • If only one FusionManager node is deployed, use the FusionManager management IP address to log in to the FusionManager node.

    2. Run the following command to disable user logout upon system timeout:

      TMOUT=0

    3. Run the following command to configure the VPC interconnection function:

      modConfig confRoutersCommunicate -vxlanRange startVxlan-endVxlan -isCommunicate (true|false)

      • startVxlan-endVxlan indicates the start virtual extensible local area network (VXLAN) ID and end VXLAN ID used by the shared network that assigns a gateway address for the router. This VXLAN range cannot be used for creating service networks.

      • true indicates that VPC interconnection is to be enabled. false indicates that VPC interconnection is to be disabled. This function is disabled by default.

      For example, run the following command to set 8000-8002 to the VXLAN range allowed for the shared network used for VPC interconnection and enable the VPC interconnection function:

      modConfig confRoutersCommunicate -vxlanRange 8001-8002 -isCommunicate true

      The following information is displayed:
      configure success

    4. If FusionManager is deployed in active/standby mode, run the following command to synchronize active node configuration to the standby node:

      haNotifySyncFiles /opt/goku/services/irm/rpool/config/network.properties

      The following information is displayed:
      execute command syncfile successfully.
