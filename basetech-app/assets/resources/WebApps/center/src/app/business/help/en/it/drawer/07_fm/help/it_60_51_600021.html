Setting Hot Swap Policies for a VM

Prerequisites

  • You have logged in to FusionManager.
  • The VM is in the Stopped state.

Scenarios

On FusionManager, set hot swap policies for a VM.

If FusionManager uses the FusionCompute hypervisor, the VM CPU hot swap policy can be:
  • CPU hot swap disabled: If you increase or decrease the number of CPUs on a running VM, the modification takes effect only after a VM restart.
  • CPU hot add enabled: If you increase the number of CPUs on a running VM, the modification takes effect without a VM restart. If you decrease the number of CPUs on a running VM, the modification takes effect only after a VM restart.
If FusionManager uses the VMware hypervisor, the VM CPU hot swap policy can be:
  • CPU hot swap disabled: The number of CPUs cannot be changed on a running VM.
  • CPU hot add enabled: If you increase the number of CPUs on a running VM, the modification takes effect only after a VM restart.
  • CPU hot swap enabled: If you increase or decrease the number of CPUs on a running VM, the modification takes effect only after a VM restart.
If FusionManager uses the VMware hypervisor, the VM memory hot swap policy can be:
  • Memory hot add disabled: The memory size cannot be changed on a running VM.
  • Memory hot add enabled: If you increase the memory size on a running VM, the modification takes effect only after a VM restart.

  • A Tools has been installed on the VM.

Procedure

  1. Click the VM name.

    The Summary page is displayed.

  2. On the Options page, click Hot Swap.

    The Hot Swap page is displayed.

  3. Set hot swap policies and click OK.

    The hot swap policies are configured.
