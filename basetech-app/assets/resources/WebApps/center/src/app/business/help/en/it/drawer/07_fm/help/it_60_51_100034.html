Scripts

Scripts are included in application instance VMs to control software and process running in the VMs.

After a locally-stored script file is uploaded to FusionManager, it can be used for:
  • Automatic script execution: When you provision application instances using an application template in which the VM template has specific script file configured, the system automatically executes the script on the VM that is to run the application instance.
  • Script distribution: After you select a script file and specify the target VMs to execute the script file, the system automatically distributes the script to the specified VMs and executes the script on them.
Note the following points for script execution:
  • A return value is displayed after script execution is complete. Value 0 indicates that the operation is successful. Other values indicate that the operation fails.
  • If you need to print operation logs during script execution, do not print the logs on the operation console. This is to prevent excessive amount of information from being generated on the console. If such a case occurs, the console may fail to return the correct operation result to the system. Instead, you need to configure the function that enables the logs to be redirected to the log file for print.

For more details, see Automatic Software Deployment.

Distributing Scripts

After you select a script file and specify the target VMs to execute the script file, the system automatically executes the script on the specified VMs.

Locate the row that contains the target script file and click Batch Install.

Modifying Script File Information

A script file that has been uploaded to the system cannot be uploaded again, and the file name is not allowed to be changed. However, other file attributes can be modified as needed.

Locate the row that contains the target script file, click More, and select Modify.

Restoring a Script File

If a script file is only deleted from the file system of the storage space but is not deleted from the FusionManager Web Client, the script file is displayed in an abnormal state on FusionManager. You can restore the script file so that the script file can be reloaded to the FusionManager system and changes to the normal state on the Web Client.

Locate the row that contains the target script file and click Restore.

Deleting a Script File

If a script file is not needed any more, you can delete it from the system.

Locate the row that contains the target script file, click More, and select Delete.

FAQ

Why is the Browser button not available to click to select the file to be uploaded?

This issue may be caused because the JRE version you are using requires you to add the site you are visiting to the reliable site list. To do so, open Java Control Panel, click the Security tab, and add the FusionManager site to the exceptions in the format of Protocol+IP address+Port.

Possible causes for the file uploading failure
  • Network disconnection

    If the network communication between the PC and FusionManager is disconnected, file uploading to FusionManager will fail. Try uploading the file again after the network is connected.

  • Incorrect password

    If you enter an incorrect password for uploading a file to FusionManager, file uploading will fail. Try uploading the file again after entering a correct password.

  • Insufficient storage space
    If the storage space in the FusionManager system is insufficient, uploading a file to the system will fail. You can use the following methods to expand system storage space:
    • Delete unnecessary software packages, or scripts to release storage space.
    • Change a storage device providing larger storage space. For details, see Expanding Space for Storing Software Packages and Scripts.
