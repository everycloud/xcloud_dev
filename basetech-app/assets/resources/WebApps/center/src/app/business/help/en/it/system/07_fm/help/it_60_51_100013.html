A hypervisor is the software that virtualizes computing, network, and storage resources. For example, FusionCompute is a Huawei hypervisor. The virtual resources used and managed by FusionManager are virtualized by the hypervisor.

The resources virtualized by a FusionCompute hypervisor are managed by its Virtual Resource Management (VRM) node, the resources virtualized by a VMware hypervisor are managed by its vCenter Server.

FusionManager supports a maximum of 255 hypervisors.

One hypervisor can be added to only one FusionManager.

Adding a Hypervisor

After installing and configuring FusionManager, add a hypervisor to FusionManager, enabling FusionManager to manage and use virtual resources on the hypervisor. Before adding the VMware hypervisor, install the VMware SDK. For details, see Appendix > FAQ > Installing the VMware SDK Toolkit in the FusionManager V100R005C00 Software Installation Guide.
NOTE:
The default user account gmsysman is used to add FusionCompute and the Virtual Service Appliance Management (VSAM) VM. The default password of user gmsysman is GMEnginE@123.

To add a hypervisor, choose Resource > Hypervisor and click Add.

Setting MAC Addresses

If you need to add multiple FusionCompute hypervisors to FusionManager, plan VM MAC addresses properly before installing FusionCompute hypervisors. If you do not plan MAC address segments for the hypervisors, the system automatically assigns MAC addresses for VMs during VM creation. However, this may cause the same MAC address to be assigned to VMs in different hypervisors, resulting in VM Mac address conflicts.

  • MAC address segment: automatically or manually assigned to VMs in the hypervisor. A maximum of five MAC address segments can be added.
  • Independent MAC address: manually assigned to VMs in the hypervisor.

FusionCompute provides 10,000 MAC addresses ranging from 28:6E:D4:88:B2:A1 to 28:6E:D4:8A:39:40. On the segment, MAC addresses ranging from 28:6E:D4:88:B2:A1 to 28:6E:D4:88:C6:28 are reserved and cannot be added. The other MAC addresses can be used by service VMs.

A change or deletion of the MAC address segment or independent MAC address takes effect for only subsequent MAC address assignments.

Choose Resource > Hypervisor, locate the row that contains the target hypervisor, click More, and select Set MAC Address.

Updating a Hypervisor

If you do not select Update data immediately after data saving after adding or modifying a hypervisor but need to immediately update data, update the hypervisor. FusionManager can use and manage resources on the hypervisor only after the hypervisor is updated.

Choose Resource > Hypervisor, locate the row that contains the target hypervisor, click More, and select Update.

Modifying a Hypervisor

If the hypervisor information is inconsistent with actual conditions, modify attributes of the hypervisor.
NOTICE:
After the hypervisor information is modified, the new data will overwrite the original data. If the new information is incorrect, the hypervisor may become unavailable. Therefore, exercise caution when you perform this operation.

On FusionManager, choose Resource > Hypervisor. On the Hypervisor page that is displayed, click More, and select Modify.

Changing the Management IP Address of the VSAM Node

When a hypervisor with the VSAM node deployed requires network adjustment, the management IP address of the VSAM node can be changed. All new services are interrupted during the management IP address change process, for example, applying for or releasing a software router or software load balancer. All existing services are not interrupted.

Choose Resource > Hypervisor, locate the row that contains the target hypervisor, click More, and select Change VSAM Management IP Address.

Deleting a Hypervisor

If you do not need a hypervisor or need to add the hypervisor again, delete it.

Choose Resource > Hypervisor, locate the row that contains the target hypervisor, click More, and select Delete.
