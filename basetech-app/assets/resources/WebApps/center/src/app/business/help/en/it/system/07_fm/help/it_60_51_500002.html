Setting Alarm Email Notification

In addition to the FusionManager Web Client, you can obtain alarm information from emails.

Before configuring the alarm notification by email function, you need to navigate to System > Email Server and complete the email server configuration.

Choose Monitoring > Alarm Settings > Alarm Email Notification.

Setting Alarm Thresholds

An alarm is generated if the alarm threshold is hit. You can set the alarm threshold based on service requirements. If the alarm threshold is not configured, the default value is used.

If the disk usage threshold for triggering a major alarm is 95%, that for triggering a minor alarm is 85%, and the offset is 5%,

then,
  • When the real disk usage is greater than or equal to 95%, a major alarm is generated.
  • When the CPU usage exceeds 85% but is lower than 95%, a minor alarm is generated.
  • When the real disk usage is lower than 90%, that is, the threshold for the major alarm (95%) minus the offset (5%), the system automatically clears the major alarm.
  • When the real disk usage is lower than 80%, that is, the alarm threshold for the minor alarm (85%) minus the offset (5%), the system automatically clears the minor alarm.

Choose Monitoring > Alarm Settings > Alarm Thresholds.

Setting Alarm Masking

You can mask unimportant alarms based on service requirements. The masked alarms are not displayed in the alarm list.

Choose Monitoring > Alarm Settings > Alarm Mask.

Configuring Accessing of Alarms Generated for Third-Party Components

To enable alarms generated for third-party components to be accessed and managed by FusionManager in a unified manner, you can add third-party component information to FusionManager.

NOTE:
  • Do not set the interconnection parameters as follows because risks may arise:
    • The SNMP Version is SNMPv2c.
    • The Authentication Protocol is MD5.
    • The Key Type is DES56.
  • Set the interconnection parameters as follows, instead:
    • The SNMP Version is SNMPv3.
    • The Authentication Protocol is SHA1.
    • The Key Type is AES128, AES192, or AES256.

Choose Monitoring > Alarm Settings > Third-Party Components.
