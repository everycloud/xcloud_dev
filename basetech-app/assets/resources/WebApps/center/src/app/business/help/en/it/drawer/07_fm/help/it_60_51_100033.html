Software Packages

Software packages are installed on application VMs in an application template to deploy application instances. A software package contains software installation files, commands, and scripts.

After a locally-stored software package is uploaded to FusionManager, it can be used for:
  • Automatic software installation: When you provision application instances using an application template in which the VM template has specific software configured, the system automatically installs the software on the VM that is to run the application instance.
  • Software distribution: After you select a software package and specify the target VMs to install the package, the system automatically installs the software on the specified VMs.
Note the following points for software package installation:
  • The software must support silent installation, which allows the software to be installed without human interventions or restart during installation.
  • A return value is displayed after software installation is complete. Value 0 indicates that the operation is successful. Other values indicate that the operation fails.
  • If you need to print operation logs during software installation, do not print the logs on the operation console. This is to prevent excessive amount of information from being generated on the console. If such a case occurs, the console may fail to return the correct operation result to the system. Instead, you need to configure the function that enables the logs to be redirected to the log file for print.
  • The software can automatically start with the OS after being successfully installed.

For more details, see Automatic Software Deployment.

Distributing Software

After you select a software package and the target VMs, FusionManager can automatically install the software on the specified VMs.

Locate the row that contains the target software package and click Batch Install.

Modifying Software Package Information

A software package that has been uploaded to the system cannot be uploaded again, and the software package name is not allowed to be changed. However, other software package attributes can be modified as needed.

Locate the row that contains the target software package, click More, and select Modify.

Restoring a Software Package

If a software package is only deleted from the file system of the storage space but is not deleted from the FusionManager Web Client, the software package is displayed in an abnormal state on FusionManager. You can restore the package so that the package can be reloaded to the FusionManager system and changes to the normal state on the Web Client.

Locate the row that contains the target software package and click Restore.

Deleting a Software Package

If an software package is not needed any more, you can delete it from the system.

Locate the row that contains the target software package, click More, and select Delete.

FAQ

Why is the Browser button not available to click to select the file to be uploaded?

This issue may be caused because the JRE version you are using requires you to add the site you are visiting to the reliable site list. To do so, open Java Control Panel, click the Security tab, and add the FusionManager site to the exceptions in the format of Protocol+IP address+Port.

Possible causes for the file uploading failure
  • Network disconnection

    If the network communication between the PC and FusionManager is disconnected, file uploading to FusionManager will fail. Try uploading the file again after the network is connected.

  • Incorrect password

    If you enter an incorrect password for uploading a file to FusionManager, file uploading will fail. Try uploading the file again after entering a correct password.

  • Insufficient storage space
    If the storage space in the FusionManager system is insufficient, uploading a file to the system will fail. You can use the following methods to expand system storage space:
    • Delete unnecessary software packages, or scripts to release storage space.
    • Change a storage device providing larger storage space. For details, see Expanding Space for Storing Software Packages and Scripts.
