Scenarios

To use the automatic software deployment function, install the Puppet Client on the VM.

FusionManager provides the automatic software deployment function to allow users to rapidly deploy software packages and scripts on VMs in batches.
  • Automatic software installation: When you provision application instances using an application template in which the VM template has specific software configured, the system automatically installs the software on the VM that is to run the application instance.
  • Software distribution: After you select a software package and specify the target VMs to install the package, the system automatically installs the software on the specified VMs.
  • Automatic script execution: When you provision application instances using an application template in which the VM template has specific script file configured, the system automatically executes the script on the VM that is to run the application instance.
  • Script distribution: After you select a script file and specify the target VMs to execute the script file, the system automatically distributes the script to the specified VMs and executes the script on them.

Procedure

    Place the software to be installed into an ISO file.

    1. Place all the software to be installed into an ISO file, so that the installation file can be copied to the VM.

      You can use any ISO image creation tool, such as UltraISO and ISOMaker, to create an ISO file for the desired software. For more information, visit Forum.

      In the following steps, the ISO file is named puppet.iso as an example, which contains the puppet folder.

    Copy the installation files contained in the ISO file to the VM by attaching the CD/DVD-ROM drive.

    1. Log in to the VM using Virtual Network Computing (VNC) and attach the ISO file puppet.iso to the VM.

      For details about how to attach the CD/DVD-ROM drive, see Mounting a CD/DVD-ROM Drive or an ISO File.

    2. Log in to the VM OS as user root.

      If you are using a Graphical User Interface (GUI) system, right-click on the VM OS desktop and choose Open in Terminal from the shortcut menu.

      The CLI window is displayed.

    3. Run the following command to disable user logout upon system timeout:

      TMOUT=0

    1. Run the following command to create the xvdd directory:

      mkdir xvdd

    2. Run the following command to attach the CD/DVD-ROM drive:

      mount Mounting directory xvdd

      For example, mount /dev/sr0 xvdd

    1. Run the following command to copy the puppet folder contained in the ISO file to the /home directory of the VM:

      cp -r xvdd/puppet/ /home

    Install the Puppet Client software.

    1. Use the installation file that has been copied to the /home directory to install the ruby, ruby-devel, and rubygems software.
    2. Use the installation file that has been copied to the /home directory to install the dependent software package and Puppet Client software.

      For details, see the product documents on the Puppet official website.

    Configure the Puppet Client software.

    1. Log in to the VM OS as user root.

      If you are using a Graphical User Interface (GUI) system, right-click on the VM OS desktop and choose Open in Terminal from the shortcut menu.

      The CLI window is displayed.

    2. Determine whether the in-use OS is a Red Hat Linux enterprise 5.5 64-bit, CentOS 5.6 32-bit, or CentOS 5.6 64-bit OS.

      • If yes, go to 12.
      • If no, go to 17.

    3. Run the following command to check whether the command output contains disabled or permissive:

      sestatus -v

      The command output containing disabled is as follows:
      SELinux status:                 disabled
      The command output containing permissive is as follows:
      SELinux status:                 enabled
SELinuxfs mount:                /selinux
Current mode:                   permissive
Mode from config file:          enforcing
      • If yes, go to 17.
      • If no, go to 13.

    4. run the following command to open the /etc/selinux/config file using the visual interface (vi) editor:

      vi /etc/selinux/config

    5. Press i to enter editing mode and set SELINUX to disabled.
    6. Enter :wq and press Enter to save the configuration and exit the vi editor.
    7. Run the following command to restart the OS:

      reboot

    8. After the OS is restarted, reattach the ISO file puppetclient_linux.iso to the xvdd directory.
    9. Run the following command to access the configuration file directory:

      cd xvdd/puppetclient_linux

      NOTE:

      If the Red Hat Linux Enterprise OS is used, check whether the files or folders in the directory contains only lowercase letters. If yes, restart the VM, attach the CD/DVD-ROM drive again, and then configure the Puppet Client.

    10. Switch to the directory of the in-use OS.
    11. Run the following command to execute the configuration script:

      sh configpuppet.sh

      The software is successfully configured if the following information is displayed:
      GalaxManager Agent has been installed successfully!
      NOTE:

      After the software is configured successfully, the system automatically generates a puppet account. The puppet account does not require any password, but this account cannot be used to log in to the system and therefore does not have any security risks.
