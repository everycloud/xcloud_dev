Resource Pool Configuration

Overview

FusionManager can manage hardware devices, virtualized resources, and applications in a cloud system. After the FusionManager system is set up, multiple hypervisors can be added to the system to form a unified cloud resource pool. Then, FusionManager can manage the computing and storage resources in these hypervisors in a unified manner, and also supports configuration of network resources for the cloud resource pool. The guideline for configuring a cloud resource pool is as follows:
  1. Create a resource zone and add hypervisors and required physical devices to FusionManager.
  2. Associate hypervisor resource clusters with the FusionManager resource zone to allow FusionManager to manage computing and storage resources from the resource clusters in the associated resource zone.
  3. Create availability zones to include resource clusters with the same attributes into the same available zone.
  4. Configure network resources for services using different types of networks. To use advanced network functions, see Feature Configuration to configure advanced network functions.
    FusionManager supports three types of networks in a virtual private cloud (VPC): direct network, internal network, and routed network.
    • Direct network: A direct network itself contains no network resources, but it has direct access to external networks. Therefore, VMs created on a direct network actually uses the IP addresses provided by the connected external network, which can be the existing network of the customer or the Internet.
      NOTE:

      When creating VMs in a direct network, you are advised to configure security groups to isolate the VMs and implement VM access security control.

    • Internal network: An internal network is isolated from other networks and uses exclusive network resources. Therefore, you can deploy services that have high security requirements, for example, database server services, on an internal network to ensure high data security.
    • Routed network: A routed network provides flexible communication functions and supports various services. Routed networks in a VPC can communicate with each other or the Internet. A routed network also provides elastic IP address, access control list (ACL), destination network address translation (DNAT), and virtual private network (VPN) services. Before creating a routed network in a VPC, ensure that you have applied for a virtual router for the VPC.
    A routed network can use two types of firewalls.
    • Hardware firewall: provided by a virtual firewall of physical firewall.
    • Software firewall: provided by a system service VMs in the FusionManager.

Create a resource zone.

A resource zone is a collection of independent layer-2 network resources available to administrators, and it is the largest resource unit in a cloud resource pool. Resource zones are isolated from each other at layer 2, and each resource zone has independent virtual local area network (VLAN) segments, network egress, and security devices. Commonly, a physical data center is a resource zone. In large-scale deployment scenarios, a physical data center can also be divided into multiple resource zones.

A resource zone is the basic unit for cloud resource pool management. All resources, including physical devices and resource clusters in a hypervisor, can be managed by FusionManager only after being included in a resource zone.

Resource zones can use the following three networking modes, and all of these networking modes support the software VSA functioning as a gateway:
  • Gateway is set on the virtual firewall
  • Gateway is set on the switch and manages the firewall
  • Physical gateway is not automatically configured
For details about these networking modes, see Introduction to Physical Networking Modes for Resource Zones.

If you do not plan MAC address segments for the hypervisors, the system automatically assigns MAC addresses for VMs during VM creation. Associate a resource cluster with a resource zone.

Associate the resource clusters in a hypervisor to a resource zone to provide computing resources for service provisioning.

Choose Resource > Resource Zones, select a resource zone, and navigate to the Computing Resource Pools page.

Create an availability zone.
An availability zone is a logical zone of physical resources (computing, storage, and network) available to users. The hardware devices in an availability zone are connected at layer 2. Commonly, resource clusters in a resource zone are grouped in the same availability zone based on the following rules:
  • All resource clusters in an availability zone come from the same hypervisor.
  • All resource clusters in an availability zone use the same storage resource.
  • All resource clusters in an availability zone use the same distributed virtual switches (DVSs).

Choose Resource > Resource Zones, select a resource zone, and navigate to the Availability Zones page.

Add a distributed virtual switch (DVS).

Functioning the same as a physical switch does, a distributed virtual switch (DVS) supports data switching among VMs and between VMs and external networks.

If the VMware Virtual Software Switch (VSS) is used, you need to convert the VSS to a FusionManager-supported DVS first.

Choose Resource > Resource Zones, select a resource zone, and choose Network Resource Pools > DVS.

Add a virtual local area network (VLAN) pool.

Virtual local area network (VLAN) pools are basic network settings required by FusionManager. The VLANs required by external networks, virtual private cloud (VPC) networks, and system services, such as virtual extensible LAN (VXLAN), must all be obtained from a VLAN pool configured in the system.

A VLAN pool can be used to create service networks only after it is associated to a DVS.

Choose Resource > Resource Zones, select a resource zone, and choose Network Resource Pools > VLAN Pools.

Add an external network.

Customers' live networks are external networks of virtual private clouds (VPCs) configured in FusionManager. External networks added to FusionManager can be used to provision VMs and create direct networks in a VPC.The created external networks that meet the following requirements can connect to a software router to provide public IP addresses for the elastic IP address service: the connection mode is subnet (common VLAN), the network is connected to the Internet, the subnet IP address assignment mode is IPv4 static injection, and the network does not contain an IPv6 subnet.

Choose Resource > Resource Zones, select a resource zone, and choose Network Resource Pools > External Networks.

Add a Virtual Service Appliance (VSA) management network.

A Virtual Service Appliance (VSA) management network, which provides management network resources for system service VMs, is a basic network configuration required by FusionManager.

Choose Resource > Resource Zones, select a resource zone, and choose Network Resource Pools > VSA Networks.

Import a VSA VM template to the system.

Import a VSA VM template to the system so that FusionManager can use this template to automatically create system service VMs and provide virtual firewall functions. For details, see Importing a VSA VM Template.

 Choose Resource > Templates and Specs > VM Templates.

Add a physical firewall device to the system.

Virtual firewalls are logical firewalls virtualized on a physical firewall. Each virtual firewall functions as an independent firewall device to monitor and filter packets over its parent physical firewall device. FusionManager implements firewall functions in the system using virtual firewalls.

If the physical firewall device has been added to the system, you do not need to add it once again.

If the physical firewall device is already added to the system before a virtual firewall is created on it, you need to discover the virtual firewall again on FusionManager after creating the virtual firewall.

 Choose Resource > Devices > Network Devices > Firewalls.

Create virtual firewalls.

Introduction to Physical Networking Modes for Resource Zones

FusionManager supports the following networking modes for a resource zone so that the network devices in the resource zone can be automatically configured and managed based on the selected networking modes:

  1. Gateway is set on the virtual firewall: In this mode, the gateway to the service network is on the virtual firewall created on the physical firewall. The virtual firewall implements subnet layer 3 forwarding and all the advanced network functions, as shown in Figure 2.
    Figure 2 Gateway is set on the virtual firewall
  2. Gateway is set on the switch and manages the firewall: In this mode, the gateway to the service network is on the aggregation switch. The aggregation switch implements subnet layer 3 forwarding and all the advanced network functions, as shown in Figure 3.
    Figure 3 Gateway is set on the switch and manages the firewall

  3. Physical gateway is not automatically configured: In this mode, the system does not automatically manage firewalls or switches. Select this mode only when the network type is set to internal network or external network.
