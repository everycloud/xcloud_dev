ACL

Overview

The access control list (ACL) is a command list that controls the transmission of data packets traversing network devices. It is also used to control incoming and outgoing network traffic, preventing illegitimate use of and access to network resources and improving network access security.

Principles

With the ACL, the system can judge, classify, and filter data streams over routers. ACLs rules can be specified based on the source address, destination address, protocol type, and port number. After the ACL rules apply to router interfaces, the router checks the header information in a data packet against the rules to determine whether to forward or reject the data packet.

For example, as shown in Figure 1, the system controls the traffic over VM 2 based on the following two ACL rules:

  1. Sends the data packet whose source address is 200.8.1.0/24 and the destination address is 10.145.12.0/24 to VM 2.
  2. Discards the data packet whose source address is 10.145.11.0/24 and the destination address is 10.145.12.0/24.
Figure 1 ACL access control

In the FusionManager, intra-zone rules control the data traffic between VMs in routed networks within a VPC. Inter-zone rules control the data traffic between VMs in and outside of a VPC. The rules are implemented based on elastic IP addresses.

Add an external network.

An external network can connect to the software router and provide public IP addresses for VMs associated with the routed network in the VPC. The external networks created on FusionManager must meet the following requirements: its connection mode is subnet (common VLAN), it is connected to the Internet, its subnet IP address assignment mode is static injection, and it does not contain an IPv6 subnet.

 Choose Resource > Resource Zones, select a resource zone, and choose Network Resource Pools > External Networks.

Add a VSA management network.
The VSA management network provides the management plane for system service VMs. It must meet the following requirements:
  • The VSA management network connects to the system management plane.
  • The VSA management network VLAN has been added to the DVS created on the FusionCompute hypervisor, but the VLAN is not contained in the FusionManager VLAN pool.
 Choose Resource > Resource Zones, select a resource zone, and choose Network Resource Pools > VSA Networks.

Import a VSA VM template to the system.
