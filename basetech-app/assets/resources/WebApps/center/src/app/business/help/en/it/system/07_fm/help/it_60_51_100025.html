Public IP addresses are used for communication between VMs and public networks. The following services require public IP address pools configured in the resource zone:
  • Elastic IP address (using hardware firewalls)
  • Network address translation (using hardware firewalls)
  • Access control list
  • Virtual private network

Adding a Public IP Address Pool

Add a public IP address pool to provide public IP addresses for VMs.

Choose Resource > Resource Zones, select a resource zone, choose Network Resource Pools > Public IP Address Pool, and click Add Public IP Address Pool.

The added public IP address pool must meet the following conditions:
  • No IP address in the pool is duplicate with any public IP addresses that are manually configured on the firewall.
  • A downstream route from the public IP address pool to the aggregation switch has been configured on the core switch.

Configuring NAT Address Multiplexing for Public IP Addresses

In the Network Address Translation (NAT) service, a public IP address cannot be used concurrently on destination network address translation (DNAT) and source network address translation (SNAT) by default. To use a public IP address in both DNAT and SNAT, enable the NAT address multiplexing function. For details, see Configuring Public IP Addresses to Be Multiplexed in Both DNAT and SNAT.

If public IP addresses are applied to both DNAT and SNAT after the address multiplexing function is enabled, the usage statistics are counted in both the DNAT and SNAT services. Therefore, the number of used public IP addresses is greater than the number of allocated public IP addresses.

Deleting a Public IP Address Pool

If a public IP address pool is not needed any more, you can delete it from the system. However, if a public IP address in the pool is in use, the pool is not allowed to be deleted.

Choose Resource > Resource Zones, select a resource zone, choose Network Resource Pools > Public IP Address Pool, and click Delete.
