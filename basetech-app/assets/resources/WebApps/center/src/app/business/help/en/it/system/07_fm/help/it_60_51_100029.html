System service VMs are automatically created by FusionManager to provide system services, such as:
  • Virtual Service Appliance (VSA) VM
  • Proxy VM (PVM) VM

VSA VM

The VSA VM provides functions including software firewall, software load balancer, and internal Dynamic Host Configuration Protocol (DHCP) server. A VSA VM can function as the following types of VMs based on the required functions:
  • DHCP VM: provides internal DHCP server services to assign IP addresses to VMs without using a physical DHCP server.
  • Virtual firewall (VFW): provides various network services, such as elastic IP addresses, network address translation (NAT), and virtual extensible LAN (VXLAN), without using a physical firewall.
  • Software load balancer (SLB): provides network load balancing services without using a physical load balancer.

PVM VM

A PVM VM provides the following functions.
  • Automatic software deployment
    • Automatic software installation: When you provision application instances using an application template in which the VM template has specific software configured, the system automatically installs the software on the VM that is to run the application instance.
    • Software distribution: After you select a software package and specify the target VMs to install the package, the system automatically installs the software on the specified VMs.
    • Automatic script execution: When you provision application instances using an application template in which the VM template has specific script file configured, the system automatically executes the script on the VM that is to run the application instance.
    • Script distribution: After you select a script file and specify the target VMs to execute the script file, the system automatically distributes the script to the specified VMs and executes the script on them.
  • Hyperic HQ-based VM process monitoring

Obtaining System Service VMs

Prerequisites

System service VMs are essentially VMs, and therefore they require the same conditions for creating a common service VM:
  • You have added the VSA management network in the network resource pool of the current resource zone.
  • You have created a VSA or PVM VM template to the system.For details, see Importing a VSA VM Template or Creating a PVM Template.

How to Obtain

System service VMs are not manually created. Instead, the system automatically creates system service VMs when a user selects the corresponding system services. The automatically created system service VMs are displayed in the system service VM list, service request page, and VM list under the current resource zone. For example:
  • When a user applies for a VLB, the system automatically creates a VFM VM.
  • When a user uses the automatic software deployment service, the system automatically creates a PVM VM.

System Service VM Power Management

Power management for a system service VM includes starting or restarting the VM.

Choose Resource > Resource Zones, select a resource zone, navigate to the System Service VMs page, select the target system service VM, and select the desired operation to perform.

Logging In to a PVM VM Using VNC

VNC is a remote terminal access tool. You can use VNC to remotely connect to a VM, view the VM operating page, and perform management and maintenance operations on it.

Choose Resource > Resource Zones, select a resource zone, navigate to the System Service VMs page, locate the row that contains the target system service VM, and click VNC Login.

Restoring System Service VMs

Restore a system service VM if it is faulty. The restoration process is to delete the faulty system service VM and then use the Virtual Service Appliance (VSA) or proxy VM (PVM) template to create another system service VM.

If a PVM that provides the Hyperic HQ-based process monitoring function is restored, all the VMs that have the monitoring function configured in the system need to be reconfigured based on the new IP address of the restored PVM.

Choose Resource > Resource Zones, select a resource zone, navigate to the System Service VMs page, locate the row that contains the target system service VM, and click Restore.
