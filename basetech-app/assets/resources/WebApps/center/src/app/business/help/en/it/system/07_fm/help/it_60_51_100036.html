VPC Specifications

  • Modifying VPC Specifications

Virtual private cloud (VPC) specifications are used to define the VPC configuration requirements, such as the number of allowed networks, and number of available elastic IP addresses, in a virtual data center (VDC).

VPC specifications define the default configurations of a new unshared VPC.

The VPC specifications do not apply to shared VPCs, which are created using the default maximum VPC specifications of the system.

Modifying VPC Specifications

If the default VPC specifications do not meet service requirements, you can modify the specifications as needed. The VPC specification modification will not change the attributes of the existing VPCs in the VDC.

Choose VDC > VPC Specs, locate the row that contains the target VPC specification set, and click Modify.
