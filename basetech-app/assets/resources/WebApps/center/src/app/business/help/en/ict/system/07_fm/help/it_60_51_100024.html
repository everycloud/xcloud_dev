Customers' live networks are external networks of virtual private clouds (VPCs) configured in FusionManager. External networks added to FusionManager can be used to provision VMs and create direct networks in a VPC.The external networks can also be used as elastic IP address pools to provide elastic IP address services.

Creating an External Network

Before creating an external network, ensure that you have configured the required VLANs or subnets on the switch and have added the VLANs to a VLAN pool.

NOTICE:

By default, an external network created on FusionManager is sharable to all VDCs using the same OpenStack instance as the external network. VDC users can configure security groups to isolate the VMs in the external network. However, security group isolation may still pose potential security risks. Therefore, exercise caution when you use external networks.

Choose Resource > OpenStack Instances, select an OpenStack instance, navigate to the External Networks page, and click Create.

Modifying an External Network

Modify the attributes of an external network.

Choose Resource > OpenStack Instances, select an OpenStack instance, navigate to the External Networks page, locate the row that contains the external network to be modified, and click Modify.

Deleting an External Network

If an external network is not needed any more, you can delete it from the system. However, an external network that is in use by a virtual data center (VDC) is not allowed to be deleted.

Choose Resource > OpenStack Instances, select an OpenStack instance, navigate to the External Networks page, locate the row that contains the target external network, and click Delete.

Creating a Subnet

Configure a subnet for a newly created external network.

Choose Resource > OpenStack Instances, select an OpenStack instance, navigate to the External Networks page, locate the row that contains the target external network, and click Create Subnet.

Modifying a Subnet

Modify the subnet information about an external network.

Locate the row that contains the target external network, click , locate the row that contains the subnet to be modified, and click Modify.

Deleting a Subnet

Delete the subnet of an external network.
