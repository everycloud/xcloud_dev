配置公网IP地址复用为DNAT和SNAT

操作场景

公网IP地址是系统中的虚拟机与公网通信所需的IP地址资源。

当公网IP地址池用于NAT业务时，系统默认一个公网IP地址不能同时用于DNAT和SNAT。该任务用于指导管理员开启或关闭复用功能，即配置是否允许一个公网IP地址同时用于DNAT和SNAT。

前提条件

  • FusionManager节点主备部署时，需获取浮动IP地址；FusionManager单节点部署时，获取管理IP地址。
  • 已获取FusionManager节点的“galaxmanager”用户密码。

操作步骤

    1. 使用“PuTTY”，以“galaxmanager”用户登录FusionManager节点。

      • FusionManager节点主备部署时，通过浮动IP地址登录主节点。
      • FusionManager单节点部署时，以管理IP地址登录。

    2. 执行以下命令，防止“PuTTY”超时退出。

      TMOUT=0

    3. 执行以下命令，配置公网IP地址复用功能。

      modConfig confSharePublicIP Share_PublicIP 参数选项

      参数选项说明如表1所示。

      表1 选项说明

      选项

      说明

      true

      开启复用功能，允许一个公网IP地址同时用于DNAT和SNAT。

      false

      关闭复用功能，不允许一个公网IP地址同时用于DNAT和SNAT。关闭后，后续申请的公网IP地址将不能再同时用于DNAT和SNAT，但之前复用功能开启时，复用的公网IP地址的使用不受影响。

      例如，执行以下命令，开启复用功能。

      modConfig confSharePublicIP Share_PublicIP true

      回显信息如下，表示配置成功。

      configure success

    4. FusionManager节点主备部署时，执行以下命令，同步主备配置。

      haNotifySyncFiles /opt/goku/services/irm/rpool/config/network.properties

      回显如下信息，表示配置成功。

      execute command syncfile successfully.
