Configuration Overview

Overview

Using physical network devices added to the system, such as switches and firewalls, FusionManager can provide tenants with virtual advanced network services, for example, hardware-based router and load balancer (LB) services.

FusionManager can also use system service VMs, such as Virtual Service Appliance (VSA) VMs and Proxy VM (PVMs), to provide network services for tenants. The PVM VM provides automatic software deployment function for VMs in the tenant network. If the FusionCompute hypervisor is used, FusionManager can create a VSA VM to provide software-based router, Dynamic Host Configuration Protocol (DHCP), and virtual LB (VLB) services for tenants, as shown in Figure 1.

Figure 1 System service VMs
  • VSAM: The Virtual Service Appliance Management (VSAM) node is created in the FusionCompute hypervisor to manage VSA VMs in the FusionManager system.
  • Management network: the network on which the FusionManager management node and the VSAM node are operating
  • VSA management network: The VSA management network is connected to the FusionManager management network to provide VSA VMs with IP addresses for the VSA VMs to communicate with the VSAM node.
  • External network: The external network provides network resources for service VMs to connect to the external network using the software-based virtual router.

Router

The router provides both routing functions and network gateways for tenants. Therefore, the routing networks in the same virtual private cloud (VPC) can communicate with each using the router. The VMs in these routing networks can use the elastic IP address and network address translating (NAT) functions provided by the router to communicate with external networks.

FusionManager supports the following types of routers:
  • Hardware router: a firewall instance created on a physical firewall to provide router functions
  • Software router: a VSA VM created on FusionManager to provide routing functions

DHCP Server

FusionManager can provide an internal DHCP server to provide DHCP services for tenants. The DHCP server functions are also implemented by a VSA VM in the system. When you create a network for the first time with its IP address assignment mode set to internal DHCP, the system automatically creates a VSA VM that provides the DHCP service. Then, the system can use this VSA VM to assign IP addresses for subsequently created networks if their IP address assignment modes are also set to internal DHCP.

VLB

A virtual load balancer provides load balancing services for VMs. A tenant can create a VLB, bind VMs that provide the same services to the VLB, and configure VLB functions, including the sticky session, health check, and allocation policies, to balance user access to VMs.

FusionManager supports hardware load balancers (LBs) and software LBs.
  • Hardware VLB: a VLB created on a physical LB to provide load balancing services
  • Software VLB: a system service VM created on FusionManager to provide load balancing services

Deployment Service

When a tenant applies for the deployment service, the system automatically creates a Proxy VM (PVM) to push FusionManager software and scripts.

The deployment service provides the following functions:
  • Automatic software installation: When you provision application instances using an application template in which the VM template has specific software configured, the system automatically installs the software on the VM that is to run the application instance.
  • Software distribution: After you select a software package and specify the target VMs to install the package, the system automatically installs the software on the specified VMs.
  • Automatic script execution: When you provision application instances using an application template in which the VM template has specific script file configured, the system automatically executes the script on the VM that is to run the application instance.
  • Script distribution: After you select a script file and specify the target VMs to execute the script file, the system automatically distributes the script to the specified VMs and executes the script on them.

Create a VLAN pool.

Add the VLANs planned for external networks to the FusionManager VLAN pool and associate the VLAN pool with a distributed virtual switch (DVS).

 Choose Resource > Resource Zones, select a resource zone, and choose Network Resource Pools > VLAN Pools.

Create an external network.

An external network can connect to the software router and provide public IP addresses for VMs associated with the routed network in the VPC. The external networks created on FusionManager must meet the following requirements: its connection mode is subnet (common VLAN), it is connected to the Internet, its subnet IP address assignment mode is static injection, and it does not contain an IPv6 subnet.

 Choose Resource > Resource Zones, select a resource zone, and choose Network Resource Pools > External Networks.

Create a VSA management network.
The VSA management network provides the management plane for system service VMs. It must meet the following requirements:
  • The VSA management network connects to the system management plane.
  • The VSA management network VLAN has been added to the DVS created on the FusionCompute hypervisor, but the VLAN is not contained in the FusionManager VLAN pool.
 Choose Resource > Resource Zones, select a resource zone, and choose Network Resource Pools > VSA Networks.

Import a VSA VM template to the system.

FusionManager can use a VSA VM template to automatically create system service VMs.

 Choose Resource > Templates and Specs > VM Templates.

Create a virtual firewall.

Create a virtual firewall on the physical firewall device. The virtual firewall can provide a firewall for tenants to apply for router services.

The virtual firewall must be configured on the physical firewall device. For details, see Creating a Virtual Firewall.

Add a physical firewall to FusionManager.

Add the physical firewall device to FusionManager so that the system can configure and manage the firewall based on service requirements.

After the firewall device is successfully added to the system, the virtual firewall configured on the firewall device is automatically added to the network resource pool on FusionManager.

 Choose Resource > Devices > Network Devices > Firewalls.

Configure the firewall resource file.

Configure the firewall resource file to specify the ranges of physical firewall resources, such as the virtual firewalls, sub-interfaces, and access control lists (ACLs), that can be used by FusionManager.

 Choose Resource > Devices > Network Devices > Firewalls.

For details about the configuration procedure, see Importing Firewall Configurations.
