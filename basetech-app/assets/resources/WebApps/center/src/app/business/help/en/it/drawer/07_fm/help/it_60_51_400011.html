VPCs

The VPCs page displays all the virtual private clouds (VPCs) created in the current cloud resource pool.

A VPC provides a secure, isolated network environment in a virtual data center (VDC). You can customize virtual networks that provide the same functions as traditional networks in a VPC. The custom virtual networks can also provide advanced network features, such as elastic IP address and security group, to keep up with service deployment requirements.

VPCs can be classified into two types:
  • Shared VPCs: created by the system administrator and provide network resources for all virtual data centers (VDCs) in the system.
  • Unshared VPCs: created by the tenant and provide network resources for only the VDCs to which the VPCs belong.

Unshared VPCs can only be created by a tenant.

Creating a Shared VPC

The system administrator can create a shard VPC to provide network resources for all VDCs in the system.

NOTE:

Shared VPCs in the same availability zone must use the same network type (direct network or routed network). Otherwise, elastic IP addresses cannot be obtained in a shared VPC if some VMs in the VPC are associated with a direct network.

Modifying VPC Information

Modify the settings of an existing VPC as needed.

Deleting a VPC

If a shared VPC is not needed any more, you can delete it from the system.
