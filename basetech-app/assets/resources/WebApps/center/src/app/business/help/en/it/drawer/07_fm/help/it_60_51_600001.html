Importing Firewall Configurations

Background

On the FusionManager, import the device configuration file of a physical firewall to configure parameters of the physical firewall for the FusionManager system to use, such as source network address translation (SNAT), network address and port translation (NAPT), subinterfaces, access control list (ACL) IDs.

Conditions

  • You have logged in to the FusionManager.
  • Firewall devices have been added to the FusionManger.

Procedure

  1. On the FusionManager, choose Resources > Devices > Network Devices.
  2. In the navigation tree on the left, choose Firewalls.
  3. Click Import Configuration.

    The Import Configuration dialog box is displayed.

  4. Click Download, and download and save the device configuration file to the local computer.
  5. Open the device configuration file and enter the parameters listed in Table 1 and their values in the device configuration file in the following format: 

    <resources>
  <!--firewall rid-->
  <resource devicerid="2"/>
    <ReservedVfw>
      <VfwName name="vfw1"/>
    </ReservedVfw>
<!--Start id of Acl-->
<item name="ID_GENERATOR_ACL_ENTRY_START" value="0"/>
<!--end id of Acl-->
<item name="ID_GENERATOR_ACL_ENTRY_END" value="9999"/>
    鈥︹€
  </resource>
</resources>

    ReservedVfw specifies the firewall reserved for the customer. If multiple firewalls need to be reserved, add this parameter in multiple lines.

    vfw1 indicates the virtual firewall reserved for the user.

    2 indicates the value of Firewall Rid, which identifies the firewall connecting to the FusionManager. You can query the value by choosing Resource > Devices > Network Devices > Firewalls.

    Parameter values listed in Table 1 are the maximum value a physical firewall supports. Users need to determine the parameter values based on the site conditions.

    If both Internet Protocol Security (IPSec) and Layer 2 Tunneling Protocol (L2TP) virtual private network (VPN) services are required, ensure that the ACL rule ranges do not overlap. In the FusionManager system, the default ACL rule start ID for the IPSec VPN service is 3000, and the default end ID is 3499; the default ACL rule start ID for the IPSec VPN service is 3500, and the default end ID is 3999. If you need to manually configure the VPN service on the firewall, ensure that the ACL rule ranges for the FusionManager system and the VPN service on the firewall do not overlap.

  6. Save the device configuration file.

    Open the file using the Internet Explorer browser and check whether the XML file format is correct.

  7. In the Import Configuration dialog box, select the device configuration file on the local PC in Device configuration file.
  8. Click Import.

    A message is displayed, indicating that the firewall configuration file is imported successfully.

  9. Click OK.
  10. If two firewalls are deployed in active/standby mode, change the value of devicerid in the configuration file to the RID of the other firewall and import the file again.

    NOTE:

    The content of the two imported configuration files is consistent, export for the firewall RIDs.
