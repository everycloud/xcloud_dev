Switches

FusionManager can monitor and manage switches added to it.

Adding a Switch to FusionManager

Before adding a switch to FusionManager:
  • Use a remote access tool, such as PuTTY, to log in to the switch by Secure Shell (SSH) and enter the user view. If the login fails due to a warning or exception, address the issue and ensure that the login is successful.
  • Ensure that the switch login password is set to never expire. This is to prevent switch login failures caused by the password change required if the password expires. For the detailed configuration method, see the product document for the in-use switch.
  • SNMP parameters are required for switches of some models. Before adding a switch to the system, configure the required switch settings based on the switch documentation.

After a switch is added to FusionManager, the switch information is displayed on the Switches page.

If the required server model is not available for selection, include more compatible hardware devices into the system. For details, see Adaptation Package Management.

Deleting a Switch

If a switch is not needed any more, you can delete it from the system. After a switch is deleted, FusionManager will not manage or monitor it, but the deletion will not affect the virtual resources assigned to the resource pool the deleted switch originally belongs to.

FAQ

Why is the port status information not displayed for some switches?

The system can monitor the port status only for the following switches:
  • Huawei Quidway S5352
  • Huawei Quidway S5710
  • Huawei E9000 XCUB
