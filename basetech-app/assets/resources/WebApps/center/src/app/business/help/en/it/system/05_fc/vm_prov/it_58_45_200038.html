Configuring Specifications Attribute Customization for a Windows VM

  • Scenarios
  • Prerequisites
  • Procedure

Scenarios

Configure specifications attribute customization for a VM. If the VM is used as a template for creating VMs. this function enables the created VMs have the following functions:
  • Have a unique initial password
  • Support computer name customization
  • Synchronize time with the host
  • Obtain an IP address in static-manual mode
NOTE:

Declaration: This feature is a high-risk feature. Using this feature complies with industry practices. However, end user data may be required for implementing the feature. Exercise caution and obtain end user's consent when using this feature.

This operation modifies user VM registries. If the end user data is involved, obtain end user's consent before you perform this operation.

This function is not required when:

  • The VMware hypervisor does not require the following configuration.

  • If the random initial password and VM specifications attribute customization functions are not required, do not configure this function.

  • The VM runs the Windows 2000 Server SP4 or Windows 2000 Advanced Server SP4 OS. The two OSs do not support this function.
  • The VM SID has been configured. SID configuration includes specifications attributes configuration. For details about VM SID configuration, see Configuring the VM SID Customization.
  • The VM SID has been configured. SID configuration includes specifications attributes configuration.

Prerequisites

Conditions

The OS and the Tools have been installed on the VM.

  • The OS and the Tools have been installed on the VM.

  • The VM SID has not been configured.

Procedure

    Delete the historical file.

    1. In the VNC login window, enable the user administrator of the OS and delete other users.
    2. Go to the root directory of the C drive and check whether the setpass.vbs file exists.

      If yes, delete the setpass.vbs file.

    Modify registration entries.

    1. On the VM desktop displayed in the VNC window, click Start, and enter regedit in the search box, and press Enter.

      A dialog box is displayed.

    2. Choose HKEY_LOCAL_MACHINE > SOFTWARE > Microsoft > Windows NT > CurrentVersion > Winlogon.

      All registry entries under Winlogon are displayed.

    3. Check whether DefaultUserName, DefaultPassword, and AutoAdminLogon are listed in the registry entries.

      • If yes, go to 9.
      • If no, go to 6.

    4. Choose Edit > New > String Value.

      The default name New Value #1 is displayed in the right pane.

    5. Right-click New Value #1, and choose Rename.

      Change the default name New Value #1 to DefaultUserName.

    6. Perform 6 to 7 twice to change the default names to DefaultPassword and AutoAdminLogon respectively.
    7. Double-click DefaultUserName, DefaultPassword, and AutoAdminLogon one by one, and enter Administrator, the current password of Administrator, and 1 in the Value data dialog box, respectively.

      NOTE:
      If the current VM does not have the password of Administrator, do not set the value of DefaultPassword.

    8. Delete AutoLogonCount if it is listed in the registry entries.
    9. Click OK.

      The dialog box is closed.

    10. Choose HKEY_LOCAL_MACHINE > SOFTWARE > Microsoft > Windows > CurrentVersion > Run.

      All registry entries under Run are displayed.

    11. Double click (Default).

      A dialog box is displayed.

    12. Enter c:\setpass.vbs in Value data.
    13. Click OK.

      The dialog box is closed.

    14. Check whether the Clock synchronization policy parameter of the VM is set to Synchronize with host clock.

      • If yes, go to 17.
      • If no, no further action is required.

    15. Choose HKEY_LOCAL_MACHINE > SYSTEM > Setup > HuaweiPV.
    16. Delete SetTime.
