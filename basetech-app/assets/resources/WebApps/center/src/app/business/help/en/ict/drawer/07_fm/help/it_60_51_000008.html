Resource Pool Configuration

Overview

FusionManager can manage hardware devices, virtualized resources, and applications in a cloud system. After setting up the FusionManager system, you can add multiple FusionSphere OpenStack instances into a unified cloud resource pool for FusionManager to manage resources in the pool in a centralized manner.

Web Client Navigation

Add Keystone to FusionManager.

Keystone is a rights management and authentication service provided by FusionSphere OpenStack. After the Keystone service is added to FusionManager, the Keystone rights management function applies to FusionManager.

Before the Keystone service is added to FusionManager, FusionManager cannot use the resources provided by OpenStack or manage rights specified in FusionSphere OpenStack.
NOTE:

After the Keystone is added to FusionManager, if you use FusionSphere OpenStack, rather than FusionManager to manage users or perform other operations, use the Keystone v3 interfaces. Otherwise, the operations may fail due to authentication issues.

User > Keystone Configuration

Associate FusionManager with an OpenStack instance.

An OpenStack instance is a hypervisor that manages all virtualized resources in a cloud platform.

FusionManager can manage and maintain virtual resources in an OpenStack instance only after the OpenStack instance is associated with FusionManager.

Choose Resource > OpenStack Instances.

Add physical devices to FusionManager.

Add physical devices to FusionManager so that FusionManager can monitor and manage these devices in a unified manner.

Choose Resource > Devices.
