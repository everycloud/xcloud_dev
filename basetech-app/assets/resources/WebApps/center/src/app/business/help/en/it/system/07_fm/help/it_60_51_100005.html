A firewall is a system implementing one or one set of access control policies. It monitors communication channels between trusted networks (internal networks) and untrusted network (external networks) to prevent risks on external networks from affecting internal networks.

After being added to FusionManager, a firewall can be used to:
  • Control the incoming and outgoing network traffic of physical devices added to FusionManager.
  • Provide virtual firewall services, such as routed networks in a virtual private cloud (VPC), elastic IP addresses, network address translation (NAT), access control list (ACL), and virtual private network (VPN).

Adding a Physical Firewall to FusionManager

Before adding a switch to FusionManager, use a remote access tool, such as PuTTY, to log in to the switch by Secure Shell (SSH) to check whether you can successfully log in to the switch and enter the switch user interface. If the login fails or stops responding due to alarms or other exceptions, address the failure and then add the switch to FusionManager.

Choose Resource > Devices > Network Devices > Firewalls and click Add Firewall. For the detailed operation procedure, see Connecting a Firewall to FusionManager.

After a firewall is added to FusionManager, the firewall information is displayed on the Firewalls page.

If the required server model is not available for selection, include more compatible hardware devices into the system. For details, see Adaptation Package Management.

Creating a Virtual Firewall

Virtual firewalls are logical firewalls virtualized on a physical firewall. Each virtual firewall functions as an independent firewall device to monitor and filter packets over its parent physical firewall device. FusionManager implements firewall functions in the system using virtual firewalls.

  1. Virtual firewalls are created on its parent physical firewall device. For the detailed operation procedure, see Creating a Virtual Firewall.
  2. If a physical firewall has been added to FusionManager before virtual firewalls are created it, you need to trigger a firewall discovery operation on FusionManager after virtual firewalls are created on it.

    Choose Resource > Devices > Network Devices > Firewalls, locate the row that contains the target firewall, click More, and select Rediscover.

Setting the Firewall Usage Scope

All functions of a firewall, such as Source Network Address Translation (SNAT), Network Address and Port Translation (NAPT), and subinterfaces, are available for users by default. You can import a custom firewall configuration file to the system to restrict the usage permission to specified firewall functions.

  1. Choose Resource > Devices > Network Devices > Firewalls and click Import Configuration.
  2. Download the configuration file template to your PC and fill the template with custom firewall settings data. For the detailed operation procedure, see Importing Firewall Configurations.
  3. Import the custom configuration file to the system.
If the system has two firewalls configured in active/standby mode, you only need to import the custom configuration file once. The imported custom settings take effect for both the active and standby firewalls.

Modifying Firewall Connection Parameters

If the management information of a firewall has been modified after the firewall is added to FusionManager, you need to synchronize the new information to FusionManager in a timely manner. Otherwise, FusionManager will lose management for the firewall.

Choose Resource > Devices > Network Devices > Firewalls, locate the row that contains the target firewall, click More, and select Modify Connection Parameters.

Deleting a Firewall

If a firewall is not needed any more, you can delete it from the system. After a firewall is deleted, FusionManager will not monitor or manage this firewall, and the services associated with this firewall will be unavailable.

If active and standby firewalls are added to FusionManager and one firewall is deleted, the other firewall is also automatically deleted.

Choose Resource > Devices > Network Devices > Firewalls, locate the row that contains the target firewall, and click Delete.
