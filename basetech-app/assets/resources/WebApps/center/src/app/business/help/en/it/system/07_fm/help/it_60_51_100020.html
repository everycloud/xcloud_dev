A storage resource pool is a collection of storage resources to be allocated to VMs or application instances in the FusionManager system.

Storage resources in a storage cluster that is added to FusionManager can be used for service provisioning only after the resource cluster is associated with a resource zone in FusionManager.

Storage resources are not required to be independently associated with a resource zone, because storage resources in a resource cluster can be included in a FusionManager resource zone when the resource cluster is associated with the resource zone on the Computing Resource Pools page. The included storage resources after association are displayed on the Storage Resource Pools page. Storage resources can be classified into the following types"
  • Primary storage: Data stores in a resource cluster associated with a resource zone are primary storage resources.
  • Secondary storage: The primary storage resources associated with all hosts are identified as secondary storage resources, which are mainly used to store VM templates for creating VMs across clusters.
  • Disaster recovery (DR) storage group: DR storage groups take effect only when the UltraVR product is used in the system. For the usage of DR storage groups, see the UltraVR Product Documentation.

Adding Secondary Storage Resources to FusionManager

The primary storage resources associated with all hosts are identified as secondary storage resources. The system stores created VM templates preferentially in secondary storage resources to facilitate cross-cluster VM creation.

Choose Resource > Resource Zones, select a resource zone, choose Storage Resource Pools > Secondary Storage, and click Add.

After a primary storage device is added to FusionManager as a secondary resource device, the storage device information is switched to the Secondary Storage page from the Primary Storage page.

Changing the Storage SLA

The storage SLA is configured during resource cluster association. You can also change another storage SLA when using the data store after cluster association. This function allows flexible use the storage resources in the system.

Choose Resource > Resource Zones, select a resource zone, navigate to the Storage Resource Pools page, locate the row that contains the target storage on the Primary Storage page, and click Modify. Alternatively, locate the row that contains the target storage on the Secondary Storage page, click More, and select Modify Storage SLA.

Setting a Storage Device to Maintenance Mode

If a storage device is set to maintenance mode, the storage device is isolated from the entire system and is not available for creating VMs and disks on it. However, the VMs or disks that are already running on the storage device will not be affected.

Choose Resource > Resource Zones, select a resource zone, navigate to the Storage Resource Pools page, locate the row that contains the target storage on the Primary Storage page, and click More, and select Enter Maintenance Mode. Alternatively, locate the row that contains the target storage on the Secondary Storage page, click Enter Maintenance Mode.

Migrating Disks in a Data Store to Another Data Store

Migrate the disks in a data store to another data store.

NOTE:
Declaration: This feature is a high-risk feature. Using this feature complies with industry practices. However, end user data may be required for implementing the feature. Exercise caution and obtain end user's consent when using this feature.

Migrating disks from one data store to another improves system disk availability. If the end user data is involved, obtain end user's consent before you migrate the disks.

Choose Resource > Resource Zones, select a resource zone, navigate to the Storage Resource Pools page, locate the row that contains the target data store, click More, and select Migrate Disk.

Exporting the Data Store List

To facilitate data backup or statistical analysis, you can export the detailed information about all the current data stores to the local PC in an .xls file.

Choose Resource > Resource Zones, select a resource zone, navigate to the Storage Resource Pools page, and click Export.

Deleting a Secondary Storage Resource

Adding or deleting a secondary storage device is only an identification change for a primary storage device. Deleting a secondary storage device only removes the secondary storage identity of a primary storage device, and data stored in the primary storage device will not be deleted.

Choose Resource > Resource Zones, select a resource zone, choose Storage Resource Pools > Secondary Storage, locate the row that contains the target storage device, click More, and select Delete.

After a secondary resource device is deleted, the primary storage information is switched to the Primary Storage page from the Secondary Storage page.

Creating a Disk

Create a disk and attach it to a VM to increase the storage space of the VM.

Choose Resource > Resource Zones, select a resource zone, navigate to the Storage Resource Pools page, and click the storage name to view storage details. On the Disk page, click Create.

Attaching a Disk to a VM

A disk can store the data of a VM only after being attached to the VM.

Choose Resource > Resource Zones, select a resource zone, navigate to the Storage Resource Pools page, and click the storage name to view storage details. On the Disk page, locate the row that contains the target disk and click Bind VM.

After attaching the disk to the VM, you also need to add the disk to the VM operating system (OS).
  • If the attached disk is a newly created disk, format the disk. For the detailed operation procedure, search from the Internet or Forum.
  • If the attached disk already contains data and the disk drive letter is the same as the disk drive letter on the VM, set the disk status from offline to online.

Detaching a Disk from a VM

Detach a disk from its attached VM.

To detach a disk from a running VM, note the following restrictions:

  • The VM has the Tools run properly on it.

  • The system disk cannot be detached.
  • A raw device mapped (RDM) disk cannot be detached.
  • Only VMs running the following operating systems (OSs) supports disk detaching when the VMs are running:

Choose Resource > Resource Zones, select a resource zone, navigate to the Storage Resource Pools page, and click the storage name to view storage details. On the Disk page, expand the disk details area, locate the row that contains the target VM, and click Unbind.

Allocating a Disk

Allocate a disk to the VDC administrator of the specified VPC. Then, inform the VDC administrator of allocating the disk to the specified user at the earliest time. If a disk has been attached to a VM, the disk cannot be independently allocated to a user. Instead, you can allocate the entire VM to the user.

Choose Resource > Resource Zones, select a resource zone, navigate to the Storage Resource Pools page, and click the storage name to view storage details. On the Disk page, select the target disk and click Allocate.

Deleting a Disk

If a disk is not needed any more, you can delete it from the system to release storage resources. After a disk is deleted, all the data on the disk will also be deleted. Therefore, exercise caution when performing this operation.

NOTE:
Declaration: This feature is a high-risk feature. Using this feature complies with industry practices. However, end user data may be required for implementing the feature. Exercise caution and obtain end user's consent when using this feature.
A disk can be deleted in two modes:
  • Ordinarily delete: This mode destroys the file system of the disk so that the data in the disk is not allowed to be retrieved. This mode features rapid deletion but the disk data may be restored using the residual data. Therefore, data security in this deletion mode is poor.
  • Safely delete: This mode deletes disk data by overwriting the data. The overwritten data cannot be restored, and therefore this deletion mode features highest data security. However, the deletion speed in this mode is slow, and the deletion consumes system resources.

Choose Resource > Resource Zones, select a resource zone, navigate to the Storage Resource Pools page, and click the storage name to view storage details. On the Disk page, locate the row that contains the target disk, click More, and select the required operation.

FAQ

How to Attach a Raw Device Mapped Disk That Has Been Detached from a VM to a Physical Server

If the disk to be detached is a raw device mapped disk and the logical unit number (LUN) the disk uses is to be attached to a physical server, adhere to the following precautions:
  • Do not delete the raw device mapped disk and the data store.
  • Add a device partition before attaching the LUN to a physical server.
For details, see How to Attach a Raw Device Mapped Disk That Has Been Detached from a VM to a Physical Server.
