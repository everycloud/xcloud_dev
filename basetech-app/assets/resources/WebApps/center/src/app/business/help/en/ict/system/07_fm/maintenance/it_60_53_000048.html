Replacing the nginx Certificate of the FusionManager Web Server

  • Scenarios
  • Impact on the System
  • Prerequisites
  • Procedure

Scenarios

This topic describes how to manually replace the nginx certificate of the FusionManager web server without interrupting the services running in the system.

The login certificate provided by FusionManager is a self-signed certificate. When you attempt to log in to the system, a message is displayed indicating that no security certificate is installed. If you expect to use your custom certificate to authenticate login, you can apply for a certificate from the third-party Certificate Authority (CA) and replace the default certificate with your own certificate by following steps described in this topic.

The default nginx certificate password of the FusionManager web server is Huawei@CLOUD8!.

The nginx certificate uses RSA 1024 as the encryption algorithm and SHA128 as the signature algorithm. However, these two algorithms are insecure algorithm and may pose security risks to some extent.

Impact on the System

The new certificate takes effect only after the system is restarted. However, during the restart, FusionManager services are interrupted. If the new certificate is incorrect, the FusionManager web client is not available to access, and the FusionManager interfaces cannot be used.

Prerequisites

  • You have obtained the public key and private key of the certificate file, and have named the public key server.crt and private key server.key.
  • You have obtained the management IP addresses of the active and standby FusionManager nodes and the galaxmanager user password.
  • PuTTY is available.
  • WinSCP is available.

Procedure

    1. Create a folder ws on the local PC to store the public key server.crt and private key server.key.
    2. Use WinSCP to log in to the active FusionManager node as user galaxmanager and copy the folder ws to the /opt/goku/data/upload directory on the active FusionManager node.
    3. Use PuTTY to log in to the active FusionManager node.

      Ensure that the active node management IP address and username galaxmanager are used to establish the connection.

    4. Run the following command to disable logout on timeout:

      TMOUT=0

    5. Run the following command to replace the certificate:

      updateCa ws

    6. Enter the certificate password to be encrypted.
    7. Run the following command to restart the services to make the modified certificate take effect:

      nginx restart

      The services are successfully restarted if the following command output is displayed:

      restart success

    8. Replace the certificate on the standby FusionManager node. For details, see 2 to 7.

    Generate the ssokey certificate used by third parties.

    After replacing the FusionManager nginx certificate, you need to create a new ssokey certificate for use by third parties.

    Before performing the following steps to generate the new ssokey certificate, ensure that you have installed Java JDK and configured correct Java environment variables on the local PC:

    1. Choose Start > Run on the PC.
    2. Enter cmd and click OK to enter the command-line interface.
    3. Run the following command:

      set TOOL="C:\Program Files\Java\jdk1.6.0_25\bin"

      "C:\Program Files\Java\jdk1.6.0_25\bin" is the Java JDK installation directory and the actual tool version. Ensure that you entered the actual installation directory and version number in the preceding command.

    4. Run the following command to generate the new ssokey certificate:

      %TOOL%\keytool -import -keystore ssokey -file Public key name to be replaced -alias ssosrv -storepass Encryption password of the certificate -keypass Encryption password of the certificate

      The output file is ssokey, which can be used by northbound third-party clients.
