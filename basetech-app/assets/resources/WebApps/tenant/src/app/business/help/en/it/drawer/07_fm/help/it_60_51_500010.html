Alarm List

The alarm list displays the alarms generated for FusionSphere. Using the alarm information, you can find and locate faults promptly.
NOTE:

You can click the alarm name in the Alarm Name column, and handle the alarm as instructed by the alarm help.

If detailed information in the alarm help fails to be displayed, run the reg add HKLM\SOFTWARE\Microsoft\HTMLHelp\1.x\ItssRestrictions /v MaxAllowedZone /t REG_DWORD /d 3 /f command and open the alarm help again. Then, you can search for an alarm by the alarm ID. If multiple alarms are displayed as the search result by the alarm ID, you can find the alarm to be queried based on the sensor type in the alarm name.

Clearing an Alarm

After handling alarms, you can manually clear those alarms that have been handled but cannot be automatically cleared.

You can also clear multiple alarms at a time. To do so, select the alarms to be cleared and click Clear on the left upper corner of the page.

If the alarm has been cleared, one of the following is displayed in Clear Type indicating how this alarm is cleared:
  • Manually Clear
  • Automatically Clear
  • Exception Clear: After the component for which this alarm is generated is restored using the historical backup data, this alarm that has been reported to the FusionManager is rolled back. Therefore, the FusionManager automatically clears this alarm.

Exporting Alarms

You can export the alarm list to the local PC so that you can view alarms offline and confirm and handle alarms without using FusionManager.
