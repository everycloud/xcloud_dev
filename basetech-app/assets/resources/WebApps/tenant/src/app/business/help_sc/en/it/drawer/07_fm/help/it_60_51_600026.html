Configuring the VM Process Monitoring Function

Scenarios

Hyperic HQ is a software resource monitoring and management platform developed using Java. With the Hyperic HQ server and client configured in the system, ServiceCenter can monitor VM processes.
  • The Hyperic HQ server is manually installed by the administrator during the PVM template creation. Therefore, the Hyperic HQ server is by default installed on the Proxy VM (PVM) after the PVM VM service is deployed on a virtual private cloud (VPC).
  • The Hyperic HQ client must be manually installed on the VM to be monitored.
ServiceCenter supports Hyperic HQ 4.6.6. Therefore, the VM operating system (OS) restrictions and supported software processes are all subject to the capabilities of Hyperic HQ 4.6.6.

Configuring the VM Process Monitoring Function on a Linux VM

    1. Place all the software to be installed into an ISO file, so that the installation file can be copied to the VM.

      You can use any ISO image creation tool, such as UltraISO and ISOMaker, to create an ISO file for the desired software. For more information, visit Forum.

      In the following steps, the ISO file is named hyperic.iso as an example, which contains the hyperic-hq folder.

    2. Log in to the VM using Virtual Network Computing (VNC) and attach the hyperic.iso file to the VM.

      For the detailed procedure for attaching a CD/DVD-ROM drive, see Mounting a CD/DVD-ROM Drive or an ISO File.

    3. If you are using a Graphical User Interface (GUI) system, right-click on the VM OS desktop and choose Open in Terminal from the shortcut menu.

      The command-line interface (CLI) is displayed.

    4. Run the following command to create the xvdd directory:

      mkdir xvdd

    5. Run the following command to attach the CD/DVD-ROM drive to the VM:

      mount Mounting directory xvdd

      For example, run the mount /dev/sr0 xvdd command.

      Table 2 lists the directories for mounting a CD/DVD-ROM drive or ISO file to a VM running the Linux OS.

    6. Run the following command to copy the hyperic-hq folder in the ISO image file to the /home directory of the VM:

      cp -r xvdd/hyperic-hq/ /home

    1. Run the following command to switch to the /home directory:

      cd /home

    2. Run the following command to change the permission to the hyperic-hq directory:

      chmod -R 775 hyperic-hq/

    3. Run the following command to switch to the conf directory in hyperic-hq, and use the visual interface (vi) editor to open the agent.properties file:

      cd hyperic-hq/conf

      vi agent.properties

    4. Press i to enter the editing mode and add a line similar to the following at the end of the file to set the VM name:

      platform.name=vm01

      The preceding example sets the VM name to vm01. The VM name must be unique in a VPC.

    5. Press ESC and enter :wq to save the configuration and exit the vi editor.
    6. Run the following command to switch to the bin directory in hyperic-hq:

      cd /home/hyperic-hq/bin

    7. Run the following command to install the process monitoring function for the VM:

      sh hq-agent.sh start

    1. Enter the IP address of the Hyperic HQ server in the following line. 

      What is the HQ server IP address: 192.168.5.14

      The Hyperic HQ server IP address is the IP address of the PVM VM that can communicate with the VM to be monitored. To be monitored by the system, a VM must have at least one IP address connected to the PVM VM.

      You can query the PVM VM IP address on the Deployment Service page of the VPC, and can use the ping command to determine which PVM VM IP address is connected to the VM to be monitored.

    2. Keep the default values of the other connection parameters. 

      Should Agent communication to HQ always be secure [default=yes]: yes
What is the HQ server SSL port [default=7443]: 7443
      The VM is successfully connected to the Hyperic HQ server if the following information is displayed:
      Testing secure connection ... Success

    3. Enter the Hyperic HQ server login username and password.

      The Hyperic HQ server login username is hqadmin, and its default password is Huawei@123.

    4. Enter the VM IP address and the default communication port number. 

      What IP should HQ use to contact the agent [default=192.168.5.25]: 192.168.5.25
What port should HQ use to contact the agent [default=2144]: 2144

    5. Enter yes.

      The Hyperic HQ process monitoring function is successfully configured if the following information is displayed:
      Are you sure you want to continue connecting? [default=no]: yes
...
- Successfully setup agent

      After about 5 minutes, the VM process monitoring information is displayed on the VM monitoring page.

    4. Navigate to the conf directory under hyperic-hq and open the agent.properties in the directory using Notepad.
    5. Add a line similar to the following at the end of the file to set the VM name:

      platform.name=vm01

    6. Save and close the file.
    7. Open the CLI interface of the VM.

      cd \hyperic-hq\bin

    9. Run the following command to install the process monitoring function for the VM:

      hq-agent.bat install

      wrapper  |  Hyperic HQ Agent service installed.

    10. Run the following command to start the process monitoring function on the VM:

FAQ-How Do I Update the Configurations?

    After you installed and configured the Hyperic HQ client by following the preceding procedure, if the administrator has restored or redeployed the PVM due to causes such as PVM faults during VM process monitoring, you need to use the IP address of the new PVM to configure the Hyperic HQ client again to make the Hyperic HQ client available to use. To do so, perform the following steps:

    1. Navigate to the hyperic-hq-agent-4.6.6\bin directory in the Hyperic HQ software installation directory.
    2. Run the required command to stop the Hyperic HQ Agent service.

      • If the VM runs the Windows OS, run the hq-agent.bat stop command.
      • If the VM runs the Linux OS, run the sh hq-agent.sh stop command.

    3. Delete the data folder.
    4. Run the required command to restart the monitoring process for reconfiguration.

      • If the VM runs the Windows OS, run the hq-agent.bat start command.
      • If the VM runs the Linux OS, run the sh hq-agent.sh start command.
