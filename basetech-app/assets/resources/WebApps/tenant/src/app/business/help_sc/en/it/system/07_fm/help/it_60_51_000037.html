Overview

With the increase of users and data traffic, processing capacities and computing density of service servers must increase accordingly. Only one server may not be able to meet the growing service requirements. If multiple servers are connected using network cables and used to deploy the same service, you must ensure that service loads are balanced among the servers.

A load balancer (LB) enables service requests to be sent to multiple servers running the same service based on the pre-configured load balancing policies. It improves service throughput and server resource utilization, ensures the proper running of the system, and provides users with quality access services.

Principles

The ServiceCenter supports hardware LBs and software LBs.

Hardware LB

Hardware LBs provide load balancing functions through load balancer instances created on physical load balancers. Before applying for hardware LBs for planned services or networks, you need to connect a physical load balancer to the ServiceCenter and create VPCs and networks.

Software LB

The system service VM in the ServiceCenter system provides software LB functions. The VSA management network provides a management plane network for system service VMs, so that system service VMs can communicate with management nodes. Upon receiving a request for creating a VLB for a PC, the ServiceCenter instructs the VSAM node to create and configure the VLB.

Each VLB has two NICs. One NIC is connected to the front-end network, and the other is connected to the back-end network. As shown in Figure 1, the network where the service VMs associated with the VLB reside is referred as the back-end network. The network that provides the load balancing service and access IP addresses for users is referred as the front-end network.
Figure 1 Software LB

Precautions

  • To use the VLB function, the assignment mode for the IP addresses of the front-end and back-end subnets must be internal DHCP or static IP addresses.
  • To create a software LB, if the front-end and back-end networks of the VLB are the same network, ensure that the service VM is also located in this network.
