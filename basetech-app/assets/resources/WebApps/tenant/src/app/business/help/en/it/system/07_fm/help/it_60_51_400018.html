Shared VPC

The shared virtual private cloud (VPC) is the VPC created by the system administrator to provide network resources for all virtual data centers (VDCs) in the system.

You can use network resources available in the shared VPC to deploy VMs or application instances, or use the elastic IP address and security group functions provided by the shared VPC.

NOTE:

When creating VMs in a shared VPC, you are advised to configure security groups to isolate the VMs and implement VM access security control.

  • When creating VMs in a shared VPC, you are advised to configure security groups to isolate the VMs and implement VM access security control.
  • In a shared VPC, the system administrator has rights to export templates from VMs, attach USB devices to the VMs, and detach USB devices from the VMs. If the tenant does not allow a system administrator to perform these operations on VMs, create the VMs in a non-shared VPC. The administrator also has rights to import VMs to a shared VPC.
