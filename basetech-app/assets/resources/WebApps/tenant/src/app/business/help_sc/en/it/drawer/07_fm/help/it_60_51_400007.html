DNAT

Destination network address translation (DNAT) enables ServiceCenter to map an internal network address to a public network address. If an external address initiates a request to set up a service connection with ServiceCenter, the gateway configured on the system firewall receives the connection request and translates the external address to an internal address, so that the external address can communicate with the system and use external services provided by the system.

Applying for the DNAT Service

Apply for the DNAT service for a VM so that the VM can connect to the public network with its private IP address translated into a public IP address.

Releasing the DNAT Service

After the DNAT service of a VM is released, the mapping relationship between the private IP address of the VM and the public IP address is deleted. As a result, the VM cannot connect to the public network.
