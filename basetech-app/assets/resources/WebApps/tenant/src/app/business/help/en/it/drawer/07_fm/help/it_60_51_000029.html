Introduction to the FusionManager Web Client

The FusionManager Web Client provides two operation views, administrator view and tenant view. The administrator can manage all resources in the system, whereas tenants can manage only the resources of their own virtual data centers (VDCs). This part describes the functions of the tenant view.

Functions of the Tenant View

Tenants can manage and monitor services, resources, and applications in their own VDCs as well as manage rights of users in the VDCs. For details, see Table 1.
Table 1 Functions of the tenant view

SN

Category

Functions

1

Resources
Resources refer to all virtual resources managed by FusionManager. You can perform the following operations on these resources:
  • Create and manage VMs.
  • Manage disks.
  • Manage the templates, software packages, and scripts required for creating VMs.

2

VPC

A VPC provides a secure, isolated network environment for the VDC. You can customize virtual networks that serve as traditional networks in the VPC and also provide advanced network features, such as elastic IP address, to keep up with service deployment requirements.

3

Applications

An application instance is a collection of VMs and service software that provides specific services, for example, SQL Server and CRM/ERP.

You can manage application instances, application templates, and related policies.

4

Monitoring
Monitored objects include alarms, performance, tasks and logs.
  • Alarms: Using the alarm information, you can find and locate faults promptly.
  • Performance: The performance list of FusionManager provides performance monitoring indicators of the VMs in the hypervisor connected to FusionManager.
  • Tasks and logs: you can view the task status and operation logs.

5

Users
You can manage users, roles, and domains.
  • A role is a collection of diverse operation rights. You can assign roles to users so that each user can have different operation rights.
  • Domains are used to classify resources. You can customize domains at the geographical or administrative level. By associating domains with different users and resources, you can enable users to manage resource in their own domains.

VDC View Switchover

FusionManager provides an independent view for each VDC to ensure system security. If a tenant is added to multiple VDCs, the tenant can query the current VDC on the upper-right corner of the page and click the VDC name to quickly switch to another VDC view.
