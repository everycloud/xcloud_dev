Setting Mapping Relationship Between haproxyIP and FusionSphere OpenStack Domain Name

Scenarios

Before using Virtual Network Computing (VNC) to log in to VMs on ServiceCenter, you need to configure the mapping relationship between the haproxyIP and the FusionSphere OpenStack domain name on the local PC. This topic describes how to configure the haproxyIP relationship.

Prerequisites

  • You have obtained the haproxyIP and the FusionSphere OpenStack domain name.

  • The IP address of the local PC can ping the haproxyIP.

Procedure

  1. On the local PC, navigate to the C:\Windows\System32\drivers\etc directory and open the hosts file using Notepad.
  2. Add following similar to the following at the end of the file to set the mapping relationship: 

    haproxyIP FusionSphere
OpenStack domain name
    For example:
    172.16.0.2 nova-novncproxy.az1.dc1.com

    In this line of information, 172.16.0.2 indicates the planned haproxyIP, and nova-novncproxy.az1.dc1.com indicates the FusionSphere OpenStack domain name.

  3. Save and close the file.
