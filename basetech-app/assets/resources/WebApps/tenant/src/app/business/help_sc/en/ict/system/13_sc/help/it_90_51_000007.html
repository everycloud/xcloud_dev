A service is a resource template that users can apply for. It is created by the administrator or virtual data center (VDC) administrator and displayed to users for them to apply for resources.

A service catalog is used to store services that have the same properties. Classifying services using a service catalog makes management and maintenance more convenient for the administrator.

You can find services that you focus on by service name or service ID in All Services. You can also find required services by the catalog on the left.

A VDC user only has permission to apply for services. The VDC administrator can only manage the created services, for example, editing and publishing the created services.

Creating a Service

After a service is created and published, users can view and apply for it.

Resources such as application templates, VM templates, and virtual private clouds (VPCs) must be created before the application service creation.

The system provides a service type for the virtual data center (VDC) administrator to create services. When creating a service, the VDC administrator can determine whether the service needs to be approved.

Choose Services > Services Catalog, and click Create.

Applying for a Service

You can query or apply for a service as required in All Services or a specific catalog.

If Type is set to Shared during cloud hard disk application, the cloud hard disk can be attached to multiple VMs.
NOTE:
If multiple VMs that use a shared disk write data into the disk at the same time, data may be lost. Therefore, you need to use application software to control disk access permission.

Choose Services > Services Catalog, select a catalog, and click Apply For in the service area.

Editing a Service

When a service is created but not published, the administrator can modify properties of the service as required, such as the resource quota and catalog of the service.

Choose Services > Services Catalog > service catalog to which the service belongs, and click Edit in the area where the service locates.

Deleting a Service

When a service is created but not published, the administrator can delete the service.

Choose Services > Services Catalog > service catalog to which the service belongs, and click Delete in the area where the service locates.

Publishing a Service

After a service is created and published, users can view and apply for it.

Choose Services > Services Catalog > service catalog to which the service belongs, and click Publish in the area where the service locates.

Canceling the Publishing of a Service

The administrator can edit or delete a service only when the publishing of the service is canceled.

The publishing of a service can be canceled only when the service is in the Publish status.

Choose Services > Services Catalog > service catalog to which the service belongs, and click Cancel Publishing in the area where the service locates.
