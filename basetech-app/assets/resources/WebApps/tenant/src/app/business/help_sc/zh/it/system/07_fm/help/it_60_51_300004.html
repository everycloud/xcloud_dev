虚拟机与物理计算机一样，是运行操作系统和应用程序的虚拟计算机。

虚拟机运行在某个主机上，并从主机上获取所需的CPU、内存等计算资源，以及图形处理器、USB设备、网络连接和存储访问等能力。多台虚拟机可以同时运行在一台主机中。

查询虚拟机初始密码

若虚拟机属于FusionCompute虚拟化环境，且为随机密码时，可查看虚拟机操作系统的初始密码。

在“资源 > 虚拟机”界面，展开虚拟机详情，在“初始密码”后，单击“查看”。

使用VNC方式登录虚拟机

VNC（Virtual Network Computer）是虚拟网络计算机的缩写，是一款远程控制工具。通过VNC方式，管理员能够连接并查看虚拟机的界面，便于进行虚拟机的管理和维护。

查看虚拟机的界面并不代表能够登录虚拟机的操作系统，即使使用VNC登录虚拟机，也只是将虚拟机的当前画面显示出来。若虚拟机的操作系统设置有密码，仍然需要对应的用户名和密码才可登录虚拟机。

在“资源 > 虚拟机”界面，勾选一个虚拟机，单击“VNC登录”。

VNC登录虚拟机，需注意以下几点：
  • 高危特性声明：该功能使用过程中可能触及最终用户数据，符合业界惯例，但仍须谨慎使用。该操作将涉及虚拟机用户隐私，使用该功能时务必遵从当地适用的法律法规。

  • 为确保安全，请虚拟机用户及时修改虚拟机密码。

  • 通过VNC方式登录虚拟机时，虚拟机用户将无法登录该虚拟机。

若登录过程出现问题，请参考VNC登录常见问题。

虚拟机电源管理

包括启动或唤醒、休眠、关闭、重启虚拟机。其中关闭和重启有安全和强制两种方式，建议使用安全方式进行下电和重启。只有当虚拟机操作系统异常，无法安全关闭或重启时，才可强制关闭或重启，强制关闭或重启可能导致虚拟机磁盘数据受损。

大量虚拟机同时启动或关闭时，会加重主机的负载。如果需同时对一个主机上的大量虚拟机执行启动或关闭操作，建议分批进行，避免对其他虚拟机的业务造成影响。

在“资源 > 虚拟机”界面，虚拟机所在行，选择“更多”中对应的操作。

导出虚拟机列表

将当前所有虚拟机详细信息，导出到本地，并保存为*.xls文件，便于信息备份和统计。

在“资源 > 虚拟机”界面，选择“更多 > 导出”。

监控虚拟机的进程

基于Hyperic HQ提供虚拟机的进程监控功能。前提条件如下：
  1. 在VPC中部署服务，即需要具备PVM虚拟机，且PVM虚拟机关联的网络，需与虚拟机使用的网络互通。
  2. 在虚拟机中安装并配置Hyperic HQ软件客户端，具体操作请参考如何配置虚拟机进程监控功能。
说明：

若管理员由于PVM虚拟机故障等原因修复或重新部署了PVM虚拟机，则虚拟机中的Hyperic HQ软件客户端需使用新的PVM虚拟机IP地址重新配置才可继续使用，具体操作请参考如何配置虚拟机进程监控功能中的“FAQ-如何更新配置”。

在“资源 > 虚拟机”界面，展开虚拟机详情，单击“查看监控详情”，进入“监控”界面“进程”页签，查看虚拟机进程监控信息。

调整磁盘

增加磁盘

通过为虚拟机绑定存储的方式扩大虚拟机的存储空间。
  • 将新创建磁盘绑定到虚拟机上。
  • 将现有的空闲磁盘绑定到虚拟机上。

虚拟机处于“已停止”或“运行中”状态，且虚拟机所在主机未进入维护模式时，才可以增加磁盘。

在“资源 > 虚拟机”界面，展开虚拟机详情，在“磁盘”后，单击进入“磁盘”界面，单击“添加”。

磁盘绑定成功后，还需在虚拟机中将磁盘添加至操作系统中。
  • 若绑定的磁盘是新创建的磁盘，则需对新增磁盘进行格式化。具体操作方法可在互联网或技术论坛中查找获取。
  • 若绑定的磁盘是已有数据的磁盘，且磁盘的盘符和虚拟机上的盘符相同，则需将磁盘由“脱机”设置为“联机”。

扩容磁盘容量

当虚拟机磁盘容量不足时，可在已有磁盘容量的基础上，增加该磁盘的容量。但必须满足如下要求：
  • 磁盘使用的数据存储类型为虚拟化存储或FusionStorage。
  • 磁盘不存在快照，若磁盘存在快照，则要提前删除快照。
  • 磁盘不是链接克隆虚拟机的系统磁盘。
  • FusionCompute虚拟化环境下的虚拟机的待扩容磁盘当前容量大于等于4GB。
  • 磁盘所属的数据存储类型为NAS存储，且磁盘的配置模式为普通时，不支持在线增加磁盘容量。

FusionCompute虚拟化环境的虚拟机在“已停止”或“运行中”状态，可以扩容磁盘。其他虚拟机处于“已停止”状态才可以扩容磁盘。

在“资源 > 虚拟机”界面，展开虚拟机详情，在“磁盘”后，单击进入“磁盘”界面，在磁盘所在行，选择“更多 > 扩容”。

使增加磁盘容量生效，还需进行以下操作：
  1. 以下场景要重启虚拟机或关闭虚拟机后再启动虚拟机，其余场景，可直接生效。
    以下场景，需重启虚拟机：
    1. 虚拟机的状态为“运行中”。
    2. 磁盘所属的数据存储类型为虚拟化本地硬盘、虚拟化SAN存储或NAS存储。
    3. 操作系统不为以下操作系统的一种：
      • Windows Server 2003
      • Windows Server 2008
      • Windows XP
      • Windows 7
    以下场景，需停止后再启动虚拟机：
    1. 虚拟机的状态为“运行中”。
    2. 磁盘所属的数据存储类型为FusionStorage。
  2. 由于新增的磁盘容量，在虚拟机操作系统中显示为未分配的磁盘分区，需要合并分区或创建新分区。具体操作方法可在互联网或技术论坛中查找获取。

将磁盘解绑定虚拟机

将绑定在虚拟机中的磁盘解绑定。

对运行中的虚拟机解绑定磁盘时，有如下约束：

  • 虚拟机已安装Tools，且Tools运行正常。

  • 不支持解绑定虚拟机上的系统盘。
  • 不支持解绑定虚拟机上的裸设备映射类型的磁盘。
  • 仅以下操作系统的虚拟机支持在运行时解绑定磁盘：
    • Windows Server 2008 R2 Enterprise 64位
    • Red Hat Linux Enterprise 5.7/5.8/6.5 （32位或64位）
    • CentOS 5.7/5.8/6.5 （32位或64位）

在“资源 > 虚拟机”界面，展开虚拟机详情，在“磁盘”后，单击进入“磁盘”界面，在磁盘所在行，选择“更多 > 解绑定”。

删除磁盘

当磁盘不再使用时，删除磁盘，以释放虚拟资源。删除磁盘的同时，会同时删除所有磁盘数据，请谨慎操作。

说明：
高危特性声明：该功能使用过程中可能触及最终用户数据，符合业界惯例，但仍须谨慎使用。

虚拟机处于“已停止”状态，才可以删除磁盘。

在“资源 > 虚拟机”界面，展开虚拟机详情，在“磁盘”后，单击进入“磁盘”界面，在磁盘所在行，单击“删除”。

调整网卡

添加网卡

为虚拟机添加网卡，实现虚拟机多网卡特性。

虚拟机处于“已停止”或“运行中”状态时，才可以添加网卡。虚拟机处于如下场景时，只有“已停止”状态，才可以添加网卡。
  • 虚拟机为Windows 2000操作系统
  • 虚拟机所在主机使用智能网卡
添加网卡有如下限制条件：
  • 添加网卡前，需已添加MAC地址段或添加独立MAC地址。
  • FusionCompute虚拟化环境的虚拟机每台最多配置12个网卡；VMware虚拟化环境的虚拟机每台最多配置8个网卡。
  • 虚拟机所绑定的网卡中，只允许一个网卡的网关为虚拟机默认网关。若多个网卡均有网关，则需手动在虚拟机中配置策略路由。
  • 虚拟机网卡不支持选择静态注入的网络。
  • 在线添加网卡时，虚拟机需已安装Tools，且Tools运行正常。

在“资源 > 虚拟机”界面，展开虚拟机详情，在“网卡”后，单击进入“网卡”界面，单击“添加”。

修改网卡属性

修改虚拟机网卡属性，调整虚拟机网卡所属网络。

虚拟机处于“已停止”状态时，才可以修改网卡属性。

在“资源 > 虚拟机”界面，展开虚拟机详情，在“网卡”后，单击进入“网卡”界面，网卡所在行，单击“修改”。

配置浮动IP地址

为虚拟机网卡配置浮动IP地址，用于实现虚拟机的主备部署。浮动IP地址来源于网卡所在网络的私有IP地址，因此配置前需在网络中申请私有IP地址。

在“资源 > 虚拟机”界面，展开虚拟机详情，在“网卡”后，单击进入“网卡”界面，网卡所在行，选择“更多 > 配置浮动IP地址”。

加入与退出安全组

若网卡所属VPC中存在安全组，可将网卡加入或退出安全组。

在“资源 > 虚拟机”界面，展开虚拟机详情，在“网卡”后，单击进入“网卡”界面，网卡所在行，选择对应操作。

删除网卡

将虚拟机不再使用的网卡删除，释放虚拟机所占用的网络资源。

虚拟机处于“已停止”或“运行中”状态时，才可以删除网卡。虚拟机处于如下场景时，只有“已停止”状态，才可以删除网卡。
  • 虚拟机为Windows 2000操作系统
  • 虚拟机所在主机使用智能网卡
删除网卡有如下限制条件：
  • 在线删除网卡时，虚拟机需已安装Tools，且Tools运行正常。
  • 虚拟机操作系统为Windows，且其所在主机使用智能网卡时，需按网卡编号顺序从后往前删除。如果删除编号顺序靠前的网卡，会导致该编号往后的网卡的配置信息错位，重新配置网卡后方能正常使用。 例如，Windows虚拟机有3个网卡iNIC 1、iNIC 2和iNIC 3，离线删除iNIC 2后，iNIC 3的配置信息会变更为iNIC 2的数据，需重新配置。

在“资源 > 虚拟机”界面，展开虚拟机详情，在“网卡”后，单击进入“网卡”界面，在网卡所在行，单击“删除”。

若虚拟机操作系统为Linux，则删除网卡后还需手动删除虚拟机中的网卡信息，具体操作方法请参见如何删除Linux虚拟机的网卡信息。

虚拟机快照管理

快照是对虚拟机磁盘某一时刻数据和状态的一种记录。用户可通过快照对虚拟机磁盘进行备份，以便能够使用快照将虚拟机数据恢复至快照创建时刻。

创建快照时，当前磁盘被置为只读，系统自动在磁盘所在数据存储中创建增量磁盘，后续对该磁盘数据的编辑将保存在增量磁盘中，即增量磁盘表示磁盘当前状况和上次执行快照时的状况之间的差异。对该磁盘再次创建快照时，原磁盘和当前增量磁盘均被置为只读，系统会在数据存储中再创建一个增量磁盘。

创建快照

在虚拟机产生重大变更之前，可采用快照方式备份虚拟机数据，以便变更失败出现故障时，能够快速还原虚拟机数据。

快照的创建和使用要求请参见虚拟机快照使用要求。

在“资源 > 虚拟机”界面，展开虚拟机详情，在“快照”后，单击进入“快照”界面，单击“创建快照”。

使用快照恢复虚拟机

在虚拟机故障或需要还原数据时，使用虚拟机已有的快照，将虚拟机的数据恢复至该快照创建时刻的状态。虚拟机当前数据会被覆盖，还原为与快照中保存的数据一致，请谨慎操作。

若虚拟机在创建快照后绑定了新磁盘，则在使用该快照还原虚拟机时，新磁盘会自动与虚拟机解绑定，可根据需要手动重新绑定。

若虚拟机的某个磁盘属性为“快照时不包含该磁盘”，则使用快照还原虚拟机时，不对该磁盘的数据进行还原。
  • 如果快照后，该磁盘被解绑定且未绑定其他虚拟机，则快照恢复的虚拟机会重新绑定该磁盘，但磁盘数据不进行还原。
  • 如果快照后，该磁盘被删除，则快照恢复的虚拟机上不存在该磁盘。

虚拟机处于“已停止”或“已休眠”状态时，才可使用快照进行恢复。

在“资源 > 虚拟机”界面，展开虚拟机详情，在“快照”后，单击进入“快照”界面，选择快照后，单击“恢复虚拟机”。

删除快照

每台虚拟机最多可以创建32个快照，快照会占用磁盘所在的数据存储的空间，为避免存储资源不足，建议对快照数量进行控制，及时删除不需要的快照。

在“资源 > 虚拟机”界面，展开虚拟机详情，在“快照”后，单击进入“快照”界面，选择快照后，单击“删除”。

转为模板

将虚拟机转为虚拟机模板，虚拟机模板可以用来创建其他虚拟机。

虚拟机处于“已停止”状态且需满足如下条件时，才可以转为模板：
  • 虚拟机未绑定共享磁盘。
  • 虚拟机不存在快照。

在“资源 > 虚拟机”界面，虚拟机所在行，选择“更多 > 转为模板”。

修复虚拟机

当运行中的虚拟机出现故障，且无法通过VNC方式登录虚拟机时，可通过虚拟机模板修复该虚拟机的操作系统。在修复过程中，系统会按照模板重新创建虚拟机，并将新虚拟机的系统磁盘挂载到故障虚拟机上，故障虚拟机原有的系统磁盘将作为用户磁盘挂载在故障虚拟机上，原来存放在系统磁盘中的数据不会丢失。

具体操作方法请参见如何修复虚拟机。

在“资源 > 虚拟机”界面，虚拟机所在行，选择“更多 > 修复虚拟机”。

挂载光驱

将本地PC中的ISO文件或者光驱挂载至虚拟机中，即可将ISO或光驱中的文件拷贝至虚拟机中。具体操作方法请参考挂载光驱。
