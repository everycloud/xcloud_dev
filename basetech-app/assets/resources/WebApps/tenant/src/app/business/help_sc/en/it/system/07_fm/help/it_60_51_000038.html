Overview

The access control list (ACL) is a command list that controls the transmission of data packets traversing network devices. It is also used to control incoming and outgoing network traffic, preventing illegitimate use of and access to network resources and improving network access security.

Principles

With the ACL, the system can judge, classify, and filter data streams over routers. ACLs rules can be specified based on the source address, destination address, protocol type, and port number. After the ACL rules apply to router interfaces, the router checks the header information in a data packet against the rules to determine whether to forward or reject the data packet.

For example, as shown in Figure 1, the system controls the traffic over VM 2 based on the following two ACL rules:

  1. Sends the data packet whose source address is 200.8.1.0/24 and the destination address is 10.145.12.0/24 to VM 2.
  2. Discards the data packet whose source address is 10.145.11.0/24 and the destination address is 10.145.12.0/24.
Figure 1 ACL access control

In the ServiceCenter, intra-zone rules control the data traffic between VMs in routed networks within a VPC. Inter-zone rules control the data traffic between VMs in and outside of a VPC. The rules are implemented based on elastic IP addresses.

Web Client Navigation

Apply for a router.

Apply for a router for the VPC to support elastic IP address, network address translation (NAT), access control list (ACL), and virtual private network (VPN) services in the VPC.

 Choose VPC > My VPC, select the target VPC, and choose Routers.

Create a routed network.

The ACL function is implemented by the firewall. It controls the data traffic traversing routed networks. Therefore, routed networks must be created.

 Choose VPC > My VPC, select the target VPC, and choose Network > Network

Apply for public IP addresses.

Before using a public IP address, ensure that you have applied for the public IP address from the FusionManager public IP address pool for the VPC.

 Choose VPC > My VPC, select a VPC, and choose Network > Public IP Address.

Apply for an elastic IP address.

In the public IP address pool created on ServiceCenter, apply for a public IP address to function as an elastic IP address.

Elastic IP addresses are used to configure inter-zone ACL rules.

 Choose VPC > My VPC, select the target VPC, and choose Network > Elastic IP Address.

Create an intra-zone ACL rule.

Create intra-zone ACL rules to control the data traffic between VMs within a VPC.

 Choose VPC > My VPC, select the target VPC, and choose Security > ACLs.

Create an inter-zone ACL rule.

Create inter-zone ACL rules to control the data traffic between VMs in and outside of a VPC.

 Choose VPC > My VPC, select the target VPC, and choose Security > ACLs.
