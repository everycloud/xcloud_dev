VLBs

A virtual load balancer (VLB) provides load balancing services for multiple VMs running the same service in a virtual private cloud (VPC).

ServiceCenter supports hardware load balancers (LBs) and software LBs.
  • Hardware VLB: a VLB created on a physical LB to provide load balancing services
  • Software VLB: a system service VM created on ServiceCenter to provide load balancing services

Creating a VLB

Hardware and software VLBs require different preparations:

  • Before creating a hardware VLB, you need to add a physical LB device to ServiceCenter.
  • Before applying for a software VLB, you need to create an Virtual Service Appliance (VSA) management network and import a VSA VM template to the system.

Adding a Listener

If a VPC contains VMs that provide different services, you can add multiple listeners to the VLB of the VPC and allow each listener to serve a group of VMs that provide the same service in the VPC.

Deleting a Listener

If a VLB listener is not needed any more, you can delete it from the system. After a listener is deleted, the VLB does not provide LB services for VMs bound to the listener.

Modifying Listener Settings

You can modify the Session Stickiness, Health Check, and Allocation Policy settings of a listener.

Binding/Unbinding a VM to/from a VLB

Bind a VM to a VLB or unbind a VM from its bound VLB.

Freezing a VLB

You can freeze a VLB to perform configuration and maintenance operations for the VLB. A frozen VLB cannot provide LB services for VMs.

Unfreezing a VLB

You can unfreeze a VLB so that the VLB can resume providing LB services for VMs.

Binding an Elastic IP Address to a VM NIC

Binding an elastic IP address to a VLB allows the VLB external IP address to be bound to the elastic IP address, so that users can access the service VMs associated with the VLB using the elastic IP address. Before elastic IP address, ensure that you have applied for an elastic IP address for the VPC.

Unbinding the Elastic IP Address from a VLB

After the elastic IP address of a VLB is unbound from the VLB, the VLB will not support access from public networks.

Deleting a VLB

If a VLB is not needed any more, you can delete it from the system.
