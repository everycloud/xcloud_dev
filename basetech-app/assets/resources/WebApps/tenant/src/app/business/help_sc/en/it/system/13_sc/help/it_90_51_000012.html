Waiting for Approval

  • Approving an Application

This page displays all services whose status is To Approve. It is used for the virtual data center (VDC) administrator to handle resource applications submitted by users.

The administrator can handle service applications submitted by users in the following modes: Agree, Reject for Change, and Reject and Close.
  • Agree: After a service application is approved by the administrator, the system automatically provisions resources for users, and resources provisioned can be used by users.
  • Reject for Change: After a service application is rejected by the administrator, users can modify the application and submit it again.
  • Reject and Close: After a service application is closed by the administrator, users cannot modify the original application. Submit an application again if the service is still required.

Approving an Application

After a resource application is submitted, the administrator must approve it. The system automatically creates required resources and provisions them to users only when the resource application is approved.

Cloud host, cloud hard disk, elastic IP address, and application: approved by the VDC administrator or not. You can determines that whether a service needs to be approved during the service creation.
NOTE:
For services that do not need to be approved, the system automatically provisions them after the resource application is submitted by users. Users can use a service after it is successfully provisioned.

Choose Services > My Todo List, and click Approve in Operation from the row where the application locates.
