VPN Gateway

A virtual private network (VPN) provides an encrypted communication channel between remote users and their virtual private clouds (VPCs) so that the users can directly obtain VPC service resources through the VPN. The VPN gateway provides routing functions for VPN channels on a VPC.

In a VPC, only routed networks can connect to the VPN. Therefore, to make VPN services available to a VPC, ensure that you have applied for a virtual router and created a routed network for the VPC.

NOTE:
When configuring the VPN service, the system uses the SHA and MD5 encryption algorithms by default. It also supports the DES and 3DES encryption algorithms. However, these algorithms are known insecure algorithms in the industry. Therefore, ensure that you understand the potential security risks before determining to use these algorithms.

Creating a VPN Gateway

VPNs use elastic IP addresses as their gateway addresses. Therefore, before creating a VPN gateway, ensure that you have applied for an elastic IP address for the VPC that is to connect to the VPN.

Choose VPC > My VPC, select a VPC, choose VPN > VPN Gateways, and click Create VPN Gateway.

Modifying VPN Gateway Settings

Modify the VPN gateway settings as needed. If the VPN gateway is in use by a VPN connection, the gateway settings cannot be modified.

Choose VPC > My VPC, select a VPC, choose VPN > VPN Gateways, and click Modify Settings.

Deleting a VPN Gateway

If a VPN gateway is not needed any more, you can delete it from the system. However, if the VPN gateway is in use by a VPN connection, the gateway cannot be deleted.

Choose VPC > My VPC, select a VPC, choose VPN > VPN Gateways, and click Delete.
