VM Migration and Specification Modification Requirements

VM Live Migration Requirements

VM Requirements
  • The VM is in the Running state.
  • Tools is running properly on the VM.
  • The VM NIC passthrough affinity may not be provided after VM migration.
Computing Resource Requirements
  • The destination host is not in maintenance mode.
  • The destination host has sufficient CPU and memory resources for migrated VMs.
  • The source and destination hosts cannot be powered off during the migration.
  • The source and destination hosts use the same CPU model and specifications.
Storage Resource Requirements

The source and destination hosts can connect to all disks of the VMs, that is, the data stores where the disks are created must be added to both source and destination hosts.

Network Resource Requirements

The source host properly communicates with the destination host, that is, the uplink on the distributed virtual switch (DVS) to which the VM network interface cards (NICs) connect must be associated with both the source and destination hosts.

VM Cold Migration Requirements

VM Requirements
  • Tools is running properly on the VM.
  • The VM is not created using ISO image files.
  • The VM is started from a cloud hard disk.
    NOTE:

    If the VM is created using ISO image files or started from the image, it can be migrated after the host mutual trust relationships are configured. However, configuring the host mutual trust relationships is not recommended because it may cause risks. If you need to configure the mutual trust relationships, visit OpenStack official website and see related configuration information.

  • The VM does not have a temporary disk or a swap disk.
  • The VM NIC passthrough affinity may not be provided after VM migration.
Computing Resource Requirements
  • At least one another host in the system is not in maintenance mode.
  • At least one another host in the system has sufficient CPU and memory resources for the VM to run properly.
Storage Resource Requirements

At least one another host in the system can connect to all disks of the VM, that is, the data stores where the VM disks are created must be added to both the source and destination hosts.

Network Resource Requirements

At least one another host in the system can communicate with the host of the VM to be migrated, that is, the uplink on the DVS to which the VM NICs connect must be associated with both the source and destination hosts.

VM Specification Modification Requirements

NOTE:

VM specification modification is implemented by creating a VM with the same disk data on another host based on the new specification requirements.

VM Requirements
  • Tools is running properly on the VM.
  • The VM is not created using ISO image files.
  • The VM is started from a cloud hard disk.
    NOTE:

    If the VM is created using ISO image files or started from the image, its specifications can be modified after the host mutual trust relationships are configured. However, configuring the host mutual trust relationships is not recommended because it may cause risks. If you need to configure the mutual trust relationships, visit OpenStack official website and see related configuration information.

  • The VM does not have a temporary disk or a swap disk.
Computing Resource Requirements
  • At least one another host in the system is not in maintenance mode.
  • At least one another host in the system has sufficient CPU and memory resources for the VM to run properly.
  • The source and destination hosts cannot be powered off or restarted during the modification.
Storage Resource Requirements

At least one another host in the system can connect to all disks of the VM, that is, the data stores where the VM disks are created must be added to both the source and destination hosts.

Network Resource Requirements

At least one another host in the system can communicate with the host of the VM to be migrated, that is, the uplink on the DVS to which the VM NICs connect must be associated with both the source and destination hosts.
