操作场景

快照是对虚拟机磁盘某一时刻数据和状态的一种记录。用户可通过快照对虚拟机磁盘进行备份，以便能够使用快照将虚拟机数据恢复至快照创建时刻。

创建快照时，当前磁盘被置为只读，系统自动在磁盘所在数据存储中创建增量磁盘，后续对该磁盘数据的编辑将保存在增量磁盘中，即增量磁盘表示磁盘当前状况和上次执行快照时的状况之间的差异。对该磁盘再次创建快照时，原磁盘和当前增量磁盘均被置为只读，系统会在数据存储中再创建一个增量磁盘。

使用要求

  • 一台虚拟机可以创建多个快照，使用其中一个快照恢复虚拟机时，不会对其他快照产生影响。
  • 一台虚拟机最多可以创建32个快照，已创建的快照只能手动进行删除。
  • 虚拟化管理节点虚拟机不能创建快照。

  • 仅当虚拟机磁盘的存储类型为Advanced SAN存储、FusionStorage、虚拟化本地硬盘、虚拟化SAN存储或NAS存储时，虚拟机支持快照功能。

    当虚拟机磁盘的存储类型为本地硬盘、SAN存储、本地内存盘时，默认不对这些磁盘做快照。

  • 包含多种不同存储类型的磁盘的虚拟机，不支持创建快照。
  • 绑定了共享磁盘的虚拟机创建快照时，不对共享磁盘进行快照。
  • 若磁盘被设置为“快照时不包含该磁盘”，则创建快照时，不对该磁盘进行快照。
  • 休眠状态时，默认生成内存快照；已关闭状态时，只能生成普通快照。
  • 虚拟机状态为运行中时，若创建内存快照，快照会保留虚拟机的实时状况；若不创建内存快照，则快照中磁盘数据可能不一致。
  • 快照生成成功后，增量磁盘会占用磁盘所在的数据存储的空间，为避免存储资源不足，建议对快照数量进行控制。
  • 创建快照过程中，不能对虚拟机进行操作。创建完毕后，不能进行存储热迁移。
  • 当虚拟机在进行热迁移、Tools升级、在线增加VCPU、重启、休眠和关机操作时，不能进行虚拟机内存快照创建操作。
  • 当一主机内有虚拟机在创建内存快照时，该主机上的其他虚拟机不能同时创建内存快照。
  • 仅当满足以下要求时，虚拟机支持一致性快照：
    • 虚拟机磁盘的存储类型为虚拟化本地硬盘、虚拟化SAN存储、FusionStorage或NAS存储。
    • 虚拟机的操作系统为Windows 7、Windows Server 2003、Windows Server 2008、Windows Server 2008 R2或Windows Vista。
