Access Control List (ACL) is an access security control method that allows permitted users to access only the specified network resources by controlling the data packets sent to or from a port.

Adding an ACL Rule

Add an ACL rule to a network in a VPC based on the service requirements. An ACL takes effect based on its rules, which are classified into intra-domain rules and inter-domain rules.

  • Intra-domain rules allow communication between routed networks in the same virtual private cloud (VPC).
  • Inter-domain rules allow communication between VPC users and external networks using elastic IP addresses.

FusionManager implements ACL control in both the traffic inbound and outbound directions. Therefore, ACL rules must be configured on both the inbound and outbound ports. For example, to allow external users to access the web server service of a VPC, you can set the ACL rule to permit access on both the inbound Transmission Control Protocol (TCP) port 80 and outbound TCP ports 1024 to 65535.

Choose VPC > My VPC, select a VPC, choose Security > ACLs, select IN or OUT, and click Add Rule.

Deleting an ACL Rule

If an ACL rule is not needed any more, you can delete it from the system.

Choose VPC > My VPC, select a VPC, choose Security > ACLs, select IN or OUT, and click Delete.
