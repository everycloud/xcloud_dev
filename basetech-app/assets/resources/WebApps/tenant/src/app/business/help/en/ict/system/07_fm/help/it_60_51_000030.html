Overview

A virtual private cloud (VPC) provides tenants with isolated and secure networks. You can create virtual networks in a VPC to provide tenants with the same network functions as those provided by a traditional network. Those virtual networks can also run VMs and application instances.

FusionManager supports three types of networks in a VPC: direct network, internal network, and routed network.

Direct Network

A direct network itself contains no network resources, but it has direct access to external networks. Therefore, VMs created on a direct network actually uses the IP addresses provided by the connected external network, which can be the existing network of the customer or the Internet. Internal Network

An internal network uses exclusive virtual local area network (VLAN) or subnet resources. Therefore, it can be used for deploying services that have high security requirements, because it is isolated from other networks. For example, you can deploy the database server on an internal network to prevent the server from being accessed by users from other networks.

Routed Network

A routed network provides flexible communication functions and supports various services. Routed networks in a VPC can communicate with each other or the Internet using elastic IP addresses or the network address translation (NAT) function. Figure 2 shows a routed network.

Create a VPC.

Create a VPC to provide tenants with a secure and isolated network environment. The service networks required by specific tenants must be created in the VPC.

Perform this task only when creating a routed network.
