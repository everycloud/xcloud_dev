Disks

Disks provide data storage space for VMs and are not dependent on VMs. A disk can be attached to different VMs and supports data migration among VMs.

FusionManager also supports disk snapshots for backing up disk data.

Creating a Disk

Create a disk and attach it to a VM to increase the storage space of the VM.

Allocating a VM

Allocate a VM assigned by the system administrator to a specified tenant.

Locate the row that contains the target VM, click More, and select Allocate.

Modifying Disk Attributes

After creating a disk, you can change the disk name if needed.

Locate the row that contains the target disk, click More, and select Modify.

Attaching a Disk to a VM

A disk can store the data of a VM only after being attached to the VM.

Locate the row that contains the target disk, click More, and select Bind VM.

After attaching the disk to the VM, you also need to add the disk to the VM operating system (OS).
  • If the attached disk is a newly created disk, format the disk. For the detailed operation procedure, search from the Internet or Forum.
  • If the attached disk already contains data and the disk drive letter is the same as the disk drive letter on the VM, set the disk status from offline to online.

Detaching a Disk from a VM

Detach a disk from its attached VM.

To detach a disk from a running VM, note the following restrictions:

  • The VM has the Tools run properly on it.

  • The system disk cannot be detached.
  • A raw device mapped (RDM) disk cannot be detached.
  • Only VMs running the following operating systems (OSs) supports disk detaching when the VMs are running:

Expand the disk details area, locate the row that contains the target VM, and click Unbind.

Deleting a Disk

If a disk is not needed any more, you can delete it from the system to release storage resources. After a disk is deleted, all the data on the disk will also be deleted. Therefore, exercise caution when performing this operation.

NOTE:
Declaration: This feature is a high-risk feature. Using this feature complies with industry practices. However, end user data may be required for implementing the feature. Exercise caution and obtain end user's consent when using this feature.
A disk can be deleted in two modes:
  • Ordinarily delete: This mode destroys the file system of the disk so that the data in the disk is not allowed to be retrieved. This mode features rapid deletion but the disk data may be restored using the residual data. Therefore, data security in this deletion mode is poor.
  • Safely delete: This mode deletes disk data by overwriting the data. The overwritten data cannot be restored, and therefore this deletion mode features highest data security. However, the deletion speed in this mode is slow, and the deletion consumes system resources.

Locate the row that contains the target disk and click Delete.
