Routers

You can apply for a virtual router in a virtual private cloud (VPC) to create routed networks. Routed networks in a VPC can communicate with each other. VMs in a routed network can connect to public networks using elastic IP addresses or source network address translation (SNAT) function.

Applying for a Router

A VPC requires only one virtual router.

Releasing a Router

If a router is not needed any more, you can delete it from the system to release router resources. However, if the router is already associated with a network, the router cannot be released.

Enabling SNAT

If SNAT is enabled for a VPC, VMs in a routed network in the VPC can connect to public networks.

Disabling SNAT

If SNAT is disabled for a VPC, VMs in a routed network in the VPC cannot connect to public networks by SNAT.

Add a Route

If a router needs to communicate with another router, you need to add a route to the destination router on the source router.
