A virtual machine (VM) is a software computer that functions like a physical computer, running an operating system (OS) and applications.

A VM runs on a host and obtains CPUs, memory, graphics cards, universal serial bus (USB) devices, network resources, and storage access capabilities of the host. Multiple VMs can concurrently run on a server.

Creating a VM

You can use a VM template to create a VM. During the creation process, you can adjust the VM attributes as needed. However, the adjusted attributes only apply to the newly created VM and will not change the attributes of the VM template.

Choose Resources > VMs and click Create.

To ensure VM security, the VM user must change the VM login password in the earliest time.

Querying the Initial Password for Logging In to the VM

If the VM is created on FusionCompute, and the password is random, you can query the initial password for logging in to the VM.

Choose Resources > VMs, expand VM details, and click View next to Initial Password.

Logging in to a VM Using Virtual Network Computing (VNC)

VNC is a remote terminal access tool. You can use VNC to remotely connect to a VM, view the VM operating page, and perform management and maintenance operations on it.

However, if the VM operating system (OS) requires password input for login, you need to enter the correct OS username and password to access the VM operating page.

Choose Resources > VMs and click VNC.

Note the following points when you log in to a VM using VNC:
  • Declaration: This feature is a high-risk feature. Using this feature complies with industry practices. However, end user data may be required for implementing the feature. Exercise caution and obtain end user's consent when using this feature.This operation involves VM user privacy. Ensure that your login complies with the local laws.

  • To ensure VM security, the VM user must change the VM login password in the earliest time.

  • If you, as an administrator, have logged in to a VM, the VM service VM will not be allowed to log in to the VM before you log out.

If any exception occurs during login, address the issue based on the method described in VNC Login Troubleshooting.

VM Power Management

Power management for a VM includes hibernate, start or wake up, stop, and restart the VM. Stop or restart a VM can be performed in a safe or forcible manner. You are advised to use the safe manner. Forcibly stop or restart a VM only if the VM cannot be safely stopped or restarted due to an OS fault. Forcibly stopping or restarting a VM may damage the VM disks and cause data loss.

Starting or stopping a large number of VMs increases host workload. If you need to start or stop a large number of VMs on a host, perform the start or stop operations in batches to prevent VM services on the host from being adversely affected.

Choose Resources > VMs, locate the row that contains the target VM, click More, and select the desired operation to perform.

Exporting the VM List

You can export all the VMs available in the system to a .xls file and save it on the local PC for information backup and statistics.

Choose Resources > VMs, locate the row that contains the target VM, click More, and select Export.

Monitoring VM Processes

Based on the Hyperic HQ architecture, FusionManager supports monitoring of service VM processes with the following conditions met:
  1. A Proxy VM (PVM) is available in the virtual private cloud (VPC), and the network associated with the PVM VM is connected to the network carrying the service VM to be monitored.
  2. The service VM has the Hyperic HQ software client installed and configured. For details about Hyperic HQ client configuration, see Configuring the VM Process Monitoring Function.
NOTE:

If the administrator has restored or redeployed the PVM due to causes such as PVM faults, the Hyperic HQ clients deployed on VMs can be used only after being reconfigured using the IP address of the new PVM. For details, see FAQ-How Do I Update the Configurations? in Configuring the VM Process Monitoring Function.

Choose Resources > VMs, expand VM details, click View Monitoring Details, navigate to Monitoring > Process, and view the VM process monitoring information.

Modifying CPU Attributes

Modify VM CPU attributes, including the number of CPUs and CPU resource control, to adjust the VM performance.

You can modify CPU attributes only when the VM is in the Running or Stopped state.

Choose Resources > VMs, expand VM details, and click next to CPU.

Modifying Memory Attributes

Modify memory attributes, including the memory size and memory resource control, to adjust the VM performance.

You can modify the memory attributes only when the following requirements are met:
  • VMs in the FusionCompute hypervisor must be in the Running or Stopped state.
  • VMs in the VMware hypervisor must be in the Stopped state.

Choose Resources > VMs, expand VM details, and click next to Memory.

Modifying Disk Capabilities

Adding a Disk

Attach a disk to a VM to increase the storage space of the VM.
  • Create a disk and attach the disk to the VM.
  • Attach an available disk to the VM.

The VM is in the Running or Stopped state. In addition, the host on which the VM is running is not in maintenance mode.

Choose Resources > VMs, expand VM details, and click next to Disk to switch to the Disk page. Then click Add.

After attaching the disk to the VM, you also need to add the disk to the VM operating system (OS).
  • If the attached disk is a newly created disk, format the disk. For the detailed operation procedure, search from the Internet or Forum.
  • If the attached disk already contains data and the disk drive letter is the same as the disk drive letter on the VM, set the disk status from offline to online.

Expanding Disk Capacity

Expand the capacity of a disk on a VM when required. You can expand the disk capacity of a VM only when the following requirements are met:
  • The virtual data store or FusionStorage is used.
  • No snapshots have been created for the disk. If any snapshots have been created for the disk, delete the snapshots before expanding the disk capacity.
  • The disk is not the system disk of a linked clone.
  • For a VM in the FusionCompute hypervisor, the capacity of the VM disk to be expanded is greater than or equal to 4 GB.
  • You cannot expand the disk capacity when the VM is running if the disk is created on NAS storage and the disk is in common configuration mode.

If the VM is in the FusionCompute hypervisor, you can expand the disk capacity of a VM when the VM is in the Stopped or Running state. If the VM is in the other hypervisor, you can expand the disk capacity of a VM only when the VM is in the Stopped state.

Choose Resources > VMs, expand VM details, and click next to Disk to switch to the Disk page. Locate the row that contains the target disk, click More, and select Expand Capacity.

Determine the VM scenario and make capacity expansion take effect.
  1. If the VM is in the following scenario, restart the VM:
    1. The VM is in the Running state.
    2. The disk is created on a Local Pome, LUN Pome, or NAS storage.
    3. The VM does not run one of the following OSs:
      • Windows Server 2003
      • Windows Server 2008
      • Windows XP
      • Windows 7
    If the VM is in the following scenario, power cycle the VM:
    1. The VM is in the Running state.
    2. The disk is created on FusionStorage.
  2. The VM OS recognizes newly expanded disk capacity as an unallocated partition. Therefore, you need to merge the partition or create a partition to identify the expanded disk capacity. For detailed operation methods, search from the Internet or Forum..

Detaching a Disk from a VM

Detach a disk from its attached VM.

To detach a disk from a running VM, note the following restrictions:

  • The VM has the Tools run properly on it.

  • The system disk cannot be detached.
  • A raw device mapped (RDM) disk cannot be detached.
  • Only VMs running the following operating systems (OSs) supports disk detaching when the VMs are running:
    • Windows Server 2008 R2 Enterprise 64–bit
    • Red Hat Linux Enterprise 5.7/5.8/6.5 (32-/64-bit)
    • CentOS 5.7/5.8/6.5 (32-/64-bit)

Choose Resources > VMs, expand VM details, and click next to Disk to switch to the Disk page. Locate the row that contains the target disk, click More, and select Unbind.

Deleting a Disk

If a disk is not needed any more, you can delete it from the system to release storage resources. After a disk is deleted, all the data on the disk will also be deleted. Therefore, exercise caution when performing this operation.

NOTE:
Declaration: This feature is a high-risk feature. Using this feature complies with industry practices. However, end user data may be required for implementing the feature. Exercise caution and obtain end user's consent when using this feature.

You can delete a disk that has been attached to a VM only when the VM is in the Stopped state.

Choose Resources > VMs, expand VM details, and click next to Disk to switch to the Disk page. Locate the row that contains the target disk and click Delete.

Adjusting NICs

Adding a NIC

Add one or more NICs to a VM so that the VM can use multiple NICs.

You can add a NIC only when the VM is in the Running or Stopped state. If the VM is in either of the following scenarios, you can add a NIC only when the VM is in the Stopped state:
  • The VM runs the Windows 2000 OS.
  • The host on which the VM is running uses an intelligent NIC (iNIC).
Be aware of the following limitations before adding a NIC:
  • You have added a MAC address segment or an independent MAC segment to the system.
  • For VMs in the FusionCompute hypervisor, a maximum of 12 NICs can be configured for a VM. For VMs in the VMware hypervisor, a maximum of eight NICs can be configured for a VM.
  • If a VM has multiple NICs bound, only the gateway of one NIC among them is allowed to function as the default gateway of the VM. If all these NICs have gateways configured, you need to manually add a policy route on the VM.
  • The VM NIC does not support networks in Static injection mode.
  • To add a NIC when the VM is running, the VM must be installed with Tools, and Tools is running properly.

Choose Resources > VMs, expand VM details, and click next to NIC to switch to the NIC page. Then click Add.

Modifying NIC Attributes

Modify the NIC attributes of a VM to adjust the network to which the VM NIC belongs.

You can modify the VM attributes only when the VM is in the Stopped state.

Choose Resources > VMs, expand VM details, and click next to NIC to switch to the NIC page. Locate the row that contains the target NIC and click Modify.

Configuring a Floating IP Address

Configure a floating IP address to implement the active/standby VM deployment. The floating IP address is a private IP address on the network of the NIC. Therefore, before configuring the floating IP address, apply for a private IP address.

Choose Resources > VMs, expand VM details, and click next to NIC to switch to the NIC page. Locate the row that contains the target NIC, click More, and select Configure Floating IP Address.

Adding a NIC to or Deleting a NIC from the Security Group

If the VPC to which a NIC belongs contains a security group, you can add the NIC to or delete it from the security group.

Choose Resources > VMs, expand VM details, and click next to NIC to switch to the NIC page. Locate the row that contains the target NIC and perform operations.

Deleting a NIC

Delete a NIC from a VM to release network resources.

You can delete a NIC only when the VM is in the Running or Stopped state. If the VM is in either of the following scenarios, you can add a NIC only when the VM is in the Stopped state:
  • The VM runs the Windows 2000 OS.
  • The host on which the VM is running uses an iNIC.
Be aware of the following limitations before deleting a NIC from a VM:
  • To delete a NIC when the VM is running, the VM must have Tools installed and Tools is running properly.
  • In a system that uses iNICs on hosts and Windows OS on VMs, the iNICs on a VM must be deleted in reverse numerical order. Otherwise, iNIC configuration information will be disordered, and the iNICs that have not been deleted can be used only after reconfiguration. For example, on a Windows VM that has three iNICs, iNIC 1, iNIC 2, and iNIC 3, if iNIC 2 is deleted, configuration information of iNIC 3 will become that of iNIC 2. Therefore, you need to reconfigure iNIC 3 information.

Choose Resources > VMs, expand VM details, and click next to NIC to switch to the NIC page. Locate the row that contains the target NIC and click Delete.

After deleting a NIC from a Linux VM, delete the configuration information of this NIC. For details, see Deleting the NIC Configuration from a Linux VM.

VM Snapshot Management

A snapshot is a reproduction of the data and state of a VM created at a specific time point, and can be used to restore a VM to the state when the snapshot was taken.

When you take a snapshot for a VM, the current disk on the VM is set to be read-only. The system automatically creates an incremental disk in the data store where the disk is located, and the subsequent edit operations on the data of the disk are saved on the incremental disk, that is, the incremental disk records the differences between the current state of the disk and the last snapshot of the disk. When you create another snapshot of the disk, the disk and its current incremental disk are both set to read-only, and the system creates another incremental disk in the data store.

Creating a Snapshot

Snapshots can be created before major changes of the VM. They can be used to restore the VM data if faults occur because the change fails.

For details, see VM Snapshot Usage Requirements.

Choose Resources > VMs, expand VM details, and click next to Snapshot to switch to the Snapshot page. Then click Create Snapshot.

Restoring a VM using a Snapshot

Use the created snapshot to restore the VM data to the state when the snapshot is created if a VM is faulty or requires data restoration. The current data on the VM will be overwritten by the data in the snapshot. Therefore, exercise caution when performing this operation.

If a new disk is attached to a VM after a snapshot is taken for the VM, the disk will be automatically detached from the VM after the VM is restored using this snapshot. If necessary, you can attach this disk to the VM again.

If a disk on a VM is set to Not taken into snapshots, data on the disk will not be restored after the VM is restored using a snapshot.
  • If a snapshot is taken for a VM, and disks on the VM are detached from the VM and not attached to any other VM, the disks will be attached to the VM after the VM is restored using the snapshot. However, data on the disks will not be restored.
  • If a snapshot is taken for a VM and disks on the VM are deleted, the disks will not appear on the VM after the VM is restored using the snapshot.

You can restore a VM using a snapshot only when the VM is in the Stopped or Hibernated state.

Choose Resources > VMs, expand VM details, and click next to Snapshot to switch to the Snapshot page. Select the target snapshot and click Restore VM.

Deleting a Snapshot

For each VM, a maximum of 32 snapshots can be created. Snapshots take space of the data store where the disk is located. To prevent insufficient storage resources, control the number of snapshots and promptly delete snapshots no longer used.

Choose Resources > VMs, expand VM details, and click next to Snapshot to switch to the Snapshot page. Select the target snapshot and click Delete.

Allocating a VM

Allocate a VM assigned by the system administrator to a specified tenant. VM allocation may cause unavailability of the network supported by the VM NIC. Therefore, after allocating a VM to a tenant, modify the network attributes of the VM NIC at the earliest time.

Choose Resources > VMs, locate the row that contains the target VM, click More, and select Allocate.

Add a VM to a Domain

Add a VM to a domain by following the operations described in Domain Management.

Choose Resources > VMs, locate the row that contains the target VM, click More, and select Add to Domain.

Converting a VM into a Template

Convert a common VM into a VM template, which can be used to create other VMs.

The following requirements are met if you want to convert a VM to a template:
  • The VM is in the Stopped state.
  • No shared disk is attached to the VM.
  • The VM does not have snapshots.

Choose Resources > VMs, locate the row that contains the target VM, click More, and select Convert to Template.

Restoring a VM

You can perform this operation when a running VM is faulty, and it cannot be logged in to using VNC. During the restoration, the system creates a VM using the template and attach the system disk of the created VM to the faulty VM. The original system disk of the faulty VM is attached to the created VM as a user disk. Data in the original system disk will not lose.

For details, see Restoring a VM.

Choose Resources > VMs, locate the row that contains the target VM, click More, and select Repair.

Mounting a CD/DVD-ROM Drive to a VM

Mount the ISO image file or CD/DVD-ROM drive in the local PC to the VM, and then the VM can use the ISO image file or DC/DVD-ROM drive to install OS or software. For details, see Mounting a CD/DVD-ROM Drive or an ISO File.

Deleting a VM

If a VM is not needed any more, you can delete it to release VM resources. After a VM is deleted, all the data on the VM disks will also be deleted. Therefore, exercise caution when performing this operation.

Choose Resources > VMs, locate the row that contains the target VM, click More, and select Delete.
