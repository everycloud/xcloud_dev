Networks

FusionManager supports three types of networks in a virtual private cloud (VPC): direct network, internal network, and routed network.
  • Direct network: A direct network itself contains no network resources, but it has direct access to external networks. Therefore, VMs created on a direct network actually uses the IP addresses provided by the connected external network, which can be the existing network of the customer or the Internet.
    NOTE:

    When creating VMs in a direct network, you are advised to configure security groups to isolate the VMs and implement VM access security control.

  • Internal network: An internal network is isolated from other networks and uses exclusive network resources. Therefore, you can deploy services that have high security requirements, for example, database server services, on an internal network to ensure high data security.
  • Routed network: A routed network provides flexible communication functions and supports various services. Routed networks in a VPC can communicate with each other or the Internet. A routed network also provides elastic IP address, access control list (ACL), destination network address translation (DNAT), and virtual private network (VPN) services. Before creating a routed network in a VPC, ensure that you have applied for a virtual router for the VPC.
FusionManager also supports the following IP address allocation modes for VMs in different types of networks to obtain IP addresses:
  • Internal: FusionManager uses its internal Dynamic Host Configuration Protocol (DHCP) server to allocate IP addresses for VMs.
  • Static injection: FusionManager directly assigns an IP address to a VM when the VM is being created.
  • Manual: After a VM is created, the administrator manually configures an IP address for the VM.

Modifying a Subnet

Deleting a Network

If a network is not needed any more, you can delete it from the system. Only networks that do not have any VMs associated with can be deleted.

Applying for a Private IP Address

To manually configure an IP address for a service VM, you need to obtain a private IP address from the routed network. The obtained private IP address is mainly used to configure a floating IP address for the VM.

Associating a Router with an Internal Network

Associate a router to an internal network to make the internal network work as a routed network.

Disassociating the Router from a Routed Network

Disassociate the router from a routed network to make the routed network change to an internal network. Consequently, all the advanced network functions available to the original routed network will not take effect for the internal network any more.

Enabling SNAT

SNAT is used to translate the private IP address of a router into a public IP address, allowing internal users to access external network services using a shared public IP address.

If SNAT is enabled for a VPC, VMs in a routed network in the VPC can connect to public networks.

Disabling SNAT

After SNAT is disabled for a routed network, VMs in the routed network cannot access the public network.
