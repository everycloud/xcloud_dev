Automatic Software Deployment

Overview

FusionManager provides the automatic software deployment function to allow users to rapidly deploy software packages and scripts on VMs in batches.
  • Automatic software installation: When you provision application instances using an application template in which the VM template has specific software configured, the system automatically installs the software on the VM that is to run the application instance.
  • Software distribution: After you select a software package and specify the target VMs to install the package, the system automatically installs the software on the specified VMs.
  • Automatic script execution: When you provision application instances using an application template in which the VM template has specific script file configured, the system automatically executes the script on the VM that is to run the application instance.
  • Script distribution: After you select a script file and specify the target VMs to execute the script file, the system automatically distributes the script to the specified VMs and executes the script on them.

Skill Requirements

The user must be familiar with the software and have the ability to compile code, because the commands and scripts involved in the automatic software deployment need to be designed and compiled based on the software requirements.

Process

As shown in Figure 1, the Proxy VM (PVM) on the Virtual Service Application (VSA) management network pushes the software packages or scripts on the management FusionManager VM to the service VMs. Then, the Puppet Client tool in the service VMs automatically installs the received software or scripts using the software installation and configuration commands or script execution commands.

Precautions

  • Automatic software deployment supports only software and scripts that meet the following conditions:
    • The software supports silent installation, which allows the software to be installed without human interventions or restart during installation.
    • A return value is displayed after software installation or script execution is complete. Value 0 indicates that the operation is successful. Other values indicate that the operation fails.
  • If you need to print operation logs during software installation or script execution, do not print the logs on the operation console. This is to prevent excessive amount of information from being generated on the console. If such a case occurs, the console may fail to return the correct operation result to the system. Instead, you need to configure the function that enables the logs to be redirected to the log file for print.
  • The software can automatically start with the OS after being successfully installed.
  • Automatic software deployment applies only to VMs that meet the following conditions:
    • The VM is created in the FusionCompute or VMware hypervisor.
    • The network type of the VM is not virtual local area network (VLAN), and the network does not have the Domain Network Service (DNS) configured.
    • The VM runs an operating system (OS) listed in Table 1.

Configure Puppet service attributes.

Configure the Puppet service attributes before distributing software to VMs with their OS parameter Supports Automatic Software Installation and Script Execution During Application Instance Creation set to No, as listed in section Precautions.

The following steps use Windows Server 2008 OS as an example.

  1. Log in to the VM using VNC.
  2. Click Start, right-click Computer, and choose Manage.

    The Server Manager dialog box is displayed.

  3. Choose Server Manager > Configuration > Local Users and Groups > Users, right-click the blank area on the right pane, and choose New User.

    The New User dialog box is displayed.

  4. Configure the following parameters and click Create:
    • User name: Enter a username, for example, puppetuser.
    • Password: Enter a password, for example, Admin@123.
    • User must change password at next logon: Keep this parameter unchecked.
    • Password never expires: Select this parameter.
  5. Click Close.

    The Server Manager dialog box is displayed, showing the new user information in the user list.

  6. Right-click the new user and choose Properties.

    The Properties dialog box is displayed.

  7. On the Member Of window, click Add.
  8. Enter Administrators and click OK.

    The Member Of dialog box is displayed, showing the new Administrators member in the member list.

  9. Click OK.
  10. Choose Start > Run.
  11. In the Run dialog box, enter services.msc and click OK.

    The Services window is displayed.

  12. In the service list, right-click gmagent, and click Properties.
  13. On the Log On page, click This account.
  14. Enter the username and password of the new user, for example, .\puppetuser and Admin@123, and click OK.
  15. Right-click gmagent and select Restart to restart the service.
  16. Configure and restart the Puppet Agent service. For details, see the preceding operations for configuring and restarting the gmagent service.

    The Puppet service attributes are successfully configured.
